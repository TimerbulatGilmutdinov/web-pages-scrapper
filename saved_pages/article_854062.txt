xmlns="http://www.w3.org/1999/xhtml"><p>В <a href="https://habr.com/ru/companies/spring_aio/articles/852526/">первой статье</a> из цикла «Как жить без IntelliJ IDEA» мы рассмотрели альтернативы привычной нам IDE как таковой. TL;DR: по мнению команды <a href="https://t.me/+QmrKn0wA8CdkNjAy">Spring АйО</a> проще собрать IntelliJ IDEA Community самостоятельно (если все возможные пути легального скачивания будут заблокированы), чем пересаживаться на что-то другое (VS Code, NetBeans, Eclipse).</p><p>Если первая статья была сконцентрирована именно на IDE как таковой и всех её возможностях в целом, к которым разработчики привыкли за последние десятилетия, то в следующих статьях из этого цикла мы сделаем акцент на более конкретных возможностях IDE. Начнем, конечно же, с поддержки Spring.</p><hr/><h2>Содержание</h2><ul><li><p><a href="https://habr.com/ru/companies/spring_aio/articles/854062#what-is-spring-support">А что есть поддержка Spring?</a></p></li><li><p><a href="https://habr.com/ru/companies/spring_aio/articles/854062#spring-tools-suite-4">Spring Tools Suit 4</a></p></li><li><p><a href="https://habr.com/ru/companies/spring_aio/articles/854062#netbeans">NetBeans</a></p></li><li><p><a href="https://habr.com/ru/companies/spring_aio/articles/854062#community-edition">Community Edition</a></p><ul><li><p><a href="https://habr.com/ru/companies/spring_aio/articles/854062#jetbrains-merketplace">JetBrains Marketplace</a></p></li><li><p><a href="https://habr.com/ru/companies/spring_aio/articles/854062#amplicode">Amplicode</a></p></li></ul></li><li><p><a href="https://habr.com/ru/companies/spring_aio/articles/854062#conclusion">Заключение</a></p></li></ul><hr/><p>Не смотря на то, что в предыдущей статье мы решили оставаться на IntelliJ IDEA, не стоит забывать, что в Community версии нет поддержки Spring от слова совсем. Прекрасный повод дать ещё один шанс IDE не от компании JetBrains, не так ли?</p><a class="anchor" id="what-is-spring-support" name="what-is-spring-support"></a><h2>А что есть поддержка Spring?</h2><p>Но прежде чем приступить к обзору, нужно определить критерии оценивания. В качестве отправной точки возьмем <a href="https://www.jetbrains.com/idea/spring/">лэндинг IntelliJ IDEA Ultimate</a>, описывающий поддержку Spring и дополним его фичами, к которым все давно привыкли:</p><ol><li><p>Создание Spring Boot приложения (альтернатива <a href="https://start.spring.io/">https://start.spring.io/</a>)</p></li><li><p>Умные автодополнения в контексте Spring</p></li><li><p>Инспекции и быстрые исправления</p></li><li><p>Навигация по Spring модели</p></li><li><p>Диаграмма зависимостей между бинами</p></li><li><p>Отображение доступных эндпоинтов приложения в одном месте</p></li><li><p>Поддержка application.properties/.yaml</p></li><li><p>Поддержка JPA (Spring Data, JPQL)</p></li><li><p>Поддержка Kotlin</p></li></ol><p>Последний, десятый пункт давайте назовём «Общее впечатление». Как‑никак, довольно высока вероятность, что общее впечатление от того или иного решения будет зависеть от множества факторов, в том числе от фич, которых может и не быть в Ultimate версии IntelliJ IDEA.</p><a class="anchor" id="spring-tools-suite-4" name="spring-tools-suite-4"></a><h2>Spring Tools Suite 4</h2><p><a href="https://spring.io/tools">Spring Tools 4</a> — это инструмент, который рекомендуют использовать сами разработчики Spring Framework. С другой стороны, рекомендуют — не значит, что используют сами. Во время публичных выступлений контрибьюторы Spring Framework постоянно используют именно IntelliJ IDEA, а не любую другую IDE со Spring Tools Suite. Вот несколько примеров: <a href="https://www.youtube.com/watch?v=YfFYTR8_lBY">Juergen Hoell</a>, <a href="https://youtu.be/NCd3-JuB5NE?si=muVaCkYw3mo5ERAH">Josh Long</a>, <a href="https://youtu.be/TrCLf9zAQfs?si=FYSob9YVd1Ujv75x">Daniel Garnier‑Moiroux</a>, <a href="https://youtu.be/8vtoa8VC-gY?si=TgXLRdDa4BG14ceU">Stéphane Nicoll</a>, и т. д. </p><p>Инструмент представлен в нескольких вариациях — для VS Code, Eclipse и Theia (можно сказать, тот же VS Code). </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/2d3/775/45d/2d377545dd38418310a93438e3bc8fa1.png" height="1562" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/2d3/775/45d/2d377545dd38418310a93438e3bc8fa1.png" width="1022"/></figure><p>Учитывая, что SpringTools работает по технологии <a href="https://microsoft.github.io/language-server-protocol/">LSP</a>, функциональность в Eclipse и VS Code будет одинаковой в точности до UI/UX этих IDE, поэтому мы рассмотрели VS Code версию этого инструмента.</p><p>На хабре уже была <a href="https://habr.com/ru/companies/cdek_blog/articles/811631">статья</a> про Java разработку на VS Code, но без упора на Spring. Постараемся закрыть этот пробел.</p><p>Устанавливается Spring Tools 4 на VS Code буквально в 2 клика. Предлагается установить <a href="https://marketplace.visualstudio.com/items?itemName=vmware.vscode-boot-dev-pack">Spring Boot Extension Pack</a> — набор сразу из 3 расширений:</p><ul><li><p><a href="https://marketplace.visualstudio.com/items?itemName=vscjava.vscode-spring-initializr">Spring Initializr Java Support</a> — название говорит само за себя</p></li><li><p><a href="https://marketplace.visualstudio.com/items?itemName=vmware.vscode-spring-boot">Spring Boot Tools</a> — собственно, языковая поддержка application.properties, application.yaml и.java файлов</p></li><li><p><a href="https://marketplace.visualstudio.com/items?itemName=vscjava.vscode-spring-boot-dashboard">Spring Boot Dashboard</a> — панелька для отображения проектов, бинов и эндпоинтов</p></li></ul><h3>1. Создание Spring Boot приложения</h3><p>Для разработчиков, привыкших к IntelliJ IDEA, будет несколько непривычным способ запуска различной функциональности в VS Code - через <code>Command Palette</code>. Жмём специальный hotkey (на OS X это <code>Command + Shift + A</code>) и получаем окошко, в которое вводим название команды.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/221/0a1/480/2210a14808ba7c60c151fb2e836012e1.png" height="254" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/221/0a1/480/2210a14808ba7c60c151fb2e836012e1.png" width="1440"/></figure><p>Похоже на <a href="https://www.jetbrains.com/help/idea/searching-everywhere.html">Shift-Shift</a> из IDEA, но в данном случае это буквально единственный способ доступа к Spring Initializr.</p><p>После этого начинается долгое путешествие по таким же попапчикам, в которых надо выбрать все то, что обычно мы вводим на <a href="https://start.spring.io/">https://start.spring.io/</a>. Особенно интересно выглядит окошко для выбора зависимостей. В принципе, каких-то сложностей такой UI не вызывает, но нужно быть осторожным и случайно его не закрыть. более-менее популярные плагины (с &gt;10k скачиваний) либо предоставляют примерно такие же возможности, либо предлагают частичную поддержку отдельных Spring элементов. Отметим факт потенциальных возможностей, которые в теории можно найти символическими 3 баллами в графе «Общее впечатление».</p><p>Итого:</p><div><div class="table"><table><tbody><tr><td><p align="left"><strong>Фича</strong></p></td><td><p align="left"><strong>Краткий итог</strong></p></td></tr><tr><td><p align="left">Создание Spring Boot приложения</p></td><td><p align="left">Работает, но браузер в IDE – такое себе (8/10)</p></td></tr><tr><td><p align="left">Умные автодополнения в контексте Spring</p></td><td><p align="left">Отсутствуют (0/10)</p></td></tr><tr><td><p align="left">Инспекции и быстрые исправления</p></td><td><p align="left">JPA Buddy предоставляет инспекции, связанные именно с JPA, но не со Spring в целом (2/10)</p></td></tr><tr><td><p align="left">Навигация по Spring модели</p></td><td><p align="left">Отсутствует, есть навигация только по JPA модели (2/10)</p></td></tr><tr><td><p align="left">Диаграмма зависимостей между бинами</p></td><td><p align="left">Отсутствует (0/10)</p></td></tr><tr><td><p align="left">Отображение доступных эндпоинтов приложения в одном месте</p></td><td><p align="left">Отсутствует (0/10)</p></td></tr><tr><td><p align="left">Поддержка application.properties/.yaml</p></td><td><p align="left">Такая же, как и в Ultimate (10/10)</p></td></tr><tr><td><p align="left">Поддержка JPA (Spring Data, JPQL)</p></td><td><p align="left">Поддержка есть, но полная. Например, нет поддержки JPQL (7/10)</p></td></tr><tr><td><p align="left">Поддержка Kotlin</p></td><td><p align="left">Так как это IntelliJ IDEA, поддержку Kotlin мы получаем из коробки (10/10)</p></td></tr><tr><td><p align="left">Общее впечатление</p></td><td><p align="left">Потенциально необнаруженные полезные плагины и тот факт, что это привычная нам IDE (5/10)</p></td></tr></tbody></table></div></div><p><strong>Общий результат 44 балла из 100.</strong></p><a class="anchor" id="amplicode" name="amplicode"></a><h3>Amplicode</h3><p><a href="https://amplicode.ru/">Amplicode</a> ​​предоставляет инструменты для эффективной разработки веб-приложений и сервисов на Spring Boot, а также административного интерфейса на React Admin. Плагин от российских разработчиков, поэтому из официального маркетплейса JetBrains его не скачать. Нужно будет указать кастомный.</p><h4>1. Создание Spring Boot приложения</h4><p>Такой возможности Amplicode не предоставляет, во всех маркетинговых материалах на сайте и YouTube производится доработка существующего приложения, либо создание нового на <a href="https://start.spring.io/">https://start.spring.io/</a>.</p><h4>2. Умные автодополнения в контексте Spring</h4><p>Есть live templates для создания методов в контроллерах:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/0c6/eb2/ee2/0c6eb2ee2934d7568b0a43afbe817651.png" height="394" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/0c6/eb2/ee2/0c6eb2ee2934d7568b0a43afbe817651.png" width="1600"/></figure><p>Инжекция бинов в класс после набора названия бина в методе работает также, как и в последних версиях IntelliJ IDEA Ultimate. </p><p>Печатаем название бина и нажимаем Enter:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/0ed/c97/35a/0edc9735a129c1f7bf4440b1f64c6805.png" height="817" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/0ed/c97/35a/0edc9735a129c1f7bf4440b1f64c6805.png" width="1600"/></figure><p>Получаем его инжекцию в текущий бин:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/145/eb2/d80/145eb2d80c882ab3df53344fb455d9dd.png" height="819" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/145/eb2/d80/145eb2d80c882ab3df53344fb455d9dd.png" width="1600"/></figure><p>Схожая по принципу поддержка с автодополнениями есть и для MapStruct. Начинаем печатать <code>mapTo</code> на доменном объекте или коллекции объектов:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/c2c/0e2/2bc/c2c0e22bc0b45e9f26bf4dc1a7d98d5e.png" height="650" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/c2c/0e2/2bc/c2c0e22bc0b45e9f26bf4dc1a7d98d5e.png" width="1600"/></figure><p>Нажимаем Enter, инжектируется маппер, и скафолдится код для маппинга:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/b9b/d50/997/b9bd50997345617dd24ebb957aae3f14.png" height="1108" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/b9b/d50/997/b9bd50997345617dd24ebb957aae3f14.png" width="1214"/></figure><h4>3. Инспекции и быстрые исправления</h4><p>Amplicode предоставляет не все инспекции, что есть в IntelliJ IDEA Ultimate, но довольно много. У нас получилось насчитать 31 инспекцию. Многие из них связаны с JPA, другие именно со Spring или связанными технологиями:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/120/fe4/363/120fe4363d7cbdd296dc7912c2db62bc.png" height="966" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/120/fe4/363/120fe4363d7cbdd296dc7912c2db62bc.png" width="1600"/></figure><h4>4. Навигация по Spring модели</h4><p>Навигация по Spring модели осуществляется схожим с IntelliJ IDEA Ultimate образом, через иконку рядом с названием бина. При этом, Amplicode показывает зависимости транзитивно и в две стороны:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/1b7/429/e23/1b7429e23e1024d6db40c9ba1d6a87ac.png" height="912" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/1b7/429/e23/1b7429e23e1024d6db40c9ba1d6a87ac.png" width="1280"/></figure><h4>5. Диаграмма зависимостей между бинами</h4><p>Диаграммы зависимостей как таковой Amplicode не предоставляет. Однако стоит отметить, что представленная выше фича с возможностью просмотра заинжектированных бинов может частично заменить диаграмму. </p><p>Для изучения самого проекта Amplicode предлагает панель Amplicode Explorer, в которой можно посмотреть на различные части Spring Boot приложения. От добавленных в проект конфигураций и до файлов развертывания:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/6fd/eca/de3/6fdecade3c2b119cbea39fe3b42d3c4b.png" height="1600" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/6fd/eca/de3/6fdecade3c2b119cbea39fe3b42d3c4b.png" width="960"/></figure><h4>6. Отображение доступных эндпоинтов приложения в одном месте</h4><p>В описанной выше панели Amplicode Explorer есть отдельная группа Endpoints, в которой можно найти все эндпоинты и перейти к ним. Визуальное отображение можно настроить: по пути, по контроллерам или в плоском виде:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/952/395/222/952395222a3e4f67f86572f663ba09ff.png" height="972" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/952/395/222/952395222a3e4f67f86572f663ba09ff.png" width="1016"/></figure><h4>7. Поддержка application.properties/.yaml </h4><p>Amplicode предоставляет автодополнения во время набора кода в файлах <code>application.properties</code> и <code>application.yaml</code> ровно также, как и IntelliJ IDEA Ultimate:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/c4f/070/166/c4f070166ad19a56e650cfbe9077d4fa.png" height="552" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/c4f/070/166/c4f070166ad19a56e650cfbe9077d4fa.png" width="1276"/></figure><p>Также есть возможность изменять и исследовать возможные значения в другой панели – Amplicode Designer:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/7a6/fed/bfa/7a6fedbfa3df6b7ab9d7f45f7f9d0d00.png" height="1007" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/7a6/fed/bfa/7a6fedbfa3df6b7ab9d7f45f7f9d0d00.png" width="1600"/></figure><h2>8. Поддержка JPA (Spring Data, JPQL) </h2><p>Так как Amplicode разрабатывает та же команда, что и JPA Buddy, все возможности, которые были в нём, и в том числе те, что теперь <a href="https://www.jetbrains.com/help/idea/jpa-buddy-paid-features.html">входят в состав IntelliJ IDEA Ultimate</a>, доступны и в Amplicode.</p><p>Поэтому всё, что связано с JPA Buddy и было описано выше, есть в Amplicode (создание JPA сущностей, Spring Data репозиториев, <code>derived</code> и <code>@Query</code> методов). Помимо этого, есть поддержка JPQL:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/fda/fba/1bf/fdafba1bf6c74f0391dbe1ddb2021500.png" height="688" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/fda/fba/1bf/fdafba1bf6c74f0391dbe1ddb2021500.png" width="1600"/></figure><p>И автодополнения во время набора <code>derived</code> методов как в IntelliJ IDEA Ultimate:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/cc8/201/af1/cc8201af1567a2bd39fabe176d1c8db2.png" height="812" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/cc8/201/af1/cc8201af1567a2bd39fabe176d1c8db2.png" width="1600"/></figure><h4>9. Поддержка Kotlin </h4><p>Все возможности, описанные выше, также работают и в Kotlin проектах.</p><h2>10. Также расстраивает отсутствие поддержки XML-конфигураций, хотя с другой стороны кажется, что на данный момент они не пользуются большой популярностью. Кроме того, есть возможности, которых нет в Ultimate, но которые выглядят интересно.
