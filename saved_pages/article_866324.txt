article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p>Я работаю преподавателем в онлайн-школе и обучаю студентов программированию. Периодически приходится рассказывать о некоторых функциях IntelliJ IDEA. На хабре не нашлось подходящей статьи с краткой выжимкой наиболее используемых функций. Кроме того, возникало несколько ситуаций, когда требовалось оптимизировать рутинные задачи на работе. Так и родилась идея этой статьи. Думаю, что даже разработчик с опытом в пару лет смогут найти для себя что-то новое и полезное.</p><h2>Горячие клавиши</h2><p>Начать хочу с базовой вещи - это горячие клавиши. Это не только увеличение скорости написания кода, но и сохранение концентрации. По себе замечал, что пока найдешь нужный файл, то контекст прошлого файла из головы уже улетучился. Поэтому с горячими клавишами работа становится более легкой. Ниже привожу список хоткеев, которые сам регулярно использую.</p><h3>Навигация</h3><ul><li><p>Последние файлы = Ctrl + Е</p></li><li><p>Перейти обратно = Ctrl + Alt + Left</p></li><li><p>Последние места редактирования = Ctrl + Shift + E</p></li><li><p>Структура файла = Ctrl + F12</p></li><li><p>Дерево навигации = Alt + home</p></li><li><p>Перейти к последнему редактированному участку кода = Ctrl + Shift + Backspace</p></li><li><p>Горизонтальная прокрутка = Shift + колесо мыши</p></li><li><p>Переключаться между вкладками = Alt + (Влево/Вправо)</p></li><li><p>Переход к следующей подчеркнутой ошибке в коде = F2</p></li></ul><h3>Рефакторинг кода</h3><ul><li><p>Подсказки конструктора или метода = Ctrl + Р</p></li><li><p>Обернуть код какой-то конструкцией = Ctrl + Alt + T</p></li><li><p>Изменение имени переменной = Shift + F6</p></li><li><p>Автогенерация кода = Alt + Insert</p></li><li><p>Удалить текущую строку = Ctrl + Y</p></li><li><p>Двигать функцию вверх / вниз = Ctrl + Shift + (Вверх/Вниз)</p></li><li><p>Двигать строку вверх / вниз = Alt + Shift + (Вверх/Вниз)</p></li><li><p>Заменить параметр енама = Ctrl + Space -&gt; Tab</p></li><li><p>Подвинуть строку по табам влево = Shift + Tab</p></li></ul><h3>Визуал</h3><ul><li><p>Поправить стиль кода = Ctrl + Alt + L</p></li><li><p>Выделить окружение строки - Ctrl + W</p></li><li><p>Закомментировать по одной строке = Ctrl + Slash</p></li><li><p>Закомментировать блок кода = Ctrl + Shift + Slash</p></li><li><p>Схлопнуть функцию = Ctrl + Minus</p></li><li><p>Развернуть функцию = Ctrl + Plus</p></li></ul><h3>Тренировки</h3><p>Я и сам много раз гуглил что-то вроде "топ горячих клавиш для IntelliJ IDEA", находил большой список и закрывал этот сайт. Новые навыки закрепляются только после тренировок, для этого в IntelliJ IDEA есть встроенный тренажер. Открыть его можно в разделе (Help -&gt; Learn Ide Features).</p><p>Кроме этого, для запоминания прекрасно поможет плагин <code>Key promoter</code>, об этом будет ниже.</p><h2>История локальных изменений</h2><figure class="full-width"><img alt="Рис. Интерфейс Local History" data-src="https://habrastorage.org/getpro/habr/upload_files/f34/c7c/726/f34c7c726dd75759a02c51365acfa976.png" height="837" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/f34/c7c/726/f34c7c726dd75759a02c51365acfa976.png" title="Рис. Интерфейс Local History" width="1722"/><div><figcaption>Рис. Интерфейс Local History</figcaption></div></figure><p>IntelliJ IDEA предоставляет встроенный инструмент для отслеживания изменений в проекте — <strong>Local History</strong>. Эта функция автоматически сохраняет версии файлов, позволяя возвращаться к предыдущим состояниям кода даже без использования  Git. Local History работает в фоновом режиме, записывая каждое изменение в файлы проекта, и предлагает удобный интерфейс для просмотра и восстановления этих изменений.</p><h4>Основные возможности Local History</h4><ol><li><p><strong>Автоматическая фиксация изменений:</strong> Local History автоматически создает новые версии файлов при каждом сохранении, компиляции или запуске программы. Это позволяет отслеживать изменения в реальном времени и не беспокоиться о ручной фиксации версий.</p></li><li><p><strong>Просмотр истории изменений:</strong> Разработчик может просматривать историю изменений любого файла в проекте. В специальном окне будут представлены все зафиксированные версии файла с указанием времени и даты изменения.</p></li><li><p><strong>Сравнение версий:</strong> Инструмент позволяет сравнивать текущую версию файла с любой из предыдущих. Это удобно для анализа изменений и выявления различий между двумя версиями кода. Для сравнения версий нужно выбрать две интересующие версии в окне истории и нажать кнопку "Compare with Current".</p></li><li><p><strong>Восстановление предыдущих состояний:</strong> Если необходимо вернуть файл к одному из предыдущих состояний, достаточно выбрать нужную версию в окне истории и нажать кнопку "Revert". Это действие отменит все последующие изменения и восстановит выбранную версию файла.</p></li><li><p><strong>Отмена изменений:</strong> Помимо полного отката к предыдущей версии, Local History позволяет отменить отдельные изменения. Для этого нужно сравнить текущую версию с нужной и вручную выбрать, какие изменения стоит оставить, а какие — удалить.</p></li><li><p><strong>Настройка частоты фиксации:</strong> Пользователь может настроить частоту автоматической фиксации изменений в настройках IntelliJ IDEA. Это позволяет оптимизировать работу инструмента под конкретные нужды проекта.</p></li></ol><h4>Примеры использования</h4><ol><li><p><strong>Восстановление удаленного фрагмента кода:</strong> Допустим, вы удалили важный фрагмент кода и сохранили файл. С помощью Local History можно быстро вернуться к предыдущей версии файла и восстановить утраченное без отката других изменений</p></li><li><p><strong>Просмотр изменений за определенный период:</strong> Если нужно просмотреть, какие изменения были внесены в проект за последние сутки или неделю, это легко можно сделать в интерфейсе Local History и детально изучить каждое изменение.</p></li><li><p><strong>Работа без Git:</strong> Даже если вы не используете систему контроля версий (Git), Local History обеспечит базовую защиту от потери данных и возможность возврата к предыдущим состояниям кода. Такая ситуация может возникнуть при написании пет проектов.</p></li></ol><p><strong>Как открыть:</strong></p><ol><li><p>ПКМ -&gt; Local History -&gt; Show History</p></li><li><p>File -&gt; Local history -&gt; Show History</p></li></ol><h2>Постфиксное дополнение (Postfix Completion)</h2><p>Постфиксное дополнение (Postfix Completion) — это удобная функция в IntelliJ IDEA, позволяющая быстро генерировать кодовые шаблоны на основе сокращений. Кроме того, эта функция позволяет автоматически включать наши переменные в эти шаблоны.</p><h4>Принцип работы</h4><p>Постфиксное дополнение основано на наборе заранее настроенных правил, которые составляют пару из сочетания букв и символов и соответствующего этому сочетанию шаблона кода. Думаю, все знают про то, что <code>sout</code> заменяется на <code>System.out.println()</code>.</p><p>Однако в этом шаблоне не используется никакая переменная. Рассмотрим следующий пример кода:</p><pre><code class="java">int [] numbers = {1,2,3,4,5};

numbers.for
</code></pre><p>У нас есть массив <code>numbers</code>, при вводе <code>numbers.for</code> IntelliJ IDEA автоматически создает шаблон цикла с использованием массива:</p><pre><code class="java">int [] numbers = {1,2,3,4,5};  
  
for (int number : numbers) {  
  
}
</code></pre><p>Можно просмотреть все варианты ввода, нажав Ctrl + J на дефолтных настройках горячих клавиш.</p><h4>Основные примеры использования</h4><ol><li><p><strong>Генерация циклов:</strong> </p><ul><li><p><code>fori</code> → <code>for (int i = 0; i &amp;lt; array.length; i++) {}</code></p></li><li><p><code>for</code> → <code>for (Element element : elements) {}</code></p></li></ul></li><li><p><strong>Создание методов:</strong> </p><ul><li><p><code>psvm</code> → <code>public static void main(String[] args) {}</code></p></li><li><p><code>soutv</code> → <code>System.out.println("value=" + value);</code></p></li></ul></li><li><p><strong>Работа с объектами:</strong> </p><ul><li><p><code>new</code> → Конструктор объекта с заполненными параметрами.</p></li><li><p><code>cast</code> → Приведение типа.</p></li></ul></li></ol><h4>Настройки</h4><p>Пользователи могут настраивать постфиксы в соответствии со своими потребностями: редактировать стандартные и добавлять свои. Для этого нужно перейти в настройки IntelliJ IDEA (File → Settings → Editor → General → Postfix Completion) и добавить или изменить существующие правила.</p><h2>Пресеты (File and Code Templates)</h2><p>Шаблоны файлов и кода (File and Code Templates) в IntelliJ IDEA — это инструмент, который помогает ускорить процесс создания новых файлов и структур кода, соответствующих принятым стандартам и соглашениям, избегая необходимости повторного ввода одинаковых фрагментов.</p><p>Мне очень помогла эта функция, когда нужно было написать одинаковые классы для новых эндпоинтов в рабочем проекте: сущность, репозиторий, сервис, контроллер, маппер и тесты.</p><h4>Что такое File Templates?</h4><p>Шаблоны файлов (File Templates) — это предустановленные структуры для новых файлов, которые создаются в проекте. Например, при создании нового класса Java, IntelliJ IDEA автоматически добавляет стандартный заголовок с комментариями, информацию о пакете и основной каркас класса. Пользовательские шаблоны включать любой код и комментарии.</p><figure class="full-width"><img alt="Рис. Самый простой вариант шаблона для класса с методом main." data-src="https://habrastorage.org/getpro/habr/upload_files/81d/3a8/5d9/81d3a85d94d91859a6e8ff1c4c6a7ddf.png" height="844" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/81d/3a8/5d9/81d3a85d94d91859a6e8ff1c4c6a7ddf.png" title="Рис. Самый простой вариант шаблона для класса с методом main." width="1674"/><div><figcaption>Рис. Самый простой вариант шаблона для класса с методом main.</figcaption></div></figure><h4>Что такое Code Templates?</h4><p>Шаблоны кода (Code Templates) предназначены для быстрой генерации стандартных блоков кода, например, при вводе <code>/**</code> над методом создастся JavaDocs для этого метода с учетом входных параметров метода, выходного параметра и ошибок в секции throws.</p><figure class="full-width"><img alt="Рис. Шаблон JavaDocs" width="747"/><div><figcaption>Рис. Для этого нужно перейти в настройки IDE (File → Settings → Editor → File and Code Templates) и изменить существующие шаблоны или создать новые.</p><h2>Конфигурация вкладок</h2><p>По умолчанию вкладки располагаются сверху, над редактором кода. Это неудобно при дебаге: постоянно открываются новые вкладки, а старые при лимите закрываются. Конфигурацию вкладов можно настроить: переместить вкладки из строки над редактором в колонку справа, а еще увеличить лимит вкладок для закрытия предыдущих. Для этого нужно перейти в раздел настроек (File -&gt; Settings -&gt; Editor -&gt; Editor Tabs)</p><figure class="full-width"><img alt="Рис. Пример расположения вкладок" data-src="https://habrastorage.org/getpro/habr/upload_files/151/7a2/dd2/1517a2dd2da0189067d4f68c9e3382de.png" height="1008" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/151/7a2/dd2/1517a2dd2da0189067d4f68c9e3382de.png" title="Рис. Пример расположения вкладок" width="1323"/><div><figcaption>Рис. Пример расположения вкладок</figcaption></div></figure><h2>Плагины</h2><p>Установка плагинов сейчас усложнилась в силу определенных событий. Сейчас последовательность действий такая:</p><ol><li><p>Нужно узнать подходящую версию плагина для своей версии IntelliJ IDEA (это можно сделать найдя плагин в разделе File -&gt; Settings -&gt; Plugins)</p></li><li><p>Далее нужно с использованием спец средств для смены локации скачать zip архив нужной версии плагина с сайта JetBrains</p></li><li><p>А затем нужно установить этот плагин в том же разделе File -&gt; Settings -&gt; Plugins, в шестеренке будет опция "Install plugin from Disk"</p></li></ol><h3>Key promoter X</h3><p>Key Promoter X — это плагин, который помогает пользователям освоить и запомнить горячие клавиши. Главная цель плагина — повысить производительность разработчиков, предлагая подсказки и напоминания о полезных сочетаниях клавиш.</p><figure class="full-width"><img alt="Рис. Пример всплывающего окна" data-src="https://habrastorage.org/getpro/habr/upload_files/ee0/ca8/eec/ee0ca8eec865105fc024a89f320370a4.png" height="968" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/ee0/ca8/eec/ee0ca8eec865105fc024a89f320370a4.png" title="Рис. Пример всплывающего окна" width="1756"/><div><figcaption>Рис. Пример всплывающего окна</figcaption></div></figure><p><strong>Функционал</strong></p><ol><li><p><strong>Подсказки:</strong> Key Promoter X показывает всплывающую подсказку с предложением горячей клавиши всякий раз, когда пользователь пытается выполнить действие с помощью мыши.</p></li><li><p><strong>Рекомендации:</strong> Плагин анализирует действия пользователя и предлагает подходящие горячие клавиши для оптимизации работы.</p></li><li><p><strong>Обучение:</strong> Key Promoter X обучает использованию горячих клавиш, показывая статистику прогресса освоения и запоминание.</p></li></ol><h3>Rainbow brackets</h3><p>Rainbow Brackets — это популярный плагин, который помогает визуально выделять скобки и парные элементы в коде. Основная цель этого плагина заключается в том, чтобы облегчить чтение и написание кода.</p><h3>SonarLint</h3><p>SonarLint – это статический анализатор кода. Его основная цель – выявление потенциальных ошибок, проблем безопасности и нарушений стандартов кодирования прямо во время написания кода. Плагин поддерживает различные языки программирования, включая Java, Python, JavaScript и другие.</p><p>Основные функции SonarLint:</p><ul><li><p><strong>Анализ в реальном времени</strong>: Плагин анализирует код сразу после его ввода, предоставляя мгновенную обратную связь о возможных проблемах.</p></li><li><p><strong>Интеграция с SonarQube</strong>: При необходимости результаты анализа можно синхронизировать с сервером SonarQube для централизованного управления качеством кода.</p></li><li><p><strong>Настраиваемые правила</strong>: Есть возможность настройки кастомных правил проверки, специфичных для компании или проекта.</p></li></ul><p>Ниже представлен пример анализа с указанием ошибок:</p><figure class="full-width"><img alt="Рис. Пример результата анализа плагином SonarLint" data-src="https://habrastorage.org/getpro/habr/upload_files/93b/1c1/5c5/93b1c15c55de2a599a198c01218d5ec7.png" height="896" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/93b/1c1/5c5/93b1c15c55de2a599a198c01218d5ec7.png" title="Рис. Пример результата анализа плагином SonarLint" width="951"/><div><figcaption>Рис. Пример результата анализа плагином SonarLint</figcaption></div></figure><h2>Gigacode</h2><p>Gigacode - это AI-плагин, который помогает генерировать, редактировать и объяснять код. Кроме этого, можно в специальном чате задавать вопросы. Ниже приведу некоторые примеры использования плагина.</p><h3>1) Inline подсказки</h3><p>Inline подсказки - это варианты окончания строки, в процессе набора кода. В некоторых случаях плагин предлагает не одну строчку, а сразу несколько. По моим ощущениям, часто подсказывает верные варианты, и это заметно ускоряет написание кода.</p><h3>2) Генерация кода по комментарию</h3><p>Эта функция активируется, когда мы пишем в комментарии то, что хотим сделать, а плагин генерирует код, необходимый для реализации этих действий. Для этой функции нужна определенная тренировка, чтобы научиться грамотно писать запрос. Рассмотрим несколько примеров:</p><h4>Пример 1:</h4><p>Шаг 1: написать метод:</p><pre><code class="java">public static void printEven(List numbers) {  
  
}
</code></pre><p>Шаг 2: Написать комментарий для метода:</p><pre><code class="java">/*  
* Задание 2:  
 * Написать код, который будет выводить только четные числа из списка 
 */
 public static void printEven(List numbers) {  
  
}
</code></pre><p>Шаг 3: Нажать enter внутри метода, гигакод сгенерирует реализацию:</p><pre><code class="java">/*  
* Задание 2:  
 * Написать код, который будет выводить только четные числа из списка 
 * 
 */
 public static void printEven(List numbers) {  
    numbers.stream()  
            .filter(number -&amp;gt; number % 2 == 0)  
            .forEach(System.out::println);  
}
</code></pre><h4>Пример 2:</h4><pre><code class="java">var strList = List.of("CString1", "bString2", "CString3", "CString4", "bString5");  
  
/**  
 * 1. Перевести в  верхний регистр 
 * 2. Получить мапу: ключ - первая буква строки, значение - строки 
 * 
 */
  
// Этот код сгенерирован гигакодом:
 Map&amp;gt; collect = strList.stream()  
        .map(String::toUpperCase)  
        .collect(Collectors.groupingBy(s -&amp;gt; s.charAt(0)));
</code></pre><h3>3) Рефакторинг кода</h3><p>В меню справа после установки плагина появляется специальная иконка, которая открывает окно с чатом:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/b43/c31/e86/b43c31e86b7ad89d714650c5d960d616.png" height="997" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/b43/c31/e86/b43c31e86b7ad89d714650c5d960d616.png" width="1438"/></figure><p>В этом чате можно задавать вопросы и использовать несколько предустановленных команд. Для демонстрации используем следующий кусок кода:</p><pre><code class="java">String[] text = new String[] {"I ", "Love ", "Java"};  
int i = 1;  
while (i &amp;lt; text.length) {  
    System.out.print(i);  
}
</code></pre><p>Выделяем его в редакторе и пишем в чате:<br/> <code>Отрефактори код, чтобы он выводил в консоль все элементы массива text</code></p><p>Нейронка выдает следующий результат:</p><pre><code class="java">public class Main {
    public static void main(String[] args) {
        String[] text = {"I ", "Love ", "Java"}; // Создаем массив строк
        for (String word : text) { // Итерируемся по массиву с помощью цикла for-each
            System.out.print(word); // Выводим элементы массива на экран
        }
    }
}
</code></pre><h3>4. Объяснение кода</h3><p>В чате можно выбрать несколько предустановленных команд, <code>/explain</code> - одна из них. С помощью этой команды можно получить объяснение кода для выделенного текста или всего файла</p><p>Ниже представлен пример объяснения от плагина:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/d99/781/349/d99781349d788bf256a50ebea1302990.png" height="893" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/d99/781/349/d99781349d788bf256a50ebea1302990.png" width="1682"/></figure><p>Кроме этого в чате можно задавать любые вопросы. Недавно мне потребовалось составить регулярку и плагин с этим справился. Никогда регулярки не любил)</p><h2>Структурный поиск</h2><p>Во время изучения статей на хабре о возможностях Inteleji Idea нашел интересную статью про структурный поиск.
