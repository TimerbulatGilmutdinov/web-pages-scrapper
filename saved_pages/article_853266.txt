article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p>Синхронизация корутин в Android имеет огромное значение для обеспечения безопасности и эффективности многопоточности. Коррутины упрощают управление асинхронными задачами, но без должной синхронизации могут возникнуть проблемы, такие как race condition, что приведет к неправильному поведению приложения.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/29a/7ac/d03/29a7acd034af97525623247d76721503.png" height="1024" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/29a/7ac/d03/29a7acd034af97525623247d76721503.png" width="1024"/></figure><p>Представим, что нам нужно изменить переменную из нескольких корутин в таком коде:</p><pre><code class="kotlin">var counter = 0

suspend fun increment() {
    withContext(Dispatchers.Default) {
        repeat(1000) {
                counter++
        }
    }
}

fun main() = runBlocking {
    List(2) {
        launch { increment() }
    }.onEach { it.join() }
    println(counter)
}</code></pre><p>В этом фрагменте кода мы объявляем глобальную переменную counter и запускаем 2 корутины для увеличения counter. Мы ожидаем, что вывод будет 2000. Но на самом деле результат будет совсем другим - от 1000 до 2000. Этот код - простой и очевидный пример race condition. Race condition возникает, когда несколько корутин пытаются получить доступ к общим данным или изменить их одновременно, и это может привести к неверным и непредсказуемым результатам. </p><p>Существует 6 способов решения проблемы состояния гонки (по крайней мере известных мне). </p><p><strong>Mutex</strong> </p><p>Цель <strong>Mutex</strong> - взаимное исключение (аналогично блокировкам в многопоточности), чтобы предотвратить одновременный доступ нескольких coroutine к общим ресурсам. Mutex (<strong>mut</strong>ual <strong>ex</strong>clusion lock ) гарантирует, что только одна корутина может удерживать блокировку в каждый момент времени. Корутины, которые пытаются получить заблокированный Mutex, будут приостановлены до тех пор, пока он не будет освобожден. </p><pre><code class="kotlin">var counter = 0
val mutex = Mutex()


suspend fun increment() {
    withContext(Dispatchers.Default) {
        repeat(1000) {
            mutex.withLock {
                counter++
            }
        }
    }
}

fun main() = runBlocking {
    List(2) {
        launch { increment() }
    }.onEach { it.join() }
    println(counter)
}</code></pre><p>Пример использования: Защита общего изменяемого состояния для предотвращения race condition. </p><p><strong>Атомарные операции</strong> </p><p><strong>Атомарные переменные</strong> позволяют безопасно выполнять изменения без необходимости явных блокировок. Используя такие классы, как AtomicInt или AtomicReference, вы можете безопасно обновлять значения из нескольких coroutine  lock-free, thread-safe  способом. </p><pre><code class="kotlin">var counter = AtomicInt(0)

suspend fun increment() {
    withContext(Dispatchers.Default) {
        repeat(1000) {
            counter.incrementAndGet()
        }
    }
}

fun main() = runBlocking {
    List(2) {
        launch { increment() }
    }.onEach { it.join() }
    println(counter)
}</code></pre><p>Вариант использования: синхронизация без блокировки для легковесных обновлений общего состояния. </p><p> <strong>Channels</strong> </p><p>Channels используются для отправки и получения данных между корутинами потокобезопасным способом, часто для шаблонов производитель-потребитель. Channel позволяет корутинам отправлять данные друг другу асинхронно. Принимающая сопрограмма приостанавливается, пока данные не станут доступны. </p><pre><code class="kotlin">var counter = 0

suspend fun increment(channel: Channel&lt;Int&gt;) {
    withContext(Dispatchers.Default) {
        repeat(1000) {
            channel.send(1) // Send increment request to the channel
        }
    }
}

fun main() = runBlocking {
    val channel = Channel&lt;Int&gt;()

    val receiver = launch {
        for (value in channel) {
            counter += value
        }
    }

    List(2) {
        launch { increment(channel) }
    }.onEach { it.join() }

    channel.close()
    receiver.join()
    println(counter)
}</code></pre><p>Вариант использования: Синхронизация потока данных между корутинами (например, производитель-потребитель).</p><p><br/> <strong>Actors</strong></p><p><strong>Actors </strong>предоставляют основанный на сопрограммах способ реализации модели actor-a для параллелизма, где корутины взаимодействуют путем отправки сообщений. <strong>Actor </strong>- это сопрограмма, которая обрабатывает сообщения по одному за раз, гарантируя последовательное обновление внутреннего состояния, избегая необходимости в блокировках. </p><pre><code class="kotlin">sealed class CounterMsg
object Increment : CounterMsg()
class GetCounter(val response: CompletableDeferred&lt;Int&gt;) : CounterMsg()

fun CoroutineScope.counterActor(): SendChannel&lt;CounterMsg&gt; = actor&lt;CounterMsg&gt; {
    var counter = 0
    val receiver = launch {
        for (msg in channel) {
            when (msg) {
                is Increment -&gt; counter++
                is GetCounter -&gt; msg.response.complete(counter)
            }
        }
    }
    receiver.join()
}

suspend fun increment(channel: SendChannel&lt;CounterMsg&gt;) {
    withContext(Dispatchers.Default) {
        repeat(1000) {
            channel.send(Increment)
        }
    }
}

fun main() = runBlocking {
    val counterActor = counterActor()

    List(2) {
        launch { increment(counterActor) }
    }.onEach { it.join() }

    val response = CompletableDeferred&lt;Int&gt;()
    counterActor.send(GetCounter(response))
    counterActor.close()
    val result = response.await()
    println(result)
}</code></pre><p>Вариант использования: Управление состоянием с помощью передачи сообщений.</p><p><strong> Semaphore</strong></p><p> Semaphores используются для ограничения количества корутин, которые могут одновременно получать доступ к общему ресурсу.  Semaphore поддерживает определенное количество разрешений, и корутины могут получать или освобождать их. Корутины, которые пытаются получить разрешения, когда их нет, будут приостановлены. </p><pre><code class="kotlin">var counter = 0
val semaphore = Semaphore(1)

suspend fun increment() {
    withContext(Dispatchers.Default) {
        repeat(1000) {
            semaphore.withPermit {
                counter++
            }
        }
    }
}

fun main() = runBlocking {
    List(2) {
        launch { increment() }
    }.onEach { it.join() }
    println(counter)
}</code></pre><p>Вариант использования: ограничение количества одновременных доступов к ресурсу (например, подключений к базе данных, сетевых запросов). </p><p><strong>SharedFlow или StateFlow</strong> </p><p>SharedFlow и StateFlow также используются для совместного использования состояния и передачи обновлений между несколькими корутинами.</p><pre><code class="kotlin">val counterFlow = MutableSharedFlow&lt;Int&gt;()

suspend fun increment() {
    withContext(Dispatchers.Default) {
        repeat(1000) {
            counterFlow.emit(1)
        }
    }
}

fun main() = runBlocking {
    var counter = 0

    val collectorJob = launch {
        counterFlow.collect { value -&gt;
            counter += value
        }
    }

    List(2) {
        launch { increment() }
    }.onEach { it.join() }

    collectorJob.cancel()

    println(counter)
}</code></pre><p><strong>Заключение:</strong></p><p>Kotlin предоставляет несколько способов синхронизации корутинв зависимости от варианта использования, включая Mutex для взаимного исключения, Channels для безопасной связи, Semaphore для ограничения параллелизма и многое другое. Эти инструменты гарантируют, что сопрограммы могут работать вместе, не вызывая race condition или несогласованных состояний в параллельных программах.</p><p></p></div></div></div><!-- --><!--
