class="article-formatted-body article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p>Новый перевод от команды <a href="https://t.me/+QmrKn0wA8CdkNjAy">Spring АйО</a> расскажет вам, что такое Model Context Protocol (MCP) и как его внедрение в экосистему Java упрощает жизнь разработчикам систем, в работе которых используется искусственный интеллект и LLM, а также передача большого количества данных между различными компонентами.  </p><hr/><h3>Что такое MCP</h3><p>Model Context Protocol (MCP), открытый протокол, разработанный <a href="https://www.anthropic.com/">Anthropic</a>, меняет представления о том, как приложения на ИИ подключаются друг к другу и делятся контекстом. Он заслужил активную поддержку в среде написания ИИ-приложений, работая как стандартный интерфейс для больших языковых моделей (Large Language Models, LLM), при помощи которого они взаимодействуют с источниками данных, инструментами и ИИ агентами. Занимаетесь ли вы построением автономных систем, которым нужен доступ к базам данных, оркестрацией сложных ИИ сценариев или созданием систем, внутри которых взаимодействуют многочисленные агенты, MCP предложит вам базовый слой, который сделает эти интеграции идеально гладкими.</p><p>MCP выгодно отличается от других решений своим фокусом на сочетаемости (composability) и стыкуемости (interoperability). MCP не просто подключается к источникам данных, он позволяет разработчикам создавать интерактивные системы на ИИ с богатым функционалом, в которых агенты могут делиться контекстом, получать доступ к инструментам и работать вместе через стабильный интерфейс. Другими словами, вы можете быстро подключиться к растущей экосистеме предварительно выстроенных интеграций, в то же время сохраняя гибкость, дающую возможность переключаться между разными LLM провайдерами. Поэтому он является идеальным фундаментом для построения ИИ-приложений со сложной, разнообразной функциональностью.</p><h3>Представляем MCP Java SDK</h3><p>То, что начиналось как экспериментальный проект в прошедшем ноябре, превратилось в многообещающую коллаборацию с командой Spring AI и Anthropic. Мы рады сообщить вам, что этот экспериментальный проект официально перемещен в MCP Java SDK.</p><p>Этот SDK является последней языковой привязкой протокола наряду с SDK для Python, TypeScript и Kotlin, на<a href="https://modelcontextprotocol.io"> </a><a href="http://modelcontextprotocol.io">modelcontextprotocol.io</a>. Java имеет долгую историю как Enterprise язык, и MCP Java SDK упрощает процесс разработки ИИ-приложений в соответствии с новейшими стандартами для организаций.</p><p>MCP Java SDK предлагает многофункциональный фундамент для интеграции ИИ моделей с внешними инструментами и источниками данных. Ключевые возможности SDK включают:</p><h4>Реализации для клиента и сервера</h4><ul><li><p>Поддерживает как синхронную, так и асинхронную передачу данных по MCP.</p></li><li><p>Позволяет выбирать совместимую версию протокола для хорошей interoperability.</p></li></ul><h4>Управление инструментами и ресурсами</h4><ul><li><p>Находить, регистрировать и работать с инструментами динамически.</p></li><li><p>В реальном времени получать списочные нотификации об изменениях для инструментов и ресурсов </p></li><li><p>Управлять ресурсами с использованием URI-шаблонов для структурированного доступа и подписок. </p></li></ul><h4>Работу с промптами и поддержку AI Sampling</h4><ul><li><p>Извлекать и управлять промптами, чтобы кастомизировать поведение ИИ модели. </p></li><li><p>Поддерживает стратегии sampling, чтобы совершенствовать взаимодействие с ИИ.</p></li></ul><h4>Многочисленные реализации передачи данных</h4><ul><li><p>Передача данных на базе STDIO для коммуникаций с прямой обработкой.</p></li><li><p>Передача данных по SSE протоколу на клиенте на базе Java HttpClient для HTTP стриминга.</p></li><li><p>Передача данных на сервере на базе сервлета для стриминга по HTTP в традиционном серверном окружении:</p><ul><li><p>Передача данных по SSE протоколу на базе Spring WebFlux для реактивных приложений.</p></li><li><p>Передача данных по SSE протоколу на базе Spring WebMVC для приложений на сервлетах.</p></li></ul></li></ul><p>Чтобы получить больше информации о том, как начать работу с MCP, обращайтесь к <a href="https://modelcontextprotocol.io/sdk/java/mcp-overview">документации</a>, а также посетите <a href="https://github.com/modelcontextprotocol/java-sdk">репозиторий GitHub</a>, где вы сможете открывать тикеты и участвовать в дискуссиях.</p><h3>Spring AI и MCP</h3><p>Проект Spring AI расширяет MCP Java SDK путем добавления возможностей для улучшения продуктивности разработчиков при интеграции со Spring Boot приложениями. Используя стартеры Spring Boot, разработчики смогут быстро сконфигурировать MCP клиент и сервер, используя инжекцию зависимостей Spring и управление конфигурациями, что облегчает интеграцию ИИ сценариев в приложения.</p><h3>Стартеры для клиента</h3><ul><li><p><code>spring-ai-mcp-client-spring-boot-starter</code> – встроенный стартер для клиента, поддерживающий передачу данных на базе STDIO и по SSE протоколу через HTTP.</p></li><li><p><code>spring-ai-mcp-client-webflux-spring-boot-starter</code> – передача данных на основе SSE протокола, реализованная на основе WebFlux для реактивных приложений.</p></li></ul><h3>Стартеры для сервера</h3><ul><li><p><code>spring-ai-mcp-server-spring-boot-starter</code> – встроенный стартер для сервера, поддерживающий передачу данных на базе STDIO.</p></li><li><p><code>spring-ai-mcp-server-webmvc-spring-boot-starter</code> – передача данных на основе SSE протокола, реализованная на Spring MVC для приложений на сервлетах.</p></li><li><p><code>spring-ai-mcp-server-webflux-spring-boot-starter</code> – передача данных на основе SSE протокола, реализованная на WebFlux для реактивных приложений.</p></li></ul><p>Ниже приведен пример того, как декларативно сконфигурировать клиент-приложение, использующее передачу данных по STDIO. Задайте следующую конфигурацию в <code>application.yml</code>:</p><pre><code class="yaml">spring:
  ai:
    mcp:
      client:
        stdio:
          servers-configuration: classpath:mcp-servers.json</code></pre><p>JSON файл, на который ссылается этот код, задает сервер, к которому надо будет подключиться в формате Claude Desktop. </p><pre><code class="json">{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/username/Desktop",
        "/Users/username/Downloads"
      ]
    }
  }
}</code></pre><p>Когда клиентское приложение стартует, оно запускает MCP сервер, устанавливает каналы коммуникации по STDIO и управляет жизненным циклом сервера. </p><p>Spring AI M6 также вводит аннотацию <code>@Tool</code>, которая упрощает создание MCP. Чтобы получить больше информации об этой аннотации, обратитесь к <a href="https://docs.spring.io/spring-ai/reference/api/mcp/mcp-overview.html">справочной документации от Spring AI по MCP</a>.</p><hr/><figure class="full-width"><img data-src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/c16/4b2/4fe/c164b24fe67857325369e7d0e25ab10e.png" height="520" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/c16/4b2/4fe/c164b24fe67857325369e7d0e25ab10e.png" width="1600"/></figure><p><a href="https://www.springnow.ru">Регистрируйтесь</a> на главную конференцию про Spring на русском языке от сообщества Spring АйО!
