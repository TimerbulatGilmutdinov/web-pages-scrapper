Всем привет! С вами Анна Жаркова, руководитель группы разработки в компании Usetech. Сегодня я вам принесла необычный материал. 27 июня 2024 года для сторонних разработчиков открыли и презентовали российскую среду разработки Giga IDE со встроенным ИИ-ассистентом от «Сбера» и «СберТеха». Заявлено, что встроенный в среду ИИ-ассистент GigaCode анализирует контекст, предлагает полные конструкции функций, циклов и других элементов, что позволяет писать код в среднем на 25% быстрее. Ассистент поддерживает как автокомплит кода, так и интеллектуальные подсказки для ввода, генерацию кода для поддерживаемых язык. Также его можно использовать для выполнения разных команд.<br/>
<br/>
Мы посмотрим, как подключить, настроить и применить GigaCode AI к мобильной разработке.<br/>
Для начала нам потребуется скачать специальную <a href="https://gigacode.ru/#/#chat">IDE GigaCode </a>. Нам предложит перейти на ресурс GitVerse (аналоги GitHub, полностью совместим с Git), где потребуется создать аккаунт и авторизоваться через СберID:<br/>
<br/>
<img data-src="https://habrastorage.org/webt/9w/o0/dl/9wo0dlwhuiakk2siizixbv4eau0.png" src="https://habrastorage.org/r/w1560/webt/9w/o0/dl/9wo0dlwhuiakk2siizixbv4eau0.png" width="720"/><br/>
<a name="habracut"></a><br/>
После регистрации мы окажемся на дашборде, где можем смотреть инфо по своим репозиториям, своей активности:<br/>
<img data-src="https://habrastorage.org/webt/md/ze/ns/mdzensvaviz7cbvl9arpas_tnps.png" src="https://habrastorage.org/r/w1560/webt/md/ze/ns/mdzensvaviz7cbvl9arpas_tnps.png" width="720"><br/>
<br/>
Также здесь мы можем скачать установочный файл IDE под свою версию ОС:<br/>
<img data-src="https://habrastorage.org/webt/cw/f8/mb/cwf8mbuezf1vb5izvmlp9ouz5e8.png" src="https://habrastorage.org/r/w1560/webt/cw/f8/mb/cwf8mbuezf1vb5izvmlp9ouz5e8.png" width="720"><br/>
<br/>
Если у вас Mac, как у меня, вы можете столкнуться с проблемой:<br/>
<img data-src="https://habrastorage.org/webt/-1/p9/zv/-1p9zvsavv8bocx-pfhv7d04ir0.png" src="https://habrastorage.org/r/w1560/webt/-1/p9/zv/-1p9zvsavv8bocx-pfhv7d04ir0.png" width="360"/><br/>

Не пугайтесь. На самом деле, это просто Mac OS блокирует установку из неизвестного источника. Для разблокировки приложения введем в терминале команду:<br/>
<pre><code class="swift">sudo xattr -rd com.apple.quarantine /Applications/GIGA\ IDE\ CE\ 2024.1.1.app</code></pre><br/>
<br/>
После этого мы можем спокойно запустить IDE. Т.к GigaCode IDE базируется на PyChar и JB Idea, то окно приложения очень похоже на оригинал:<br/>
<img data-src="https://habrastorage.org/webt/4p/62/pf/4p62pf6-cu44rwjcxjryotozhoe.png" src="https://habrastorage.org/r/w1560/webt/4p/62/pf/4p62pf6-cu44rwjcxjryotozhoe.png" width="720"/><br/>

<img data-src="https://habrastorage.org/webt/yt/om/pa/ytompa-3gls9qsk9q_rsez7y-f0.png" src="https://habrastorage.org/r/w1560/webt/yt/om/pa/ytompa-3gls9qsk9q_rsez7y-f0.png" width="720"/><br/>

А вот чтобы работать с Android проектами, нам потребуется активировать GigaChat AI-ассистента и подключить его к Android. Для этого переходим <a href="https://gigacode.ru/#/#IDEs">по ссылке</a> и выбираем нужную IDE.<br/>
<img data-src="https://habrastorage.org/webt/7v/t8/ga/7vt8gaz7fm31zpktd3uhl7zqbha.png" src="https://habrastorage.org/r/w1560/webt/7v/t8/ga/7vt8gaz7fm31zpktd3uhl7zqbha.png" width="720"/><br/>
Попадаем на страницу активации расширения на GitVerse. В процессе может снова потребоваться снова авторизоваться через СберID. <br/>
Далее мы попадаем на страницу подробных инструкций для подключения.<br/>
Для начала нужно зайти в настройки Android Studio и добавить в разделе управления репозиториями плагинов ссылку на хранилище плагинов GigaCode:<br/>
<pre><code class="swift">
https://gigacode.ru/jarvis/updatePlugins.xml
</code></pre><br/>
Во вкладке Marketplace выберите GigaCode и нажмите «Install»:<br/>
<img data-src="https://habrastorage.org/webt/-f/ps/of/-fpsofk5tt7s5g5w5_bhsbicu7o.png" src="https://habrastorage.org/r/w1560/webt/-f/ps/of/-fpsofk5tt7s5g5w5_bhsbicu7o.png" width="720"/><br/>
Перезапускаем IDE и открываем, либо создаем проект. Наш плагин активирован.<br/>
Внизу окна появится кнопка GigaCode. Нажав попадаем снова на GitVerse, где наш экземпляр IDE добавляется в список активированных устройств:<br/>
<img data-src="https://habrastorage.org/webt/fx/9m/4i/fx9m4iuqnltlph81zbggk_ejgfw.png" src="https://habrastorage.org/r/w1560/webt/fx/9m/4i/fx9m4iuqnltlph81zbggk_ejgfw.png" width="720"/><br/>
Снова перезапускаем IDE. Теперь мы, наконец, можем использовать AI-ассистента Giga в работе.<br/>
Давайте посмотрим, что он умеет делать. Тапаем на иконку чата и открываем его:<br/>
<img data-src="https://habrastorage.org/webt/qe/29/wu/qe29wuxdcyxdf_wd7izwgkxxzm4.png" src="https://habrastorage.org/r/w1560/webt/qe/29/wu/qe29wuxdcyxdf_wd7izwgkxxzm4.png" width="720"/><br/>
Визуально это похоже на Gemini AI, который встроен в Koala Preview версию Android Studio. Только работает без впн, и активировать в Gemini нужно не под экземпляр IDE, а под каждый проект с помощью ключа.<br/>
<br/>
Что же нам предлагается: <br/>
 — создать тест для выделенного блока кода<br/>
 — объяснить блок кода<br/>
 — сгенерировать документацию.<br/>
Для этого есть готовые команды, которые мы можем ввести в поле ввода.<br/>
<br/>
Что ж. Начнем с генерации теста.<br/>
Итак, у нас есть код:<br/>
<pre><code class="kotlin">class MainTestConfig : IConfigurator {

    override fun create(view: IView): IInteractor? {
        val presenter = MainTestPresenter(view = view)
        val router = MainTestRouter(view = view)
        val interactor: IMainTestInteractor = MainTestInteractor(
            presenter = presenter,
            router = router
        )

        view.interactor = interactor

        return interactor
    }
}
</code></pre><br/>
Просто выделим его и вобьем в поле чата /test. Нам сгенерирует сэмпл теста:<br/>
<img data-src="https://habrastorage.org/webt/uu/mh/a3/uumha30uetivqesd5nynd9hdscq.png" src="https://habrastorage.org/r/w1560/webt/uu/mh/a3/uumha30uetivqesd5nynd9hdscq.png" width="720"/><br/>
<br/>
При повторном вызове для того же блока кода сэмпл усложняется и даже становится похожим на настоящий с импортами:<br/>
<pre><code class="kotlin">import org.junit.Before
import org.junit.Test
import org.junit.runner.RunWith
import org.mockito.Mock
import org.mockito.Mockito.verify
import org.mockito.junit.MockitoJUnitRunner

@RunWith(MockitoJUnitRunner::class)
class MainTestConfigTest {

    @Mock
    lateinit var view: IView

    lateinit var config: MainTestConfig

    @Before
    fun setup() {
        config = MainTestConfig()
    }

    @Test
    fun testCreate() {
        val interactor = config.create(view)

        assert(interactor is MainTestInteractor)
    }

    @Test
    fun testCreate_Presenter() {
        val interactor = config.create(view)

        verify(view).interactor = interactor

        assert(view.interactor is MainTestInteractor)
    }

    @Test
    fun testCreate_Presenter_Router() {
        val interactor = config.create(view)

        assert(view.interactor is MainTestInteractor)
        assert((view.interactor as MainTestInteractor).presenter is MainTestPresenter)
        assert((view.interactor as MainTestInteractor).router is MainTestRouter)
    }
}

</code></pre><br/>
Бонусом получаем объяснение:<br/>
<img data-src="https://habrastorage.org/webt/sn/qo/se/snqosenrc0ifrsc6zk_rucxwdfu.png" src="https://habrastorage.org/r/w1560/webt/sn/qo/se/snqosenrc0ifrsc6zk_rucxwdfu.png" width="720"/><br/>
