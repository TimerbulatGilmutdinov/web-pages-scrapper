xmlns="http://www.w3.org/1999/xhtml"><p>Привет, Хабр! </p><p>На связи Глеб Гутник, мобильный разработчик из компании xStack. В этой статье мы рассмотрим, как можно эффективно кастомизировать взаимодействие с клавиатурой в Jetpack Compose и Compose Multiplatform для создания комфортного UX.</p><p>Для того, чтобы пользователь мог полноценно взаимодействовать с интерфейсом, мы, мобильные разработчики, ежедневно решаем простую, казалось бы, задачу: поля ввода формы на экране всегда должны быть видны и находиться выше клавиатуры.</p><p>На Android для этого предусмотрен флаг <code>android:windowSoftInputMode="adjustResize"</code>, но он сжимает окно приложения без учета анимации клавиатуры, поэтому пользователь видит пустое поле долю секунды, пока клавиатура открывается:</p><div class="floating-image"><figure class="float full-width"><img alt="Интерфейс до открытия клавиатуры" data-blurred="true" data-src="https://habrastorage.org/getpro/habr/upload_files/085/e02/153/085e02153a541ebd991a4dc800cb9db2.jpg" height="1280" src="https://habrastorage.org/r/w780q1/getpro/habr/upload_files/085/e02/153/085e02153a541ebd991a4dc800cb9db2.jpg" title="Интерфейс до открытия клавиатуры" width="590"/><div><figcaption>Интерфейс до открытия клавиатуры</figcaption></div></figure><p></p></div><div class="floating-image"><figure class="float"><img alt="Интерфейс в момент, когда фокус на поле уже наведен, но клавиатура только начала открываться" data-blurred="true" data-src="https://habrastorage.org/getpro/habr/upload_files/87e/659/c18/87e659c18cd9d6b1e731d348e96c9dac.jpg" height="1123" src="https://habrastorage.org/r/w780q1/getpro/habr/upload_files/87e/659/c18/87e659c18cd9d6b1e731d348e96c9dac.jpg" title="Интерфейс в момент, когда фокус на поле уже наведен, но клавиатура только начала открываться" width="512"/><div><figcaption>Интерфейс в момент, когда фокус на поле уже наведен, но клавиатура только начала открываться</figcaption></div></figure><p></p></div><p>Чтобы исправить это поведение, разработчики Jetpack Compose предложили новый способ взаимодействия с клавиатурой: edge-to-edge режим в сочетании с <code>Modifier.imePadding()</code>. </p><details class="spoiler"><summary>Скрытый текст</summary><div class="spoiler__content"><p>Важно: в Android <code>Modifier.imePadding()</code> работает только при обязательном выполнении двух условий: флаг <code>adjustResize</code> в AndroidManifest и <code>enableEdgeToEdge()</code> в коллбэке <code>onCreate()</code> в нашем Activity. </p></div></details><p>Итак, применим этот модификатор к основной <code>Column</code> нашей формы:</p><pre><code class="kotlin">Column(
    Modifier
        .padding(horizontal = 32.dp)
        .imePadding()
        .verticalScroll(rememberScrollState())
) {
    Spacer(Modifier.height(it.calculateTopPadding()))
    32.dp.VerticalSpacer()
    AccountIcon()
    32.dp.VerticalSpacer()
    Text(
        text = stringResource(Res.string.create_your_account),
        style = MaterialTheme.typography.titleLarge,
        fontSize = 32.sp,
        modifier = Modifier.align(Alignment.CenterHorizontally)
    )
    // ...
}</code></pre><p>Теперь поведение уже намного лучше: интерфейс плавно адаптируется под открывающуюся клавиатуру, а если она перекрывает поле ввода, то выделенное поле плавно «едет» вверх ровно настолько, на сколько нужно:</p><div class="tm-iframe_temp" data-src="https://embedd.srv.habr.com/iframe/66effd81d1b5b66d8d27d921" data-style="" id="66effd81d1b5b66d8d27d921" width=""></div><p>Казалось бы, это то, чего мы добивались. Однако если мы захотим «пошарить» наш интерфейс на iOS и перенести приложение на Compose Multiplatform, возникнет новый нюанс: в новых версиях iOS клавиатура полупрозрачная с типичным для Apple размытием по Гауссу. Но наш <code>Modifier.imePadding()</code> — это именно отступ (разработчики библиотеки нам не врут в нейминге метода), поэтому интерфейс формы не будет «просвечивать» под полупрозрачной клавиатурой:</p><div class="tm-iframe_temp" data-src="https://embedd.srv.habr.com/iframe/66effd8187bb36d4b3edf931" data-style="" id="66effd8187bb36d4b3edf931" width=""></div><p>К сожалению, «коробочных» методов решения этой проблемы в Compose не предусмотрено. Чтобы наш интерфейс выглядел более нативно, напишем собственную обертку, которая будет привязывать состояние скролла к высоте клавиатуры. Я назвал ее <code>ImeAdaptiveColumn</code>. С ней пришлось довольно изрядно поэкспериментировать, но результат оказался вполне удовлетворительным.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/61d/325/38d/61d32538ddedd06651d708e12f90de08.png" height="460" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/61d/325/38d/61d32538ddedd06651d708e12f90de08.png" width="668"/></figure><pre><code class="kotlin">class FocusedAreaEvent {
    var id: String by mutableStateOf("")
    var rect: Rect? by mutableStateOf(null)
    var spaceFromBottom: Float? by mutableStateOf(null)
}

class FocusedArea {
    var rect: Rect? = null
}

data class History&lt;T&gt;(val previous: T?, val current: T)

// emits null, History(null,1), History(1,2)...
fun &lt;T&gt; Flow&lt;T&gt;.runningHistory(): Flow&lt;History&lt;T&gt;&gt; =
    runningFold(
        initial = null as (History&lt;T&gt;?),
        operation = { accumulator, new -&gt; History(accumulator?.current, new) }
    ).filterNotNull()

data class ClickData(
    val unconsumed: Boolean = true,
    val offset: Offset = Offset.Zero
)

@OptIn(ExperimentalFoundationApi::class)
@Composable
fun ImeAdaptiveColumn(
    scrollState: ScrollState = rememberScrollState(),
    scrollable: Boolean = true,
    modifier: Modifier = Modifier,
    horizontalPadding: Dp = 16.dp,
    content: @Composable ColumnScope.() -&gt; Unit
) {
    val screenHeight = LocalScreenSize.height
    val imeHeight by rememberUpdatedState(imeHeight())

    var clickData by remember { mutableStateOf(ClickData()) }
    val focusedAreaEvent = remember { FocusedAreaEvent() }
    val focusedArea = remember { FocusedArea() }
    LaunchedEffect(
        key1 = focusedAreaEvent.id
    ) {
        if (focusedAreaEvent.id.isNotEmpty()) {
            focusedAreaEvent.spaceFromBottom?.let { capturedBottom -&gt;
                snapshotFlow { imeHeight }
                    .runningHistory()
                    .collectLatest { (prev, height) -&gt;
                        val prevHeight = prev ?: 0
                        if (height &gt; capturedBottom) {
                            if (prevHeight &lt; capturedBottom) {
                                val difference = height - capturedBottom
                                scrollState.scrollBy(difference)
                            } else {
                                val difference = height - prevHeight
                                scrollState.scrollBy(difference.toFloat())
                            }
                        } else {
                            if (prevHeight &gt; capturedBottom) {
                                val difference = prevHeight - capturedBottom
                                scrollState.scrollBy(-difference)
                            }
                        }
                    }
            }
        }
    }

    Column(
        modifier = modifier
            .onFocusedBoundsChanged { coordinates -&gt;
                coordinates?.boundsInWindow()?.let {
                    focusedArea.rect = it
                    if (clickData.unconsumed &amp;&amp; clickData.offset in it) {
                        focusedAreaEvent.run {
                            id = uuid()
                            rect = it
                            spaceFromBottom = screenHeight - it.bottom
                        }
                        clickData = clickData.copy(unconsumed = false)
                    }
                }
            }
            .pointerInput(Unit) {
                awaitEachGesture {
                    val event = awaitPointerEvent(PointerEventPass.Main)
                    // If the software keyboard is hidden, register a new focused area.
                    if (event.type == PointerEventType.Press &amp;&amp; imeHeight == 0) {
                        val offset = event.changes.firstOrNull()?.position ?: Offset.Zero
                        clickData = ClickData(
                            unconsumed = true,
                            offset = offset
                        )
                    }
                }
            }
            .background(MaterialTheme.colorScheme.surface)
            .padding(horizontal = horizontalPadding)
            .verticalScroll(scrollState, enabled = scrollable),
        content = content
    )
}

@Composable
fun imeHeight() = WindowInsets.ime.getBottom(LocalDensity.current)
</code></pre><p>Чтоздесь происходит? Я ставил себе цель написать такое Composable API, которое подстраивалось бы под клавиатуру независимо от содержимого. Иначе говоря, это такой «content‑agnostic» компонент, когда пользователю кода не нужно ничего дополнительного вызывать на своей стороне, все происходит под капотом. </p><p>Например, можно теперь вызвать <code>ImeAdaptiveColumn</code> таким образом:</p><pre><code class="kotlin">ImeAdaptiveColumn(horizontalPadding = 32.dp) {
    Spacer(Modifier.height(it.calculateTopPadding()))
    32.dp.VerticalSpacer()
    AccountIcon()
    32.dp.VerticalSpacer()
    Text(
        text = stringResource(Res.string.create_your_account),
        style = MaterialTheme.typography.titleLarge,
        fontSize = 32.sp,
        modifier = Modifier.align(Alignment.CenterHorizontally)
    )
    32.dp.VerticalSpacer()
    SignupFormTextField(
        label = stringResource(Res.string.first_name_title),
        placeholder = stringResource(Res.string.first_name_placeholder)
    )
    // ...
}</code></pre><p>Такого удобства удается достичь за счет двух главных API, доступных в Compose: <code>Modifier.pointerInput()</code>  и <code>Modifier.onFocusBoundsChanged()</code>. </p><p>Разложим поэтапно, что происходит, когда пользователь тапает по полю ввода:</p><ol><li><p>Событие клика распространяется по дереву UI‑компонентов. Наш метод модификатора (pointerInput) отслеживает это событие, если клавиатура в данный момент закрыта, и складывает координату клика в переменную (clickData.offset).</p></li><li><p>Область, на которую наведен фокус, меняется, вызывается коллбэк onFocusBoundsChanged, который складывает событие клика в focusedAreaEvent.</p></li><li><p>На каждое событие focusedAreaEvent запускается LaunchedEffect, который отслеживает изменения в высоте клавиатуры и скроллит нашу Column, если эта высота перекрывает полученные в focusedAreaEvent координаты.</p></li></ol><p>Таким образом, мы получаем такой результат на iOS (который лучше всего виден, если включить темную тему). Первое видео — интерфейс с обычным <code>imePadding</code>, второе — с <code>ImeAdaptiveColumn</code>.</p><div class="tm-iframe_temp" data-src="https://embedd.srv.habr.com/iframe/66effd82ee68229db68a45f9" data-style="" id="66effd82ee68229db68a45f9" width=""></div><div class="tm-iframe_temp" data-src="https://embedd.srv.habr.com/iframe/66effd8228dfc09d9e8cf1df" data-style="" id="66effd8228dfc09d9e8cf1df" width=""></div><p>Этот не слишком замысловатый прием лишний раз показывает, насколько технологии, рассчитанные под одну платформу (Android), по‑другому ведут себя, если перенести их на другую (iOS). И все же мне кажется, что одна такая небольшая уловка стоит того, чтобы приблизить user experience в Compose Multiplatform к нативным технологиям iOS.</p><details class="spoiler"><summary>Скрытый текст</summary><div class="spoiler__content"><p>Полный код, написанный для этой статьи, доступен <a href="https://github.com/gleb-skobinsky/AdaptiveComponents" rel="noopener noreferrer nofollow">здесь</a>.</p><p>Отмечу вместо постскриптума, что с iOS частью необходимо использовать <code>ignoreSafeArea(.all)</code> и <code>OnFocusBehavior.DoNothing</code>, чтобы добиться желаемого эффекта.</p></div></details><p></p></div></div></div><!-- --><!-- --></div><!-- --><!-- --></div><!--]--><!-- --><div class="tm-article-presenter__meta" data-test-id="article-meta-links"><div
