xmlns="http://www.w3.org/1999/xhtml"><p>В предыдущей <a href="https://habr.com/ru/companies/spring_aio/articles/854062/">статье</a> "Как жить без IntelliJ IDEA" мы разбирали, какие есть альтернативы Ultimate в части поддержки Spring.</p><p>В этой части команда <a href="https://t.me/+QmrKn0wA8CdkNjAy">Spring АйО</a> исследует, чем можно заменить встроенный HTTP-клиент JetBrains, за что его так любят разработчики и какие инструменты могут стать достойной альтернативой.</p><hr/><p>Ранее мы выяснили, что жить без IntelliJ IDEA Ultimate можно, пока Community версия доступна для скачивания без ограничений. И даже если её скачивание станет невозможным, всегда можно будет собрать Community вручную из открытого исходного кода.</p><p>Если спросить у Spring-разработчика за что он любит IntelliJ IDEA Ultimate, то ответ вероятнее всего будет следующим: "Spring поддерживается, SQL-запросы удобно выполнять вот в этой панельке Database справа, а ещё по HTTP можно эндпоинты дёргать прямо из IDE". Про поддержку Spring мы рассказали во второй статьей из цикла, про выполнение SQL-запросов поговорим в следующей, а сегодня разберемся с HTTP-клиентом.</p><h2>В чём сила HTTP-клиента от JetBrains?</h2><p>Как всегда, прежде чем рассматривать альтернативы, давайте разберемся, за что разработчикам так полюбился HTTP-клиент от JetBrains.</p><p>Если и есть причина, за которую разработчики могут полюбить ту или иную фичу – это понятный и удобный с UX (User eXperience). Другими словами, такие фичи как:</p><ul><li><p>Генерация запроса прямо из кода. Например, из метода Spring контроллера или из OpenAPI схемы.</p></li><li><p>Автодополнения путей в момент набора запроса вручную. Кстати появилось только в версии 2023.1. Так что, если вы сидите на более старой IntelliJ IDEA, эту фичу вы еще не видели.</p></li><li><p>Возможность запуска запроса прямо из IDE через хорошо знакомую зеленую стрелочку.</p></li></ul><p>формируют интуитивное понимание простоты и удобства использования инструмента в повседневном flow разработчика.</p><p>Также стоит отметить, что выбранный JetBrains понятный текстовый формат описания запросов, сильно похожий на стандарт <a href="https://datatracker.ietf.org/doc/html/rfc7230"><u>RFC 7230</u></a>, также значительно снижает порог входа в использование инструмента. Вам не нужно знать что-то ещё, кроме структуры HTTP-запроса, чтобы выполнить его прямо в IDE.</p><p>Не можем не упомянуть поддержку множества протоколов, не только REST, наличие профилей (environment), а также возможность писать небольшие тесты и запускать их на CI.</p><p>Отдельно хочется отметить возможность записывать в <em>variables</em> произвольные значения, полученные в том числе через разбор результата выполнения запроса, что позволяет, например, тестировать сценарии, затрагивающие несколько последовательных запросов.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/9d6/51d/2be/9d651d2bed53f0607551d144a3a52229.png" height="582" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/9d6/51d/2be/9d651d2bed53f0607551d144a3a52229.png" width="1060"/></figure><p><em>А как вы использовали HTTP Client? Пишите в комментариях!</em></p><h2>Плагины для IntelliJ IDEA</h2><p>Первым делом давайте посмотрим на бесплатные плагины для IntelliJ IDEA. Может, мы сможем найти что-то подходящее и закончить на этом статью? В маркетплейсе есть некоторое количество плагинов, в названии которых есть слово Restful.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/b7d/325/19e/b7d32519ed68a142467352ef9d102a9c.png" height="730" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/b7d/325/19e/b7d32519ed68a142467352ef9d102a9c.png" width="1600"/></figure><h3>RestfulToolkitX</h3><p>RestfulToolkitX даже встроен в GigaIDE. Должно быть, неплохой плагин. Плагин добавляет новую боковую панельку в IDE, в которой перечислены все доступные эндпоинты нашего Spring приложения.</p><p>Каждый из эндпоинтов можно вызвать, и RestfulToolkitX даже генерирует нам шаблон RequestBody, разобрав, какие параметры передаются в метод контроллера. Однако общий UX плагина достаточно плохой. Никакой подсветки синтаксиса для JSON; плагин не запоминает <code>RequestBody</code>, <code>RequestParameters</code> и <code>Headers</code>, которые я указал для данного эндпоинта; в случае ошибки я просто вижу эксепшн; ну и никакой специализированной поддержки для последовательного вызова нескольких запросов. Вызвать произвольный запрос можно, но не самым очевидным образом - нужно просто полностью переписать параметры любого другого, предзаполненного запроса. На самом деле, больше про плагин сказать и нечего. Выполнить запрос он позволяет, но ничего больше. Есть инструменты и поудобней, о чем мы поговорим ниже.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/421/3c5/8b9/4213c58b9c2cbb222f01964a8842b8f4.png" height="1600" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/421/3c5/8b9/4213c58b9c2cbb222f01964a8842b8f4.png" width="827"/></figure><h3>HURL</h3><p>Вообще <a href="https://hurl.dev/"><u>HURL</u></a> это CLI инструмент, который работает с файлами расширения .hurl, который по синтаксису очень похож на .http. Помимо самих запросов, позволяет писать несложные тесты, а также, извлекать данные из ответов, для дальнейшего их использования в последующих запросах. Нас он особенно заинтересовал тем, что для него есть плагин, который также похож на HTTP Client.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/e6a/f7f/937/e6af7f93788947d93ecd3d87d59e1c79.png" height="1367" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/e6a/f7f/937/e6af7f93788947d93ecd3d87d59e1c79.png" width="1600"/></figure><p>Выглядит это все очень интересно, но, к сожалению, плагин очень сырой и его UX очень сильно не дотягивает до JetBrains HTTP Client. Идея с сохранением значений, извлеченных из запросов классная, но реализация - хромает. Эти самые значения, по всей видимости, хранятся в оперативной памяти hurl, и поэтому, их можно использовать только при запуске всего файла целиком, а сделать это средствами плагина не так-то просто: нужно открыть редактор Run Configuration и изменить параметры запуска.</p><p>Отсутствует генерация запросов от эндпоинта, отсутствует навигация, нельзя запускать запросы на разных Environment. Ну, как нельзя, на самом деле сам клиент это делать позволяет - можно передать файл с параметрами с помощью аргумента командной строки --variables-file, но плагин вам в этом не поможет. <a href="https://github.com/jazzytomato/intellij-hurl-plugin"><u>Код плагина</u></a> открыт, так что доработкой может заняться любой желающий!</p><h2>Пишите тесты</h2><p>Прежде чем рассматривать полноценные альтернативы, стоит отметить, что для проверки ответа HTTP-эндпоинта нередко более правильным подходом оказывается написание теста, а не использование отдельного HTTP-клиента. Этот подход имеет несколько преимуществ:</p><ul><li><p>Тестовые фреймворки или REST-клиенты, как правило, уже включены в зависимости проекта, а разработчики умеют ими пользоваться. Таким образом мы избавляемся от необходимости изучать дополнительные инструменты</p></li><li><p>Вместо разового вызова эндпоинта вы создаёте тест, который можно использовать повторно или доработать в будущем, что может положительно сказаться на коде проекта в целом</p></li><li><p>Всё выполняется непосредственно в среде разработки, нам не нужно выходить за её пределы (прямо как и с HTTP-клиентом!)</p></li><li><p>С каждым новым релизом Spring'а API для тестирования и выполнения запросов становится всё лаконичнее и лаконичнее. Так может проще написать пару строчек кода в .java файле, чем в .http?</p></li></ul><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/69c/07a/01b/69c07a01b09ee40b1ed568416ec668db.png" height="1464" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/69c/07a/01b/69c07a01b09ee40b1ed568416ec668db.png" width="2488"/></figure><p>Однако есть и ограничения. HTTP-клиенты также используются для взаимодействия с внешними сервисами, для проверки и изучения их поведение. Для таких задач написание тестов обычно не практикуется. Кроме того, в проекте могут быть строгие правила относительно тестирования, и вызов каждого эндпоинта может не соответствовать этим правилам.</p><p>В общем, если есть возможность написать тест, то лучше его написать. Мысль тривиальная, но мы не могли её не озвучить.</p><h2>VS Code</h2><p>Одним из самых популярных расширений в маркетплейсе VS Code является REST Client. Одним из ключевых его преимуществ в контексте поиска аналога HTTP-клиенту от JetBrains является поддержка <code>.http</code> файлов, которые как раз и использует IntelliJ IDEA. Если у вас уже есть довольно большое количество <code>.http</code> файлов, первое время можно попробовать пожить между двумя IDE. Что, конечно, не очень удобно, но зато формат запросов будет привычным.</p><p>Решение далеко от идеала, но подойдет тем разработчикам, которые решили перейти на VS Code. В прошлой <a href="https://habr.com/ru/companies/spring_aio/articles/854062/">статье</a> мы как раз обозревали возможность использования этой IDE для Spring разработки, хотя она очень сильно не дотягивает до IntelliJ IDE.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/06a/fd0/ebb/06afd0ebb1205a4c84a7723a879c0898.png" height="1325" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/06a/fd0/ebb/06afd0ebb1205a4c84a7723a879c0898.png" width="1600"/></figure><h2>CLI инструменты</h2><p>Среди различных CLI-инструментов, упрощающих работу с HTTP-запросами, думаю практически каждый из вас знаком с curl. Это мощный инструмент, который позволяет отправлять HTTP-запросы и получать ответы, будь то простой GET-запрос или сложные сценарии с авторизацией, отправкой файлов и настройками заголовков.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/25e/917/dee/25e917dee65bebba6cf423536d38989e.png" height="1194" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/25e/917/dee/25e917dee65bebba6cf423536d38989e.png" width="1600"/></figure><p>Скептики могут спросить: "Куда еще лучше?". Но давайте всё же рассмотрим некоторые из наиболее интересных решений.</p><p>HTTPie – это, пожалуй, самый популярный CLI-инструмент для работы с HTTP, судя по количеству звезд на GitHub. Он предоставляет интуитивно понятный синтаксис, позволяя писать запросы почти как обычные предложения, что делает его особенно удобным для чтения. HTTPie также поддерживает цветовое выделение, обработку аутентификации и обладает гибкими возможностями по обработке JSON и других форматов.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/0c3/7a9/e3a/0c37a9e3a942501876f848dbe9b8c15c.png" height="1211" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/0c3/7a9/e3a/0c37a9e3a942501876f848dbe9b8c15c.png" width="1600"/></figure><p><em>Наверняка мы забыли про ещё какой-нибудь CLI-tool, которым пользуетесь именно вы. Если так – напишите нам про него в комментариях ;)</em></p><p>Все эти тулы, конечно, прекрасны, но в контексте альтернативы HTTP-клиенту встроенному в IDE имеют очевидный недостаток – они не встроены в IDE :D. Нет, мы конечно можем открыть терминал прямо в IDE и выполнить запрос оттуда, но вот сгенерировать запрос отталкиваясь от эндпоинта у нас не получится.</p><h3>Это что за покемон TUI?</h3><p>Знаете ли вы, что такое <a href="https://terminaltrove.com/categories/tui/"><u>TUI Terminal Tools</u></a>? Это новый этап в эволюции CLI-инструментов. Всё взаимодействие с инструментом происходит в терминале, но теперь с более приятным интерфейсом. Один из таких инструментов — <a href="https://github.com/darrenburns/posting"><u>Posting</u></a>, HTTP-клиент, который похож на Postman или Insomnia, но работает прямо в терминале. </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/2a5/28f/e22/2a528fe2267562d077fd15a52b031ce4.png" height="1866" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/2a5/28f/e22/2a528fe2267562d077fd15a52b031ce4.png" width="3104"/></figure><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/2e7/713/bf3/2e7713bf3c1e7b58b46e22ac952b95d0.png" height="1094" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/2e7/713/bf3/2e7713bf3c1e7b58b46e22ac952b95d0.png" width="1600"/></figure><p>Благодаря интерфейсу TUI, его удобно использовать даже через SSH, а рабочие процессы оптимизированы под взаимодействие исключительно через клавиатуру. Запросы сохраняются локально в простых YAML-файлах, что делает их легко читаемыми и удобными для контроля версий. К тому же Posting поддерживает импорт спецификаций OpenAPI.</p><h2>Postman и КО</h2><p>В мире разработки и тестирования API существует множество удобных UI-инструментов для выполнения HTTP-запросов. Наиболее популярным и узнаваемым среди них является Postman, но также существует множество достойных альтернатив:</p><ul><li><p><a href="https://insomnia.rest"><u>Insomnia</u></a></p></li><li><p><a href="https://hoppscotch.io"><u>Hoppscotch</u></a></p></li><li><p><a href="https://www.usebruno.com"><u>Bruno</u></a></p></li><li><p><a href="https://github.com/scalar/scalar"><u>Scalar</u></a></p></li><li><p><a href="https://apidog.com"><u>Apidog</u></a></p></li><li><p><a href="https://httpie.io/download"><u>HTTPie Desktop</u></a> — знакомый многим HTTPie, только в удобной оболочке с графическим интерфейсом</p></li></ul><figure class="full-width"><img alt="Insomnia" data-src="https://habrastorage.org/getpro/habr/upload_files/51a/fe1/66e/51afe166ec78e41864dc84bddaa140d7.png" height="961" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/51a/fe1/66e/51afe166ec78e41864dc84bddaa140d7.png" title="Insomnia" width="1600"/><div><figcaption>Insomnia</figcaption></div></figure><figure class="full-width"><img alt="Bruno" data-src="https://habrastorage.org/getpro/habr/upload_files/8d9/974/ea2/8d9974ea2525e0e09ff597849a8ad416.png" height="962" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/8d9/974/ea2/8d9974ea2525e0e09ff597849a8ad416.png" title="Bruno" width="1600"/><div><figcaption>Bruno</figcaption></div></figure><p>Большинство этих инструментов в своей рекламе указывают, что являются “open-source альтернативой Postman”. Другими словами, Postman стал своего рода нарицательным именем для подобных инструментов, однако конкуренты пытаются выделиться за счёт своей открытости и особенностей.</p><p>Функциональность большинства из них в целом схожа, но есть моменты, на которые стоит обратить внимание при выборе подходящего решения:</p><ol><li><p><strong>Лицензия на использование</strong>. Например, Postman частично бесплатен, но некоторые функции доступны только по подписке. Если вы планируете использовать инструмент в долгосрочной перспективе, важно учитывать возможные ограничения и затраты.</p></li><li><p><strong>Open-source и доступность исходного кода</strong>. Наличие исходников и лицензии на них — важный аспект для многих команд. С ростом популярности облачных технологий многие open-source проекты пересматривают свои лицензии, поэтому стоит убедиться, что выбранный вами инструмент будет оставаться доступным. Будет ли возможен форк и продолжение его развития в случае изменения лицензии? Ситуации с <a href="https://habr.com/ru/news/537610/"><u>ElasticSearch</u></a>, <a href="https://habr.com/ru/companies/flant/articles/804969/"><u>Redis</u></a> и <a href="https://habr.com/ru/companies/flant/news/754094/"><u>Terraform</u></a> показывают, что это не просто теория.</p></li><li><p><strong>Интеграция с OpenAPI</strong>. Большинство инструментов поддерживают импорт и экспорт OpenAPI спецификаций. Это удобная функция, которая избавляет от необходимости вручную воссоздавать структуру API. Например, используя Spring, можно легко подключить стартер <a href="https://springdoc.org"><u>springdoc-openapi</u></a>, чтобы генерировать спецификации автоматически.</p></li><li><p><strong>Разворачивание через Docker</strong>. В современных проектах часто используют Docker Compose для развертывания локального окружения, включая такие вспомогательные сервисы, как pgAdmin, Kafka UI или Mongo Express. Возможность добавить туда инструмент для тестирования API через Docker значительно упрощает работу. Если добавить автоматический импорт OpenAPI спецификации, которую генерирует бекенд, можно мгновенно получить доступ ко всем эндпоинтам прямо в удобном UI-интерфейсе.</p></li></ol><p>Помимо хорошо известного редактора запросов, Postman имеет отдельный инструмент для визуального программирования. Postman Flows позволяет тестировать достаточно большие сценарии с множеством запросов, которые выполняются последовательно, параллельно, может иметь логику ветвления и даже циклы! </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/070/542/3d1/0705423d12344698f9c3201878e5078f.png" height="1142" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/070/542/3d1/0705423d12344698f9c3201878e5078f.png" width="1600"/></figure><p>Все это очень занятно, но на наш взгляд, это больше инструмент для команды тестирования, чем для разработки. У нас не получилось выяснить, можно ли запускать Flow на CI. Другим большим минусом является то, что их нельзя положить в git, и соответственно - отслеживать историю изменений. Вообще, то что можно написать с помощью кода, лучше написать с помощью кода.</p><h2>Пишите скрипты</h2><p>Летом мы рассказывали вам о магии Markdown файлах в IntelliJ IDEA. Эти файлы могут быть местом для размещения наших curl вызовов, рядом с которыми можно и документацию написать. Помимо этого, мы можем писать скрипты на Kotlin, используя какой-нибудь ktor в качестве самого HTTP клиента. Не будем повторяться и просто дадим <a href="https://habr.com/ru/companies/spring_aio/articles/826930/"><u>ссылку</u></a> на статью. </p><p>Подход имеет как плюсы так и минусы. Во-первых, скрипты можно дебажить!  Во вторых, отметим невероятную гибкость Kotlin скриптов: можно протестировать практически какой-угодно сценарий. Однако, это же и его минус. Отсутствие хоть какого-то формализма может и, скорее всего, приведет к бардаку. Ну и конечно, отсутствие специализированной поддержки в IDE, а это именно то, что нам так нравится в HTTP Client от JetBrains.</p><h2>Старый добрый Swagger UI</h2><p>Если мы хотим вручную проверить работоспособность запроса в нашем Spring Boot приложении, то нам вполне подойдет стартер <code>springdoc-openapi</code>. </p><p>Подключая стартер, мы, во-первых, получаем openapi схему для нашего приложения; во-вторых, возможность вызова эндпоинтов из web ui. Swagger помогает чем может и для каждого запроса генерирует параметры по-умолчанию, хотя и не всегда правильно. Сформированный запрос можно скопировать в виде curl.</p><p>Инструмент может быть полезен и для команды QA, которая сможет протестировать запросы на развернутом стенде. Но, честно говоря, скорее они будут использовать упомянутый выше Postman или похожий инструмент.</p><p>Кроме того, в swagger-ui нельзя менять произвольные параметры запроса, такие как Headers. Доступны на редактирование будут только те из них, которые явно описаны в документации к запросу.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/db9/5b2/9db/db95b29db1f532bc114a119aa546a25d.png" height="1600" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/db9/5b2/9db/db95b29db1f532bc114a119aa546a25d.png" width="1339"/></figure><h2>А как бы выглядел идеальный HTTP Client?</h2><p>Является ли при всем перечисленном JetBrains HTTP Client пределом мечтаний? Можем ли мы выдумать более крутой инструмент? Хочется взять все лучшее из всех перечисленных выше подходов.</p><p>Идеальный HTTP Client должен обладать следующими возможностями:</p><ul><li><p>Глубокая интеграция с IDE: возможность генерации запросов на основе кода контроллеров, удобная навигация, автодополнение и визуальные элементы интерфейса, такие как зеленая стрелочка для запуска запросов.</p></li><li><p>Поддержка тестирования произвольных сценариев, включая сложные последовательности запросов с обработкой и сохранением результатов.</p></li><li><p>Возможность запуска на CI/CD: интеграция с процессами непрерывной интеграции для проверки корректности работы API на различных стадиях разработки.</p></li><li><p>Лёгкое совместное использование: возможность сохранять запросы в читаемом формате, которые можно положить в git и отслеживать изменения, а также делиться ими с коллегами по проекту.</p></li><li><p>Поддержка написания ассертов и обработки результатов с помощью языка, более привычного для Java разработчиков, чем JavaScript, к примеру, с помощью Kotlin.</p></li><li><p>Дебаг и пошаговое выполнение.</p></li></ul><p>И, кажется, все это есть в HTTP Client, кроме последних двух пунктов. Ну и конечно доступность в РФ - было бы неплохо. Дождемся ли мы такого инструмента? Кто знает…</p><hr/><ul><li><p><a href="https://habr.com/ru/companies/spring_aio/articles/852526/"><strong>Как жить без IntelliJ IDEA? Часть №1. Собери сам</strong></a></p></li><li><p><a href="https://habr.com/ru/companies/spring_aio/articles/854062/"><strong>Как жить без IntelliJ IDEA? Часть №2. Собери сам</strong></a></p></li><li><p><a href="https://habr.com/ru/companies/spring_aio/articles/866500/"><strong>Как жить без IntelliJ IDEA? Часть №4. Database Client</strong></a></p></li></ul><figure class="full-width"><img data-src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/725/c0c/ddb/725c0cddbe78ce9d7a9dd868c3632ad6.png" height="334" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/725/c0c/ddb/725c0cddbe78ce9d7a9dd868c3632ad6.png" width="1560"/></figure><p>Присоединяйтесь к русскоязычному сообществу
