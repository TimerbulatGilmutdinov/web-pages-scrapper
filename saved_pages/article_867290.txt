--><div class="tm-article-body" data-gallery-root="" lang="ru"><div><!--[--><!--]--></div><div id="post-content-body"><div><div class="article-formatted-body article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p><strong>Android 15 ворвался в наши жизни с новым усилением безопасности и удобством разработки. Сёрфер Полина, Android-разработчик в Surf, всё посмотрела, попробовала, проанализировала и теперь расскажет, что же там такого интересного.</strong> </p><p>Пристальное внимание создатели 15-й версии уделили защите данных пользователей, в том числе, обновлениям в управлении персональной информацией и создании защищенной зоны Private Space для приложений с дополнительной аутентификацией. </p><p>Ещё одно любопытное нововведение — активная поддержка планшетов и складных устройств, новые API и улучшенные инструменты. В Android Studio теперь можно менять размеры экранов виртуальных устройств — теперь разработка адаптивных приложений станет проще.</p><h3>Что нового</h3><h4>Контроль использования приложений на переднем дисплее</h4><p>Пользователи теперь будут решать, какие приложения смогут<strong> активировать передний дисплей</strong> при складывании устройства, а какие — нет. Это позволит, например, включать его для игр и видео.</p><p>Но это не всё. Помимо списков разрешённых и запрещённых программ для активации переднего дисплея, можно ввести режимы фокусировки. Они автоматически активируют приложения на переднем дисплее в зависимости от времени или активности пользователя — работа, отдых, хобби.</p><h4>Постоянная панель задач</h4><p>Теперь можно <strong>выбрать панель задач</strong> для устройств с большими экранами. Она может быть временная или постоянная.</p><p>По сути, это укороченная версия обычной панели задач Android, которую можно вывести с помощью «смахивания» в нижней части экрана. А ещё можно сделать так, чтобы она отображалась постоянно. </p><p>Так, четыре приложения доступны с панели задач. Пятая иконка даёт доступ к селектору для быстрого открытия других приложений в режиме разделенного экрана.</p><h4>Поддержка складных телефонов </h4><figure class="full-width"><img data-blurred="true" data-src="https://habrastorage.org/getpro/habr/upload_files/32f/25f/f63/32f25ff637003f170bdba0a4be0e29ab.jpeg" height="576" src="https://habrastorage.org/r/w780q1/getpro/habr/upload_files/32f/25f/f63/32f25ff637003f170bdba0a4be0e29ab.jpeg" width="1024"/></figure><p>Android 15 <strong>улучшил поддержку складных устройств</strong>: Samsung Galaxy Z Fold и Motorola Razr. Разработчики ввели новые API и функции, которые позволяют приложениям адаптироваться к различным конфигурациям экрана и переключениям между режимами (подробности ищите в <a href="https://developer.android.com/guide/topics/large-screens/make-apps-fold-aware?hl=ru"><u>рекомендациях для разработчиков</u></a>).</p><p>Ещё появилась <strong>функция непрерывности</strong>. Она позволяет автоматически переключаться между внутренним и внешним дисплеем при закрытии и открытии устройства. Когда смартфон складывается, изображение с внутреннего дисплея переводится на внешний.</p><blockquote><p><em>Внешний дисплей теперь можно использовать для просмотра презентаций и видео или для воспроизведения музыки, а внутренний — для заметок или управления слайдами. </em></p><p><em>Кроме того, уведомления будут автоматически отображаться на внешнем экране — так что пользователю не нужно открывать устройство для их прочтения. </em></p></blockquote><h4>Адаптивная вибрация </h4><p>Операционная система теперь может определять уровень шума в окружении и положение устройства, чтобы автоматически <strong>подстраивать силу вибрации</strong> уведомлений и звонков.</p><blockquote><p><em>Зачем это нужно? Чтобы, к примеру, увеличить силу вибрации в шумной обстановке и уменьшить её, когда пользователь находится в тихом месте.</em></p><p><em>А ещё можно настроить силу вибрации для важных событий или дедлайнов. Например, в приложениях для управления задачами или календарях.</em></p></blockquote><h4>Архивирование приложений </h4><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/a3e/694/a19/a3e694a193b73818b4cec5349e718387.png" height="600" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/a3e/694/a19/a3e694a193b73818b4cec5349e718387.png" width="800"/></figure><p>Киллер фича Android 15 — архивирование. Оно позволяет сохранять данные приложений без их полного удаления — процесс восстановления более удобным. Эта функция также может быть полезна для управления пространством на устройствах с ограниченной памятью. </p><blockquote><p><em>Допустим, на вашем устройстве есть старая-добрая GTA: San Andreas. Она весит 2,5 ГБ — это не считая пользовательских данных, которые она собирает. </em></p><p><em>Вдруг настаёт момент, когда памяти устройства начинает не хватать. И вы решаете удалить эту игру — вам нужно срочно установить другое приложение или скачать файлы там.</em></p><p><em>Вы её удалили, всё отлично. А потом файлы перестали быть вам нужны, и вы снова решаете вернуться в игру. </em></p><p><em>Вы её переустанавливаете и... начинате играть с нуля — все ваши достижения, машины, дома в игре — все исчезло. Потому что пользовательские данные удалены. Что делать? <s>Страдать</s> Проходить заново.  </em></p><p><em>Так вот функция архивирования поможет сохранить эти данные, не удаляя его полностью. </em></p></blockquote><p>Архивирование позволяет уменьшить «вес» приложения с 2,5 ГБ до 150 МБ.</p><h4>Управление виджетами</h4><p>Виджеты теперь становятся неактивными при принудительной остановке приложения.</p><h4>Предиктивная анимация возврата</h4><p><br/></p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/4fb/0da/f82/4fb0daf82712ab960bacfe173115ac54.gif" height="592" src="https://habrastorage.org/getpro/habr/upload_files/4fb/0da/f82/4fb0daf82712ab960bacfe173115ac54.gif" width="526"/></figure><p>Когда пользователь выполняет жест возврата — сделает свайп назад — система покажет анимацию, которая продемонстрирует, что будет на экране после завершения жеста. Это позволит пользователям понять, куда они вернутся после жеста.</p><h3>Мультимедиа</h3><h4>Улучшенное управление HDR </h4><p>Новые API позволяют разработчикам более точно контролировать параметры HDR, включая насыщенность и динамический диапазон. Это помогает создавать более качественные визуальные эффекты на устройствах с большими экранами.</p><figure class=""><img data-src="https://habrastorage.org/getpro/habr/upload_files/c8b/6dd/5c2/c8b6dd5c29ecf18628ff878fa9cdd4f5.png" height="351" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/c8b/6dd/5c2/c8b6dd5c29ecf18628ff878fa9cdd4f5.png" width="512"/></figure><blockquote><p><em>Разработчики смогут адаптировать параметры воспроизведения в зависимости от способностей устройства. Например, видеоплеер сможет автоматически определять тип экрана (HDR или SDR) и настраивать яркость и контрастность. </em></p></blockquote><h4>Поддержка динамической громкости </h4><p>Android 15 внедрил стандарт громкости CTA-2075. Он помогает избежать резких изменений уровня громкости при переключении между разными мультимедийными источниками. Это особенно полезно для пользователей, которые часто меняют контент на больших экранах.</p><blockquote><p><em>О чём тут речь? Когда ваш телефон на всю улицу начнёт орать песню из игры, хотя вы просто зашли собрать урожай, вы точно будете не в восторге. Но Android 15 всё исправил.</em></p><p><em>Теперь разработчики смогут обеспечить согласованность уровня звука при переключении между приложениями. Так он будет подстраиваться под предыдущий.</em></p></blockquote><h4>Режим веб-камеры высокого качества </h4><p>В Android 15 появился режим, который позволяет использовать устройства в качестве высококачественных веб-камер. Кстати, этот режим отключает оптимизацию энергопотребления — важно для всех, кто хочет подольше сохранить заряд.</p><h4>Расширенные возможности работы с аудио </h4><p>Новые API упрощают интеграцию с устройствами MIDI и поддерживают виртуальные MIDI-приложения. </p><p>Теперь музыкальные приложения смогут управлять синтезаторами так же, как если бы они использовали USB-устройства MIDI 2.0.</p><h4>Частичная демонстрация экрана</h4><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/18b/a08/765/18ba08765a418fa87c74f9eb6e857d20.png" height="705" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/18b/a08/765/18ba08765a418fa87c74f9eb6e857d20.png" width="897"/></figure><p>Пользователи смогут делиться только определенными окнами приложений. Так что теперь не будет неловких ситуаций, когда на звонке кто-то прислал вам глупое сообщение и все его увидели. </p><h4>Поддержка страниц размером 16 КБ</h4><p><br/>Android 15 теперь поддерживает устройства с 16 КБ страницами памяти. </p><blockquote><p><a href="https://developer.android.com/guide/practices/page-sizes?hl=ru"><em><u>Требует пересборки</u></em></a><em> приложений, использующих NDK-библиотеки, для корректной работы на таких устройствах.</em></p></blockquote><p>Устройства будут работать быстрее и эффективнее. Поддержка страниц уменьшит время загрузки приложений и системы, улучшит работу камеры и снизит потребление энергии.</p><h4>Поддержка Private Space</h4><p><em>Отличия Private Space в Android Это может повлиять на приложения, которые работают с аудио.</p><h4>Изменения в поведении служб переднего плана</h4><p>Введены ограничения на время работы служб dataSync и mediaProcessing — максимум 6 часов за 24 часа. Для корректного завершения служб нужно реализовать метод Service.onTimeout(int, int).</p><h4>Удаление PNG-формата эмодзи</h4><p>В Android 15 больше нет старого шрифтп эмодзи (NotoColorEmojiLegacy.ttf), остались только векторные файлы. Придётся адаптировать приложения для использования новых форматов.</p><h4>Ограничения на запуск служб при получении BOOT_COMPLETED</h4><p>Новые ограничения запрещают запуск служб приоритета (dataSync, camera и других.) при получении широковещательного сообщения BOOT_COMPLETED.</p><h3>Рекомендации для разработчиков</h3><h4>Обновление конфигурации сборки</h4><p>Для использования новых API нужно обновить файл build.gradle:</p><pre><code class="kotlin">android {

Давайте разберёмся, как реализовать эту функцию.</p><h3>Реализация функции непрерывности</h3><h4>Адаптивный интерфейс</h4><p>С помощью Configuration определяем текущее состояние экрана (сложенный или разложенный) и устанавливаем макет.</p><pre><code class="kotlin">override fun onConfigurationChanged(newConfig: Configuration) {
	super.onConfigurationChanged(newConfig)
    
	if (newConfig.isScreenWide()) {
    	// Измените макет для широкого экрана
    	setContentView(R.layout.activity_wide)
	} else {
    	// Измените макет для узкого экрана
    	setContentView(R.layout.activity_narrow)
	}
}</code></pre><h3>Обработка переключений между дисплеями</h3><p>Для отслеживания состояния дисплеев и переключения контента между ними используем <code>DisplayManager</code></p><pre><code class="kotlin">val displayManager = getSystemService(Context.DISPLAY_SERVICE) as DisplayManager
displayManager.registerDisplayListener(object : DisplayManager.DisplayListener {
	override fun onDisplayAdded(displayId: Int) {
    	// Логика при добавлении дисплея
	}

	override fun onDisplayRemoved(displayId: Int) {
    	// Логика при удалении дисплея
	}

	override fun onDisplayChanged(displayId: Int) {
    	// Логика при изменении состояния дисплея
    	updateContentForDisplay(displayId)
	}
}, null)

// Переключение контента между экранами
private fun updateContentForDisplay(displayId: Int) {
	val display = displayManager.getDisplay(displayId)
	if (display != null &amp;&amp; display.isValid) {
    	// Получите информацию о текущем контенте и перенесите его на другой экран
    	transferContentToExternalDisplay()
	}
}</code></pre><h4>Изменения в поведении приложений </h4><p>Изменения в API-level 35 повлияют на совместимость приложений по нескольким аспектам.</p><h4>Ужесточение требований к целевому уровню API </h4><p>С 31 августа 2024 года все новые приложения и обновления должны поддерживать Android 14 (API 34) или выше. </p><p>Это значит, приложения, не обновленные до этого уровня, могут стать недоступными для установки на устройствах с более новыми версиями Android.</p><h4>Изменения в разрешениях на доступ к мультимедиа </h4><p>В Android 15 появились новые разрешения для доступа к медиафайлам.
