xmlns="http://www.w3.org/1999/xhtml"><p>Как я уже неоднократно повторял, Amplicode придерживается <strong>CDD – Community Driven Development</strong>’а. По этой причине мы стараемся по максимумому общаться с комьюнити разработчиков, узнавать у них, что им нравится, что нет, чем пользуются, а чем нет, и почему. И мы далеко не всегда разговариваем только про Amplicode, часто диалог уходит в разговоры о технологиях, вечных вопросах программирования или других инструментах. </p><p>Так вот, практически каждый раз, когда мы говорили про IDE, а конкретно про IntelliJ IDEA Ultimate и её самые любимые фичи, мы так или иначе слышали следующие два слова: HTTP клиент. </p><p>Мы решили, что нам вполне по силам будем реализовать свой собственный HTTP-клиент и пару недель назад, в <a href="https://t.me/amplicode/116">последнем мажорном релизе</a> 2024-го года, мы его выпустили! </p><p>Сегодня я расскажу, чем наш клиент лучше HTTP-клиента от JetBrains, покажу базовые сценарии его использования, а также немного расскажу о планах на будущее. </p><p>Статья также доступна в формате видео на <a href="https://youtu.be/P-aN3GuWYvo">YouTube</a>, <a href="https://vk.com/video-222549074_456239144">VK Видео</a> и <a href="https://rutube.ru/video/88af5d56692e5df6d68f661eb3e49e66/?r=wd">RUTUBE</a>, так что можно и смотреть, и читать — как вам удобнее! </p><div class="tm-iframe_temp" data-src="https://embedd.srv.habr.com/iframe/67643414c07fea6aff120738" data-style="" id="67643414c07fea6aff120738" width=""></div><details class="spoiler"><summary>Спойлер</summary><div class="spoiler__content"><p><em>В большинстве случаев я буду использовать изображения для демонстрации фрагментов кода. Такой подход позволит мне выделить важные части и более подробно их объяснить. Если вы хотите проверить всё самостоятельно, запустив код, о котором пойдет речь, то найти его можно на </em><a href="https://github.com/Amplicode/amplicode-tutorials/blob/main/crud-methods-generation/README.md"><em>GitHub</em></a><em>.</em> </p></div></details><h2>Тезисно про HTTP клиент от Amplicode</h2><p>Итак, тезисно о нашем решении и основных его преимуществах: </p><ul><li><p>Есть глубокая интеграция с IDE </p></li><li><p>Существует возможность тестировать произвольные сценарии, включая сложные последовательности запросов с обработкой и сохранением результатов </p></li><li><p>Использовать в команде супер легко – сохраняйте запросы в читаемом формате, располагайте в Git и отслеживайте изменения, а также делитесь ими с коллегами по проекту </p></li><li><p>А писать <code>assert</code>’ы можно на хорошо знакомым многим Kotlin’е </p></li></ul><p> А в будущем также планируем реализовать </p><ul><li><p>Возможность запуска на CI/CD </p></li><li><p>И дебаг запросов с его пошаговым выполнением </p></li></ul><p>Но на словах и Database Navigator прекраснейшая вещь, однако как доходит до дела – возникают вопросики. Так что давайте проверим наше решение на практике!  </p><h2>Простые GET и POST запросы</h2><p>Я возьму приложение, которое разрабатывал в <a href="https://habr.com/ru/companies/haulmont/articles/866060/">статье про CRUD REST API</a>, запущу необходимые сервисы (1) и само приложение (2).  </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/f56/5e9/2a7/f565e92a74248814cba6f98f74e0d7b8.png" height="1600" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/f56/5e9/2a7/f565e92a74248814cba6f98f74e0d7b8.png" width="989"/></figure><p> Первым делом я хочу дёрнуть эндпоинт для получения записи по <code>ID</code>. </p><pre><code class="java">@GetMapping("/{id}")
public OwnerDto getOne(@PathVariable Integer id) {
        Optional&lt;Owner&gt; ownerOptional = ownerRepository.findById(id);
        return ownerMapper.toOwnerDto(ownerOptional.orElseThrow(
                () -&gt; new ResponseStatusException(HttpStatus.NOT_FOUND,
					"Owner with id `%d` not found".formatted(id))
        ));
}</code></pre><p>Как я уже говорил ранее, наш HTTP-клиент имеет глубокую интеграцию с IDE, а это значит, что нам не нужно будет писать запросы вручную целиком для существующих эндпоинтов контроллера. Вместо этого достаточно нажать на <strong>Gutter</strong>-иконку напротив нужного эндпоинта и выбрать действие <strong>Generate HTTP Request</strong>.  </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/f86/567/614/f86567614d2b254fc52c7e483c30388f.png" height="1028" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/f86/567/614/f86567614d2b254fc52c7e483c30388f.png" width="1480"/></figure><p> Amplicode сгенерировал код запроса, корректно обработал часть запроса с <code>PathVariable</code> и сразу же предлагает подставить значение через конструкцию <code>pathParams</code>.  </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/b76/e81/6f1/b76e816f180d5d76dfc650c30c47c03c.png" height="438" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/b76/e81/6f1/b76e816f180d5d76dfc650c30c47c03c.png" width="1174"/></figure><p> Давайте получим запись с <code>id</code> “1”. </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/a71/259/82e/a7125982e6b641d6ab07e6fb77133611.png" height="432" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/a71/259/82e/a7125982e6b641d6ab07e6fb77133611.png" width="1160"/></figure><p>Отлично! Запрос выполнился успешно! И прямо в IDE мы можем посмотреть, что из себя представлял сам запрос, включая используемые заголовки и тело запроса, а также то, как выглядит ответ.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/85b/e33/a40/85be33a405ef4136f2dcacde044108dd.png" height="1600" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/85b/e33/a40/85be33a405ef4136f2dcacde044108dd.png" width="1168"/></figure><p> Аналогичным образом давайте сгенерируем запрос для получения всех записей.  </p><p> Сам код энпоинта выглядит так: </p><pre><code class="java">@GetMapping
public PagedModel&lt;OwnerMinimalDto&gt; getAll(@ModelAttribute OwnerFilter filter,
                                              Pageable pageable) {
        Specification&lt;Owner&gt; spec = filter.toSpecification();
        Page&lt;Owner&gt; owners = ownerRepository.findAll(spec, pageable);
        Page&lt;OwnerMinimalDto&gt; ownerMinimalDtoPage = owners.map(ownerMapper::toOwnerMinimalDto);
        return new PagedModel&lt;&gt;(ownerMinimalDtoPage);
}</code></pre><p>Воспользуемся уже знакомым нам действием:  </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/52a/599/e73/52a599e73442f28733d74b1b96f63a08.png" height="600" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/52a/599/e73/52a599e73442f28733d74b1b96f63a08.png" width="1600"/></figure><p> Amplicode сгенерировал нам <strong>GET</strong>-запрос:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/398/146/f74/398146f74674710f97fdde7b7f152c82.png" height="620" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/398/146/f74/398146f74674710f97fdde7b7f152c82.png" width="1168"/></figure><p> В запрос мы можем добавить и параметры запроса.   </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/006/4fe/7d6/0064fe7d653c1b4a5814a73245ad8672.png" height="710" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/006/4fe/7d6/0064fe7d653c1b4a5814a73245ad8672.png" width="1600"/></figure><p> Схожим с передачей параметров образом, обратимся к методу <code>queryParam</code> и укажем, что нам нужны все записи, у которых имя начинается с буквы “J”. </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/d7b/579/5a7/d7b5795a7e9d8cc9dbb34af685dabefc.png" height="620" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/d7b/579/5a7/d7b5795a7e9d8cc9dbb34af685dabefc.png" width="1182"/></figure><p> Опять же можем изучить запрос и ответ прямо в IDE. Для этого отправляем нам запрос: </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/6b4/508/803/6b4508803b2aeb35a0f9ca2e0dbbf744.png" height="672" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/6b4/508/803/6b4508803b2aeb35a0f9ca2e0dbbf744.png" width="1164"/></figure><p>И получаем ответ: </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/c31/cd2/3fe/c31cd23fe04c18c7426f1de2cd227f60.png" height="1600" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/c31/cd2/3fe/c31cd23fe04c18c7426f1de2cd227f60.png" width="1238"/></figure><p>Ну и наконец, давайте сгенерим запрос к эндпоинту создания нового владельца животных. Напомню, как выглядит сам эндпоинт: </p><pre><code class="java">@PostMapping
public OwnerDto create(@RequestBody @Valid OwnerDto dto) {
        if (dto.getId() != null) {
            throw new ResponseStatusException(HttpStatus.BAD_REQUEST, "Id must be null");
        }
        Owner owner = ownerMapper.toEntity(dto);
        Owner resultOwner = ownerRepository.save(owner);
        return ownerMapper.toOwnerDto(resultOwner);

Нажимаем ПКМ по <strong>Configurations</strong> (1) -&gt; выбираем <strong>Spring Security Configuration</strong> из списка со всеми поддерживаемыми Amplicode конфигурациями (2). </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/2b9/62f/61d/2b962f61d37a20c333869bcf98084662.png" height="1600" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/2b9/62f/61d/2b962f61d37a20c333869bcf98084662.png" width="861"/></figure><p> Откроется окно для настройки конфигурации. import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.Customizer;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.config.annotation.web.configurers.AbstractHttpConfigurer;
import org.springframework.security.core.userdetails.User;
import org.springframework.security.core.userdetails.UserDetailsService;
import org.springframework.security.provisioning.InMemoryUserDetailsManager;
import org.springframework.security.web.SecurityFilterChain;

@Configuration
@EnableWebSecurity
public class WebSecurityConfiguration {

	@Bean
	public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
		http.authorizeHttpRequests(authorizeHttpRequests -&gt; authorizeHttpRequests
			.anyRequest().authenticated());
		http.headers(Customizer.withDefaults());
		http.sessionManagement(Customizer.withDefaults());
		http.formLogin(Customizer.withDefaults());
		http.anonymous(Customizer.withDefaults());
		http.csrf(AbstractHttpConfigurer::disable);
		http.userDetailsService(inMemoryUserDetailsService());
		return http.build();
	}

	public UserDetailsService inMemoryUserDetailsService() {
		User.UserBuilder users = User.builder();
		InMemoryUserDetailsManager userDetailsManager = new InMemoryUserDetailsManager();
		userDetailsManager.createUser(users.username("admin")
			.password("{noop}admin")
			.roles("ADMIN")
			.build());
		userDetailsManager.createUser(users.username("user")
			.password("{noop}user")
			.roles("USER")
			.build());
		return userDetailsManager;
	}
}
