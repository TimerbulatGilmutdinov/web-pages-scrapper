xmlns="http://www.w3.org/1999/xhtml"><p>Команда <a href="https://t.me/+QmrKn0wA8CdkNjAy">Spring АйО</a> перевела статью, которая отлично подойдёт тем, кто ещё не знаком со Spring Data Envers. В статье на простых примерах объясняется, как отслеживать изменения данных в приложении, используя этот инструмент.</p><hr/><h3>Введение</h3><p>В этой статье мы рассмотрим проект Spring Data Envers и разберемся, как извлечь из него максимум пользы.</p><p><a href="https://hibernate.org/orm/envers/"><u>Hibernate Envers</u></a> — это расширение Hibernate ORM, которое позволяет отслеживать изменения сущностей с минимальными изменениями на уровне приложения.</p><p>Так же, как Envers интегрируется с Hibernate ORM для ведения журнала изменений сущностей, проект Spring Data Envers подключается к Spring Data JPA, чтобы добавить возможность ведения журнала изменений с использованием JPA репозиториев.</p><h3>Доменная модель</h3><p>Предположим, у нас есть сущность <code>Post</code>, которая отмечена аннотацией <code>@Audited</code> из проекта Hibernate Envers:</p><pre><code class="java">@Entity
@Table(name = "post",
       uniqueConstraints = @UniqueConstraint(
               name = "UK_POST_SLUG", columnNames = "slug"
       )
)
@Audited
public class Post { ⠀
   @Id
   @GeneratedValue
   private Long id; ⠀
   @Column(length = 100)
   private String title; ⠀
   @NaturalId
   @Column(length = 75)
   private String slug; ⠀
   @Enumerated(EnumType.ORDINAL)
   @Column(columnDefinition = "NUMERIC(2)")
   private PostStatus status;
}</code></pre><p>Сущность <code>Post</code> имеет дочернюю сущность <code>PostComment</code>, которая также аннотирована <code>@Audited</code>:</p><pre><code class="java">@Entity
@Table(name = "post_comment")
@Audited
public class PostComment { ⠀
   @Id
   @GeneratedValue
   private Long id; ⠀
   @Column(length = 250)
   private String review; ⠀
   @ManyToOne(fetch = FetchType.LAZY)
   @JoinColumn(foreignKey = @ForeignKey(name = "FK_POST_COMMENT_POST_ID"))
   private Post post;
}</code></pre><details class="spoiler"><summary>Скрытый текст</summary><div class="spoiler__content"><p><em>Как я объяснял в </em><a href="https://vladmihalcea.com/the-best-way-to-implement-an-audit-log-using-hibernate-envers/"><em><u>этой статье</u></em></a><em>, мы будем использовать стратегию </em><code>ValidityAuditStrategy</code><em>, так как она может ускорить выполнение запросов связанных с журналом изменений.</em></p><p><em>Чтобы включить стратегию </em><code>ValidityAuditStrategy</code><em>, необходимо установить следующее свойство Hibernate-конфигурации:</em></p><pre><code class="java">properties.setProperty(
       EnversSettings.AUDIT_STRATEGY,
       ValidityAuditStrategy.class.getName()
);</code></pre><p></p></div></details><p>При генерации схемы с использованием <a href="https://vladmihalcea.com/hibernate-hbm2ddl-auto-schema/"><u>инструмента hbm2ddl</u></a>, Hibernate создаст следующие таблицы в базе данных:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/80d/946/57b/80d94657b74f4b6c3cd93ac1e86435b4.png" height="1600" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/80d/946/57b/80d94657b74f4b6c3cd93ac1e86435b4.png" width="1526"/></figure><p>Каждый раз, когда транзакция завершается, создается ревизия, которая сохраняется в таблице <code>revinfo</code>.</p><p>Таблица <code>post_aud</code> отслеживает изменения записей в таблице <code>post</code>, а таблица <code>post_comment_aud</code> хранит информацию о журнале изменений для таблицы <code>post_comment</code>.</p><h3>Spring Data Envers Репозитории</h3><p>Проект Spring Data Envers предоставляет интерфейс <a href="https://docs.spring.io/spring-data/commons/docs/current/api/org/springframework/data/repository/history/RevisionRepository.html">RevisionRepository</a>, который ваши JPA-репозитории могут расширять, чтобы добавить возможность выполнения запросов связанных с журналом изменений.</p><p>Например, репозиторий <code>PostRepository</code> расширяет <code>JpaRepository</code> из Spring Data JPA и <code>RevisionRepository</code> из Spring Data Envers:</p><pre><code class="java">@Repository
public interface PostRepository extends JpaRepository&lt;Post, Long&gt;, 
                                        RevisionRepository&lt;Post, Long, Long&gt; {

}</code></pre><p>Точно так же <code>PostCommentRepository</code> расширяет как <code>JpaRepository</code>, так и <code>RevisionRepository</code> из Spring Data Envers:</p><pre><code>@Repository
public interface PostCommentRepository extends JpaRepository&lt;PostComment, Long&gt;,
                                  RevisionRepository&lt;PostComment, Long, Long&gt; {
   void deleteByPost(Post post);
}</code></pre><p>На сервисном слое у нас есть класс <code>PostService</code>, который предоставляет методы для сохранения и удаления сущностей <code>Post</code> и <code>PostComment</code>. Эти методы помогут нам увидеть, как работает механизм ведения журнала изменений:</p><pre><code class="java">@Transactional(readOnly = true)
public class PostService {
   @Autowired
   private PostRepository postRepository;
   @Autowired
   private PostCommentRepository postCommentRepository;


   @Transactional
   public Post savePost(Post post) {
       return postRepository.save(post);
   }


   @Transactional
   public Post savePostAndComments(Post post,
                                   PostComment... comments) {
       post = postRepository.save(post);
       if (comments.length &gt; 0) {
           postCommentRepository.saveAll(Arrays.asList(comments));
       }
       return post;
   }


   @Transactional
   public void deletePost(Post post) {
       postCommentRepository.deleteByPost(post);
       postRepository.delete(post);
   }
}
</code></pre><h3>Отслеживание операций INSERT, UPDATE и DELETE</h3><p>При создании родительской сущности <code>Post</code> вместе с двумя дочерними сущностями <code>PostComment</code>:</p><pre><code class="java">Post post = new Post()
       .setTitle("High-Performance Java Persistence 1st edition")
       .setSlug("high-performance-java-persistence")
       .setStatus(PostStatus.APPROVED);
postService.savePostAndComments(
       post,
       new PostComment()
               .setPost(post)
               .setReview("A must-read for every Java developer!"),
       new PostComment()
               .setPost(post)
               .setReview("Best book on JPA and Hibernate!")
);</code></pre><p>Hibernate сгенерирует следующие SQL-запросы:</p><pre><code class="sql">SELECT nextval('post_SEQ')
SELECT nextval('post_comment_SEQ')
SELECT nextval('post_comment_SEQ')


INSERT INTO post (slug, status, title, id)
VALUES ( 'high-performance-java-persistence', 1, 'High-Performance Java Persistence 1st edition', 1 )


INSERT INTO post_comment (post_id, review, id)
VALUES ( 1, 'A must-read for every Java developer!', 1 ),
      ( 1, 'Best book on JPA and Hibernate!', 2 )


SELECT nextval('REVINFO_SEQ')


INSERT INTO REVINFO (REVTSTMP, REV)
VALUES (1726724588078, 1)


INSERT INTO post_AUD (REVEND, REVTYPE, slug, status, title, REV, id)
VALUES ( null, 0, 'high-performance-java-persistence', 1, 'High-Performance Java Persistence 1st edition', 1, 1 )


INSERT INTO post_comment_AUD (REVEND, REVTYPE, post_id, review, REV, id)
VALUES ( null, 0, 1, 'A must-read for every Java developer!', 1, 1 ),
      ( null, 0, 1, 'Best book on JPA and Hibernate!', 1, 2 )</code></pre><p>В то время как Hibernate ORM выполняет INSERT-запросы для записей в таблицах <code>post</code> и <code>post_comment</code>, Hibernate Envers создает записи в таблицах <code>REVINFO</code>, <code>post_AUD</code> и <code>post_comment_AUD</code>.</p><p>При изменении сущности <code>Post</code>:</p><pre><code class="java">post.setTitle("High-Performance Java Persistence 2nd edition");
postService.savePost(post);</code></pre><p>Hibernate сгенерирует следующие запросы:</p><pre><code class="sql">SELECT p1_0.id, p1_0.slug, p1_0.status, p1_0.title
FROM post p1_0
WHERE p1_0.id = 1 UPDATE post


SET status = 1, title = 'High-Performance Java Persistence 2nd edition'
WHERE id = 1


SELECT nextval('REVINFO_SEQ')


INSERT INTO REVINFO (REVTSTMP, REV)
VALUES (1726724799884, 2)


INSERT INTO post_AUD (REVEND, REVTYPE, slug, status, title, REV, id)
VALUES ( null, 1, 'high-performance-java-persistence', 1, 'High-Performance Java Persistence 2nd edition', 2, 1 )


UPDATE post_AUD
SET REVEND = 2
WHERE id = 1 AND REV &lt;&gt; 2 AND REVEND IS NULL</code></pre><p>Обратите внимание, что была создана новая запись в <code>REVINFO</code>, которая связана с записью в <code>post_AUD</code>.</p><p>А при удалении сущности <code>Post</code>:</p><pre><code class="sql">postService.deletePost(post);</code></pre><p>Hibernate выполнит следующие запросы:</p><pre><code class="sql">SELECT pc1_0.id, pc1_0.post_id, pc1_0.review
FROM post_comment pc1_0
WHERE pc1_0.post_id = 1


SELECT p1_0.id,p1_0.slug,p1_0.status,p1_0.title
FROM post p1_0
WHERE p1_0.id = 1


DELETE
FROM post_comment
WHERE id = 1

WHERE id = 2

INSERT

VALUES ( null, 2, null, null, 3, 1 ),
      ( null, 2, null, null, 3, 2 )


INSERT
INTO post_AUD (REVEND, REVTYPE, slug, status, title, REV, id)
VALUES ( null, 2, null, null, null, 3, 1 )


UPDATE post_comment_AUD
SET REVEND = 3
WHERE id = 1 AND REV &lt;&gt; 3 AND REVEND IS NULL


UPDATE post_comment_AUD
SET REVEND = 3
WHERE id = 2 AND REV &lt;&gt; 3 AND REVEND IS NULL


UPDATE post_AUD
SET REVEND = 3
WHERE id = 1 AND REV &lt;&gt; 3 AND REVEND IS NULL</code></pre><h3>Загрузка ревизий с использованием Spring Data Envers</h3><p>Интерфейс <code>RevisionRepository</code> из Spring Data Envers предоставляет несколько методов для загрузки ревизий сущностей.</p><p>Например, если вы хотите загрузить последнюю ревизию сущности <code>Post</code>, можно использовать метод <code>findLastChangeRevision</code>, который унаследован от <code>RevisionRepository</code>:</p><pre><code class="java">Revision&lt;Long, Post&gt; latestRevision = postRepository.findLastChangeRevision(post.getId())
       .orElseThrow();
LOGGER.info("The latest Post entity operation was [{}] at revision [{}]", latestRevision.getMetadata()
       .getRevisionType(), latestRevision.getRevisionNumber()
