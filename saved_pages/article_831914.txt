любительской статье разберемся, что такое KeyStore в контексте мобильной разработки, для чего нужен и применим его в крайне легком варианте. Статья будет разделена на 3 так называемых раздела - KeyStore, UI и ViewModel. </p><p>Целью данного гайда это вот такой экран с последующим переходом:</p><figure class=""><img data-src="https://habrastorage.org/getpro/habr/upload_files/25f/953/3a1/25f9533a14a32ee84f1059b626deb446.png" height="745" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/25f/953/3a1/25f9533a14a32ee84f1059b626deb446.png" width="332"/></figure><p>P.S. Хочу сразу предупредить, что мой код не является показательным или каким-то супер профессиональным, я просто делал, то, что мог, именно поэтому, если есть желание, то жду комментариев с пояснениями, где я мог лучше.</p><h2>KeyStore</h2><p><a href="https://developer.android.com/reference/java/security/KeyStore" rel="noopener noreferrer nofollow">Инструмент </a>для хранения и управления чувствительными данными, такими, как пароли, ключи, коды и сертификаты, в безопасном хранилище. Данный API используют не только в контексте мобильной разработки под Android, но и на серверах в качестве безопасного места для хранения сертификатов. Я не буду углубляться в понятие KeyStore, так как это достаточно специализированная тема, да и к тому же есть множество гайдов раскрывающие эту тему, в том числе на хабре (<a href="https://habr.com/ru/articles/445786/" rel="noopener noreferrer nofollow">Хабр статья</a>, <a href="https://learnitweb.com/java-security/a-guide-to-java-keystore-and-keystore-api/" rel="noopener noreferrer nofollow">Англоязычная</a> от EditorialTeam in Java security - более подробная). </p><p>Мы же возьмем от него только самое необходимое, а точнее модули EncryptedSharedPreferences и MasterKey.</p><p>Перед использованием KeyStore необходимо его заиплементить в build.gradle</p><pre><code class="kotlin">// Keystore
implementation("androidx.security:security-crypto:1.1.0-alpha06")</code></pre><pre><code class="kotlin">class KeystoreManager(context: Context) {
    private val masterKeyAlias = MasterKey.Builder(context)
        .setKeyScheme(MasterKey.KeyScheme.AES256_GCM)
        .build()

    private val sharedPreferences = EncryptedSharedPreferences.create(
        context,
        "encrypted_prefs",
        masterKeyAlias,
        EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
        EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM
    )

    fun savePin(pin: String) {
        sharedPreferences.edit().putString("pin", pin).apply()
    }

    fun getPin(): String? {
        return sharedPreferences.getString("pin", null)
    }
}</code></pre><p>Создаем такой класс для последующего использования в качестве хранилища. В начале создаем masterKey. Он используется для защиты как ключей, так и значений внутри <code>SharedPreference</code>. И выбираем схему шифрования для самого ключа. В данном случае используется AES-256 в режиме GCM (Galois/Counter Mode). Этот режим шифрования обеспечивает конфиденциальность и целостность данных, а также аутентификацию. <br/>Для <code>SharedPreference</code> мы используем EncryptedSharedPreferences. PrefKeyEncryptionScheme схема шифрования для ключей в <code>SharedPreferences</code>, а PrefValueEncryptionScheme схема шифрования для значений в <code>SharedPreferences</code>.  <br/>И собственно сами методы по сохранению пина и получению его.</p><p>Является ли безопасным метод getPin()? Да. Потому что используется с уже зашифрованным sharedprefs и дешифруется в контексте данного класса. Последующее использование будет гарантировать безопасность, только если вы не решите оставить лог с данным пином.</p><h2>Разработка UI</h2><p>Отчасти именно из-за того, что нигде в интернете нет более удобных решений для PinLockScreen с JetpackCompose я решил написать простенький гайд, для сохранения своих знаний и, возможно, помощи другим. К слову, есть <a href="https://github.com/raheemadamboev/pin-lock-compose" rel="noopener noreferrer nofollow">библиотека </a>на просторах гитхаба, осуществляющая UI пинлока (без какой-либо логики защиты), однако, использование сторонних библиотек для осуществления легких элементов является крайне нежелательным.</p><p>В качестве цветов я использую цвета, которые я прописал в Theme, сделано это для того, чтобы поддержать темную и светлую тему, вы можете использовать любые цвета.</p><p> Далее цвета в моем проекте определяются так:</p><pre><code class="kotlin">MaterialTheme.colorScheme.tertiary</code></pre><p>Самое главное в пинлоке это клавиатура, а она в свою очередь состоит из кнопок, сделаем кнопку, в которую будем подставлять content с аннотацией Composable, тк в качестве контента будет элемент Text().</p><pre><code>@Composable

И интерфейс с events, которые будут использованы как действия в приложении.</p><pre><code class="kotlin">data class PinLockState(
    val isAuthenticated: Boolean = false,
    val inputPin: String = "",
    val error: ErrorPin? = null,
    val confirmPin: String? = null
)

sealed interface PinLockEvent {

    data class AddDigit(val digit: String) : PinLockEvent
    data object DeleteDigit : PinLockEvent
    data object ClearPin : PinLockEvent
    data object CheckPin : PinLockEvent

}</code></pre><p>В стейте лежит флажок isAuthenticated для того, чтобы понять, есть ли у пользователя уже пинкод или нет, inputPin — сам пинкод, куда будут входить вводимые значения пользователя, error — ошибки, я использую также в качестве флага, для последующего входа, confirmPin — тоже пинкод, только нужен для регистрации и только.</p><p>Сама ViewModel представлена ниже.</p><pre><code class="kotlin">class PinLockViewModel(
    private val keystoreManager: KeystoreManager
) : ViewModel() {

    private val _state = MutableStateFlow(PinLockState())
    val state: StateFlow&lt;PinLockState&gt; = _state

    init {
        val pinSet = keystoreManager.getPin().isNullOrBlank()
        _state.update {
            it.copy(
                isAuthenticated = !pinSet
            )
        }
    }

    fun onEvent(event: PinLockEvent) {
        when (event) {
            is PinLockEvent.DeleteDigit -&gt; {
                _state.update {
                    it.copy(
                        inputPin = it.inputPin.substring(0, it.inputPin.length - 1)
                    )
                }
            }
            is PinLockEvent.AddDigit -&gt; {
                _state.update {
                    it.copy(
                        inputPin = it.inputPin.plus(event.digit)
                    )
                }
            }
            is PinLockEvent.ClearPin -&gt; {
                _state.update {
                    it.copy(
                        inputPin = ""
                    )
                }
            }
            is PinLockEvent.CheckPin -&gt; {
                val currentState = _state.value
                if (currentState.inputPin.length == 4) {
                    if (currentState.isAuthenticated) {
                        val storedPin = keystoreManager.getPin()
                        if (currentState.inputPin == storedPin) {
                            _state.update {
                                it.copy(
                                    error = ErrorPin.SUCCESS
                                )
                            }
                        } else {
                            _state.update {
                                it.copy(
                                    error = ErrorPin.INCORRECT_PASS
                                )
                            }

                            onEvent(PinLockEvent.ClearPin)
                        }
                    } else {
                        if (currentState.confirmPin.isNullOrBlank()) {
                            _state.update {
                                it.copy(
                                    confirmPin = currentState.inputPin,
                                    error = ErrorPin.TRY_PIN
                                )
                            }
                            onEvent(PinLockEvent.ClearPin)
                        } else {
                            if (currentState.inputPin == currentState.confirmPin) {
                                keystoreManager.savePin(currentState.inputPin)
                                _state.value = currentState.copy(
                                    isAuthenticated = true,
                                    error = ErrorPin.SUCCESS,
                                    confirmPin = null
                                )
                            } else {
                                _state.value = currentState.copy(error = ErrorPin.INCORRECT_PASS, confirmPin = null)
                                onEvent(PinLockEvent.ClearPin)
                            }
                        }
                    }
                } else {
                    _state.value = currentState.copy(error = ErrorPin.NOT_ENOUGH_DIG)
                    onEvent(PinLockEvent.ClearPin)
                }
            }
        }
    }


}

enum class ErrorPin(val error: Int){
    SUCCESS(0),
    INCORRECT_PASS(1),
    NOT_ENOUGH_DIG(2),
    TRY_PIN(3)
}
</code></pre><p>В начале оглашаем StateFlow для последующего использования и в качестве init производим идентификацию, есть ли вообще у пользователя пароль или нет. Далее это действия (onEvent).
