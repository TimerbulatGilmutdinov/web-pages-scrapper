class="tm-article-body" data-gallery-root="" lang="ru"><div><!--[--><!--]--></div><div id="post-content-body"><div><div class="article-formatted-body article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p>Привет, Android разработчики!</p><p>Хотел бы рассказать вам про мою библиотеку (<a href="https://github.com/IlyaGulya/anvil-utils" rel="noopener noreferrer nofollow"><strong>Anvil Utils</strong></a>) - кодогенератор позволяющий слегка уменьшить количество болилерплейта в многомодульных проектах использующих Dagger 2 и Anvil.</p><h3>В чём собственно проблема?</h3><p>Представьте: у вас есть многомодульное приложение.<br/>Почти наверняка у вас есть набор фичей, каждая из которых состоит из <code>api</code> и <code>implementation</code> модулей.<br/>Представим, что один из модулей предоставляет фабрику какой-то сущности (например, Decompose компонента).<br/>Если у вас в проекте используется Dagger 2, то реализацию этой фабрики логичнее всего отдать ему, используя Assisted Injection.<br/>Однако, вам всё равно необходимо каким-то образом предоставить биндинг этой Assisted Factory к вашему публичному интерфейсу из модуля <code>api</code>.</p><p>Попробуем посмотреть на варианты как это можно сделать.</p><h3>Голый Dagger 2</h3><p>Предположим, у вас есть подобный публичный API вашего feature модуля.</p><pre><code class="kotlin">interface MyClass {
	interface Factory {
	    fun create(param1: String, param2: Int): MyClass
	}
}</code></pre><p>Обычно чтобы реализовать подобный API и сделать его доступным в вашем графе зависимостей, нужно:</p><ol><li><p>Создаём <code>@AssistedFactory</code> в <code>implementation</code> модуле, наследуем её от публичного интерфейса фабрики <code>MyClass.Factory</code></p></li><li><p>Пишем Dagger модуль который занимается биндингом сгенерированной Dagger 2 фабрики к публичному интерфейсу фабрики из модуля <code>api</code>.</p></li></ol><p>Вот пример:</p><pre><code class="kotlin">class DefaultMyClass @AssistedInject constructor(
    @Assisted param1: String,
    @Assisted param2: Int
) : MyClass {
	@AssistedFactory
	interface Factory: MyClass.Factory {
		override fun create(param1: String, param2: Int): DefaultMyClass
	}
}

@Module
interface MyClassFactoryBindingModule {
	@Binds
	fun bindFactory(impl: DefaultMyClass.Factory): MyClass.Factory
}</code></pre><p>Также, вам придётся поддерживать руками набор параметров во всех трёх сущностях:</p><ul><li><p>Параметры конструктора</p></li><li><p>Аргументы публичного интерфейса фабрики</p></li><li><p>Аргументы assisted фабрики Dagger</p></li></ul><p>Достаточно много боилерплейта для такой простой задачи, не так ли?</p><figure class="full-width"><img alt="" data-src="https://habrastorage.org/getpro/habr/upload_files/51b/f16/12d/51bf1612d1f447cf93ddb5a9df035748.png" height="720" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/51b/f16/12d/51bf1612d1f447cf93ddb5a9df035748.png" title="" width="1280"/></figure><p>Давайте упростим!</p><h3>Используем Anvil</h3><p><a href="https://github.com/square/anvil" rel="noopener noreferrer nofollow">Anvil</a> это плагин компилятора Kotlin который помогает значительно уменьшить количество боилерплейта необходимого для использования Dagger 2 в вашем приложении.<br/>Также, будучи правильно сконфигурированным, позволяет увеличить скорость сборки проекта путём избавления от необходимости запуска процессора аннотаций Dagger 2 в ваших фича-модулях.<br/>Я не буду погружаться в подробности того что такое <a href="https://github.com/square/anvil" rel="noopener noreferrer nofollow">Anvil</a> и почему вам стоит подумать о том чтоб использовать его в вашем проекте. Возможно, напишу о нём статью попозже.<br/>Пока что, если интересует, можете прочитать документацию: <a href="https://github.com/square/anvil" rel="noopener noreferrer nofollow">https://github.com/square/anvil</a></p><p>Благодаря Anvil, мы можем избавиться от нашего <code>MyClassFactoryBindingModule</code> и использовать вместо него аннотацию <code>@ContributesBinding</code>:</p><pre><code class="kotlin">class DefaultMyClass @AssistedInject constructor(
    @Assisted param1: String,
    @Assisted param2: Int
) : MyClass {
	@ContributesBinding(AppGraph::class, MyClass.Factory::class)
	@AssistedFactory
	interface Factory: MyClass.Factory {
		override fun create(param1: String, param2: Int): DefaultMyClass
	}
}
</code></pre><p><a href="https://github.com/square/anvil" rel="noopener noreferrer nofollow">Anvil</a> сгенерирует для нас соответствующий модуль который создаст биндинг <code>DefaultMyClass.Factory</code> к <code>MyClass.Factory</code>.<br/>Этот подход уменьшает количество боилерплейта, но он всё ещё тут!<br/>Нам необходимо вручную следить за соответствием публичного интерфейса фабрики, интерфейса Assisted фабрики, и параметров конструктора.<br/>Может есть вариант как упростить это ещё сильнее?</p><h3>Встречайте @ContributesAssistedFactory !</h3><p>Благодаря моей библиотеке <a href="https://github.com/IlyaGulya/anvil-utils" rel="noopener noreferrer nofollow"><strong>Anvil Utils</strong></a>, вы можете попрощаться с необходимостью вручную создавать и поддерживать Assisted фабрики. Аннотация <code>@ContributesAssistedFactory</code> всё сделает за вас!</p><p>Она автоматически сгенерирует assisted фабрику и забиндит её в соответствующий скоуп при помощи <code>@ContributesBinding</code>.</p><p>Теперь мы можем полностью удалить интерфейс assisted фабрики и аннотировать наш <code>DefaultMyClass</code> аннотацией <code>@ContributesAssistedFactory</code>:</p><pre><code class="kotlin">@ContributesAssistedFactory(AppGraph::class, MyClass.Factory::class)
class DefaultMyClass @AssistedInject constructor(
    @Assisted param1: String,
    @Assisted param2: Int
) : MyClass
</code></pre><p>И всё! <a href="https://github.com/IlyaGulya/anvil-utils" rel="noopener noreferrer nofollow"><strong>Anvil Utils</strong></a> сгенерирует всё что нужно сама.</p><h3>Пример из реального проекта</h3><p>Для того чтобы продемонстрировать что польза не иллюзорна, я создал пулл реквест в прекрасный open source проект - приложение-компаньон для Flipper - тамагочи для хакеров.<br/>Можете ознакомиться с ним по этой ссылке, если интересно: <a href="https://github.com/flipperdevices/Flipper-Android-App/pull/798" rel="noopener noreferrer nofollow">https://github.com/flipperdevices/Flipper-Android-App/pull/798</a></p><figure class=""><img data-src="https://habrastorage.org/getpro/habr/upload_files/e8b/a48/276/e8ba4827677abfd000f553f0c2d80f57.png" height="84" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/e8b/a48/276/e8ba4827677abfd000f553f0c2d80f57.png" width="272"/></figure><h3>В заключение</h3><p>Если вы используете в своём проекте Anvil и у вас есть идеи что ещё можно упростить при помощи кодогенерации - открывайте пулл реквесты, возможно я реализую 🙂</p></div></div></div><!-- --><!-- --></div><!-- --><!-- --></div><!--]--><!--
