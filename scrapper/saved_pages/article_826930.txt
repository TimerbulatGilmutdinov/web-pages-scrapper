lang="ru"><div><!--[--><!--]--></div><div id="post-content-body"><div><div class="article-formatted-body article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p>Spring разработчики часто пишут документацию и README/HELP файлы к своим приложениям. Часто это делается в Markdown файлах. <a href="https://plugins.jetbrains.com/plugin/7793-markdown">Markdown</a> плагин в IntelliJ IDEA распознает файлы, предоставляет специальный редактор с подсветкой, автокомплитом и форматированием, а также дает визуальное отображение в панели предварительного просмотра в реальном времени. Кроме стандартной функциональности в IntelliJ IDEA есть пару уникальных фич, которые команда <a href="https://t.me/+QmrKn0wA8CdkNjAy">Spring АйО</a> хотела бы выделить особенно.</p><h2>Отображение Mermaid и PlantUML диаграмм </h2><p>Что насчет отображения <a href="https://mermaid.js.org/">Mermaid</a> и <a href="https://plantuml.com/">PlantUML</a> диаграмм? Для поддержки их отображения потребуется установить соответствующие плагины. Достаточно объявить блок кода в markdown файле с указанием <code>plantuml</code> или <code>mermaid</code> языка, и во вкладке с визуальным отображением вы увидите полноценную диаграмму.</p><figure class="full-width"><img alt="Отображение PlantUML и Mermaid диаграмм в markdown файле" data-src="https://habrastorage.org/getpro/habr/upload_files/3a5/2c8/575/3a52c85759af7238a08049122c14f761.png" height="618" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/3a5/2c8/575/3a52c85759af7238a08049122c14f761.png" title="Отображение PlantUML и Mermaid диаграмм в markdown файле" width="831"/><div><figcaption>Отображение PlantUML и Mermaid диаграмм в markdown файле</figcaption></div></figure><p><br/>Кстати, отображение Mermaid диаграмм поддерживает еще и <a href="https://github.blog/2022-02-14-include-diagrams-markdown-files-mermaid/">GitHub markdown preview</a>. К сожалению, поддержки для PlantUML в GitHub нет.</p><h2>Выполнение команд терминала</h2><p>При клонировании проекта обычно имеется файл README.md с инструкциями и командами для запуска приложения, настройки среды и т. д. IntelliJ IDEA обнаруживает эти команды и предоставляет значки для их запуска. Следующие блоки кода будут выполняемыми в IDEA:<br/></p><figure class="full-width"><img alt="Выполняемыми команды из markdown файла" data-src="https://habrastorage.org/getpro/habr/upload_files/42a/c22/d70/42ac22d70fece29156431017afa469aa.png" height="429" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/42a/c22/d70/42ac22d70fece29156431017afa469aa.png" title="Выполняемыми команды из markdown файла" width="907"/><div><figcaption>Выполняемыми команды из markdown файла</figcaption></div></figure><p>Поскольку IDEA Ultimate с поддержкой <a href="https://www.jetbrains.com/help/idea/http-client-in-product-code-editor.html">http client</a> становится все дальше и дальше от российских разработчиков (<a href="https://habr.com/ru/news/825756/">раз</a>, <a href="https://habr.com/ru/news/655531/">два</a> и <a href="https://habr.com/ru/news/763676/">три</a>), отдельно стоит рассмотреть выполнение cURL команд через markdown файл. Также, как и в http файлах вы можете описать GET, POST, DELETE и другие HTTP запросы и выполнять их из markdown файла для проверки работоспособности ваших эндпоинтов.</p><figure class="full-width"><img alt="Выполнение cURL команд из markdown файла" data-src="https://habrastorage.org/getpro/habr/upload_files/063/598/26a/06359826a03f59e978cb0683b2a1f20d.png" height="567" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/063/598/26a/06359826a03f59e978cb0683b2a1f20d.png" title="Выполнение cURL команд из markdown файла" width="905"/><div><figcaption>Выполнение cURL команд из markdown файла</figcaption></div></figure><h2>Не только bash скрипты</h2><p>Но если функциональности bash не достаточно (или у вас аллергия на bash), то можно воспользоваться полноценным языком таким как kotlin, java, groovy, и т.д. Достаточно из markdown вызвать заранее заготовленные скрипты на kotlin/groovy scripts или <a href="https://www.jbang.dev/">JBANG</a> (если кто-то помнит что это такое).</p><p>Давайте опять попробуем импортозаместить IntelliJ http client через один файл <a href="https://kotlinlang.org/docs/custom-script-deps-tutorial.html">kotlin script</a>.</p><ol><li><p>Создаем <code>.kts</code> файл. Тут есть интересная особенность. Чтобы заработал запуск файла и поддерживались все автодополнения, добавления зависимостей и т.д., файл должен оканчиваться на <code>.main.kts</code>.  </p></li><li><p>Добавим в наш скрипт библиотеку <a href="https://rest-assured.io/">RestAssured</a>, через специальную конструкцию:  </p><pre><code class="kotlin">@file:DependsOn("io.rest-assured:rest-assured:5.4.0")</code></pre></li><li><p>Напишем простенький тест с проверкой и выводом результата:</p><pre><code class="kotlin">import io.restassured.RestAssured.*
import org.hamcrest.Matchers.*

baseURI = "http://localhost:8089/rest"

get("/owners?city=Madison&amp;sort=firstName")
  .then().log().body(true)
  .statusCode(200)
  .body("content[0].firstName", equalTo("David"))
  .body("content[1].firstName", equalTo("Maria"))
  .body("content[2].firstName", equalTo("Peter"))</code></pre><p>Мы уже можем запустить этот скрипт вызвав команду <code>Run .main.kts</code> из контекстного меню этого файла. </p></li><li><p>Добавим вызов этого скрипта через утилиту <a href="https://kotlinlang.org/docs/command-line.html#run-scripts">kotlinc</a> в markdown файл:</p><pre><code class="bash">kotlinc -script ./test/rest.main.kts</code></pre><p>Преимущества такого подхода перед http client очевидны, у нас есть все возможности языка и настоящий фреймворк для тестирования. </p><figure class="full-width"><img alt="Запуск kotlin script из markdown файла" data-src="https://habrastorage.org/getpro/habr/upload_files/c5c/110/4e1/c5c1104e1aa72ece8a31556e1f1fd791.png" height="844" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/c5c/110/4e1/c5c1104e1aa72ece8a31556e1f1fd791.png" title="Запуск kotlin script из markdown файла" width="1143"/><div><figcaption>Запуск kotlin script из markdown файла</figcaption></div></figure></li></ol><p> <br/><br/>Про другие возможности редактирования и просмотра markdown файлов вы можете прочитать в <a href="https://www.jetbrains.com/help/idea/markdown.html">документации</a>.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/94b/b45/c0f/94bb45c0f97b5a13c24ec2dfd6ae7f7d.png" height="412" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/94b/b45/c0f/94bb45c0f97b5a13c24ec2dfd6ae7f7d.png" width="1920"/></figure><p>Присоединяйтесь к русскоязычному сообществу разработчиков на Spring Boot в телеграм - <a href="https://t.me/+oBwACbkvJ19iODAy"><u>Spring АйО</u></a>, чтобы быть в курсе последних новостей из мира разработки на Spring Boot и всего, что с ним связано.</p><p>Ждем
