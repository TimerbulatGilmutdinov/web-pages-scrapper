article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/a7c/f97/631/a7cf97631442c4a297c290c9eb91d149.png" height="440" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/a7c/f97/631/a7cf97631442c4a297c290c9eb91d149.png" width="780"/></figure><p>Всем привет! На связи Дима Котиков, и это заключительная статья цикла про File Templates. В предыдущих частях мы <a href="https://habr.com/ru/companies/tbank/articles/872684/">познакомились с инструментом File Templates, разобрали синтаксис написания шаблонов</a> и <a href="https://habr.com/ru/companies/tbank/articles/873234/">написали шаблоны для кода типового экрана SomeFeature мобильного приложения</a>. </p><p>В заключительной части разберем, как можно экспортировать и импортировать File Templates в Intellij-IDE, а также подведем общие итоги по циклу статей.</p><h2>Экспорт и импорт File Templates отдельно от остальных настроек</h2><p>В средах разработки Intellij можно экспортировать и импортировать настройки, в том числе и шаблоны. Это может быть полезно при смене ПК или при необходимости поделиться с кем-нибудь шаблонами.</p><p>Чтобы экспортировать шаблоны, нужно нажать в верхнем меню <code>File → Manage IDE Settings → Export Settings.</code></p><figure class="full-width"><img alt="Экспортирование настроек" data-src="https://habrastorage.org/getpro/habr/upload_files/a27/5f1/0aa/a275f10aa2e2111c8b0688277984e88a.png" height="1199" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/a27/5f1/0aa/a275f10aa2e2111c8b0688277984e88a.png" title="Экспортирование настроек" width="1387"/><div><figcaption>Экспортирование настроек</figcaption></div></figure><p>В диалоговом окне выбираем пункты с шаблонами, путь сохранения и нажимаем OK:</p><figure class="full-width"><img alt="Диалоговое окно экспорта настроек" data-src="https://habrastorage.org/getpro/habr/upload_files/06f/420/19a/06f42019a215239d7b2109a2a145ffa2.png" height="956" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/06f/420/19a/06f42019a215239d7b2109a2a145ffa2.png" title="Диалоговое окно экспорта настроек" width="1420"/><div><figcaption>Диалоговое окно экспорта настроек</figcaption></div></figure><p>Важный момент: в выходной архив будут экспортированы <strong>все пользовательские шаблоны</strong>, поэтому, если нужно поделиться только какими-то специфичными шаблонами, нужно вручную удалить из архива те, которые не нужны. </p><p>Для этого распаковываем архив, заходим в папку <code>fileTemplates</code> и удаляем то, что не нужно:</p><figure class="full-width"><img alt="Содержимое папки fileTemplates" data-src="https://habrastorage.org/getpro/habr/upload_files/387/046/601/3870466010b7967ef4b10d855529c6a6.png" height="854" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/387/046/601/3870466010b7967ef4b10d855529c6a6.png" title="Содержимое папки fileTemplates" width="1361"/><div><figcaption>Содержимое папки <code>fileTemplates</code></figcaption></div></figure><p>В папке <code>includes</code> найдем обобщенные шаблоны, а ненужные можем удалить. Но на этом еще не все: нужно вернуться в корневую папку с шаблонами, перейти в папку <code>options</code>, в которой будет находиться файл <code>file.template.settings.xml</code>, открыть его:</p><figure class="full-width"><img alt="Содержимое файла file.template.settings.xml" data-src="https://habrastorage.org/getpro/habr/upload_files/3eb/e96/939/3ebe969392aaa3d4cc041d0650bcf4fb.png" height="346" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/3eb/e96/939/3ebe969392aaa3d4cc041d0650bcf4fb.png" title="Содержимое файла file.template.settings.xml" width="1182"/><div><figcaption>Содержимое файла <code>file.template.settings.xml</code></figcaption></div></figure><p>В этом файле в тегах <code>&lt;template ... /&gt;</code> в поле <code>name</code> ищем названия ненужных шаблонов и удаляем вместе с блоками <code>&lt;template ... /&gt;</code>, после чего сохраняем изменения в файле.</p><p>В корневой папке есть файл <code>installed.txt</code>, в котором может быть информация об установленных плагинах в вашей студии разработки. Там можно подчистить те, которые не хотите добавлять в выходной архив с настройками.</p><p>Далее нужно запаковать файлы обратно в архив. Если у вас Mac OS, стандартным архиватором это лучше не делать, так как архив будет недоступен для импорта. Для архивации я использовал бесплатную программу <a href="https://apps.apple.com/us/app/unzip-rar-zip-7z-unarchiver"><u>unzip - rar zip 7z unarchiver:</u></a></p><figure class="full-width"><img alt="Упаковка откорректированных настроек с шаблонами в архив" data-src="https://habrastorage.org/getpro/habr/upload_files/cad/ddd/1bb/cadddd1bb6430eb9d7373bc9315cc4b1.png" height="848" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/cad/ddd/1bb/cadddd1bb6430eb9d7373bc9315cc4b1.png" title="Упаковка откорректированных настроек с шаблонами в архив" width="1600"/><div><figcaption>Упаковка откорректированных настроек с шаблонами в архив</figcaption></div></figure><p>Полученный архив готов к передаче коллегам и импорту в среды разработки.</p><p>Чтобы импортировать шаблоны из архива, необходимо в среде разработки нажать в верхнем меню <code>File → Manage IDE Settings → Import Settings</code>:</p><figure class="full-width"><img alt="Импортирование настроек" data-src="https://habrastorage.org/getpro/habr/upload_files/674/89d/f41/67489df417060c458918f61385236d3f.png" height="1204" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/674/89d/f41/67489df417060c458918f61385236d3f.png" title="Импортирование настроек" width="1272"/><div><figcaption>Импортирование настроек</figcaption></div></figure><p>Выбрать архив с шаблоном и в появившемся диалоговом окне нажать <code>OK</code>:</p><figure class="full-width"><img alt="Диалоговое окно подтверждения импорта" data-src="https://habrastorage.org/getpro/habr/upload_files/e52/679/180/e526791808e5e3e40ed7a48384729b2c.png" height="510" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/e52/679/180/e526791808e5e3e40ed7a48384729b2c.png" title="Диалоговое окно подтверждения импорта" width="992"/><div><figcaption>Диалоговое окно подтверждения импорта</figcaption></div></figure><p>После выполнения импорта будет запрошен перезапуск среды разработки для применения настроек и после перезапуска можно будет использовать импортированные шаблоны.</p><h2>Плюсы и минусы инструмента, выводы</h2><p>Мы изучили механизм генерации bolierplate-кода с помощью инструмента File Templates.</p><p><strong>Плюсы:</strong></p><ul><li><p>Инструмент встроен в среду разработки — его не нужно скачивать и настраивать, как этого требуют сторонние плагины.</p></li><li><p>Можно гибко настраивать генерацию как одного файла, так и нескольких.</p></li><li><p>Можно настроить генерацию файлов в папках и подпапках.</p></li><li><p>Можно задать пользовательские переменные, по которым будет происходить генерация шаблонов.</p></li><li><p>Можно выделить обобщенные шаблоны для использования в других шаблонах.</p></li><li><p>Есть возможность экспорта и импорта шаблонов в среде разработки.</p></li></ul><p><strong>Минусы:</strong></p><ul><li><p>Необходимо хотя бы минимальное знание языка шаблонов <a href="https://velocity.apache.org/">Apache Velocity.</a> Однако на изучение базовых конструкций не требуется много времени — для простых шаблонов достаточно понимания конструкций <code>#if</code>, <code>#set</code> и знания базовых переменных шаблонов по типу <code>$NAME</code> и <code>$PACKAGE_NAME</code>.</p></li><li><p>Нет возможности генерировать код в уже созданных файлах. Но инструмент и не призван это делать, для этих целей существуют <a href="https://www.jetbrains.com/help/idea/creating-and-editing-live-templates.html">Live Templates</a>, также доступные из коробки в среде разработки.</p></li></ul><p>File Templates — мощный инструмент, позволяющий сильно сократить время на генерацию файлов с шаблонным кодом. Он доступен из коробки и не требует лишних действий перед использованием, достаточно прост в освоении, а один раз написанные шаблоны могут использоваться в больших командах, что в целом позволит
