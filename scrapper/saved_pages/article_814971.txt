xmlns="http://www.w3.org/1999/xhtml"><p>Привет, Хабр! Когда‑то давно я наткнулся на ролик, где парень писал консольный клиент для YouTube. Идея была просто потрясающей! Написать свой минималистичный и быстрый клиент, который не будет надоедать лентами рекомендаций, рекламой и т. д. </p><p>Но вот беда, автор, не найдя возможности использовать YouTube API, решил спарсить одну из фронтенд обёрток над YouTube. </p><p>Стоит ли говорить о том, что такое решение сильно теряет в производительности и абсолютно нежизнеспособно на какой‑либо длительный период времени?</p><p>И вот относительно недавно я решил написать свой <a href="https://apps.rustore.ru/app/ru.zhirnov.youtubeinbackground" rel="noopener noreferrer nofollow">минималистичный ютуб клиент на базе Android приложения</a>. </p><p>Часто бывает, что во время прогулок по городу хочется послушать ютуб в фоновом или в PiP режиме, не отвлекаясь от общения с друзьями в мессенджере. <br/>Теперь это возможно!</p><figure class="full-width"><img alt="YouTube в фоне доступен в магазине приложений RuStore" data-src="https://habrastorage.org/getpro/habr/upload_files/f27/02b/f45/f2702bf456d6b957b8d7ebc40d117492.png" height="900" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/f27/02b/f45/f2702bf456d6b957b8d7ebc40d117492.png" title="YouTube в фоне доступен в магазине приложений RuStore" width="1600"/><div><figcaption>YouTube в фоне доступен в магазине приложений RuStore</figcaption></div></figure><p>В этой статье я хочу поделиться как бесплатно работать с YouTube API.</p><p>Как оказалось, для этого существует невероятно простая Python библиотека <a href="https://github.com/tombulled/innertube" rel="noopener noreferrer nofollow">InnerTube</a>:</p><blockquote><p>Библиотека обрабатывает низкоуровневое взаимодействие с базовым API InnerTube, используемым каждой службой YouTube.</p><p>Что-то типа брутфорса*</p></blockquote><p>Библиотека используется в основе известной утилиты <strong>yt-dlp</strong></p><p>Приложение ViMusic также использует эту библиотеку и уже более двух лет работает без обновлений. Так что вcё надежно, как швейцарские часы!<br/><br/>Простой пример использования InnerTube. При выводе мы получим данные в формате JSON, которые в дальнейшем можно обработать по вашему усмотрению.</p><pre><code class="python">import innertube

client = innertube.InnerTube("WEB")

# Поиск списка видео по запросу
client.search(query="уроки программирования на python")

# Получить данные для воспроизведения видео Rick Astley - Never Gonna Give You Up
client.player(video_id="dQw4w9WgXcQ")</code></pre><p>Тащить целиком эту библиотеку в наше Android приложение совсем необязательно.<br/><br/>Достаточно будет понять, какие параметры и тело запроса нам нужно передавать. <br/>Если порыться в коде проекта InnerTube, то можно найти файл <a href="http://config.py" rel="noopener noreferrer nofollow">config.py</a> с параметрами для создания запроса к WEB клиенту.</p><p>К слову, существует множество вариантов клиентов, таких как TVLITE, ANDROID, IOS и др. <br/>У всех них будут разные версии, и где-то могут отличаться возвращаемые данные.</p><pre><code class="python">REFERER_YOUTUBE: str = "https://www.youtube.com/"
USER_AGENT_WEB: str = (
    "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/74.0.3729.157 Safari/537.36"
)

config: Config = Config(
    base_url="https://youtubei.googleapis.com/youtubei/v1/",
    clients=[
        ClientContext(
            client_id=1,
            client_name="WEB",
            client_version="2.20230728.00.00",
            user_agent=USER_AGENT_WEB,
            referer=REFERER_YOUTUBE,
            api_key="AIzaSyAO_FJ2SlqU8Q4STEHLGCilw_Y9_11qcW8",
        )
    ],
)</code></pre><p>Ну а теперь переходим к запросам из Android приложения. Стандартом для работы с сетью, естественно, является Retrofit. </p><p>Самым сложным для меня оказалось конвертировать JSON объект с 40 тысячами строк ответа в дата классы.</p><pre><code class="kotlin">interface InnerTubeService {
    @POST("v1/player")
    suspend fun getPlayerInfo(@Body body: PlayerBody): Response&lt;PlayerResponse&gt;

    @POST("v1/search?key=AIzaSyAO_FJ2SlqU8Q4STEHLGCilw_Y9_11qcW8")
    suspend fun searchVideos(@Body body: SearchBody): Response&lt;SearchResponse&gt;
}

// Body для поиска видео по названию
val searchBody = SearchBody(
                contentCheckOk = true, context = Context(
                    client = Client(
                        androidSdkVersion = 31,
                        clientName = "WEB",
                        clientScreen = "WATCH",
                        clientVersion = "2.20230728.00.00",
                        gl = "US",
                        hl = "en"
                    ), thirdParty = ThirdParty(embedUrl = "https://www.youtube.com/")
                ), playbackContext = PlaybackContext(
                    contentPlaybackContext = ContentPlaybackContext(
                        signatureTimestamp = 19250
                    )
                ), racyCheckOk = true, query = query
            )

//Body для получения данных проигрывателя
val playerBody = PlayerBody(
                contentCheckOk = true, context = Context(
                    client = Client(
                        androidSdkVersion = 31,
                        clientName = "TVLITE",
                        clientScreen = "WATCH",
                        clientVersion = "2",
                        gl = "US",
                        hl = "en"
                    ), thirdParty = ThirdParty(embedUrl = "https://www.youtube.com/")
                ), playbackContext = PlaybackContext(
                    contentPlaybackContext = ContentPlaybackContext(
                        signatureTimestamp = 19250
                    )
                ), racyCheckOk = true, videoId = videoId
            )</code></pre><p>Вот так выглядит ответ с потоковой ссылкой для просмотра.</p><pre><code class="kotlin">data class PlayerResponse(val streamingData : StreamingData,
                          val videoDetails : VideoDetails)

data class StreamingData(val formats : List&lt;Formats&gt;)

data class Formats(val itag : Long, val url : String)</code></pre><h2>Итог</h2><p>Всё работает достаточно шустро и напрямую с YouTube API.</p><p>Я уже реализовал историю просмотра видео.
