xmlns="http://www.w3.org/1999/xhtml"><p>Всем привет! Как следует из названия статьи, речь пойдет о HTTP-клиентах для IntelliJ IDEA. Да, опять). В последнее время было несколько публикаций на эту тему, и мы бы хотели подробно рассказать о нашем взгляде на эту проблему и нашей реализации. А также поговорить о плюсах и минусах текущих решений для IDEA. Ранее мы уже рассказывали о <a href="https://habr.com/ru/companies/explyt/articles/854304/">нашем</a> плагине для Spring - о том как у нас реализована поддержка Dependency Injection в частности, теперь настала очередь HTTP client'а.</p><h3>Проблема</h3><p>Как известно, в IDEA Ultimate "был" отдельный плагин HTTP Client, который предоставлял кастомный <a href="https://ru.wikipedia.org/wiki/%D0%9F%D1%80%D0%B5%D0%B4%D0%BC%D0%B5%D1%82%D0%BD%D0%BE-%D0%BE%D1%80%D0%B8%D0%B5%D0%BD%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D1%8B%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA">DSL</a> (Domain Specific Language) для написания HTTP запросов и имел свой механизм для их исполнения. В настоящее время также стали появляться похожие инструменты от других разработчиков плагинов для IDEA. Почти все они так или иначе основаны на подобном подходе – создании кастомного DSL. Рассмотрим более подробно плюсы и минусы такого подхода. Как и в любом решении, здесь есть компромиссы: то, что даёт нам гибкость, одновременно требует дополнительных усилий.</p><p>Плюсы:</p><ul><li><p>у нас есть полный контроль над выполнением HTTP-запросов;</p></li><li><p>соответственно, мы можем реализовывать практически любые фичи, как на уровне DSL, так и на уровне интеграции с IDE;</p></li><li><p>теоретически кастомный DSL можно рассматривать отдельно от IDEA и использовать в девопс практиках и в тестировании, как это произошло, например, с DSL от IDEA Ultimate.</p></li></ul><p>Минусы:</p><ul><li><p>реализовывать свой DSL – это достаточно трудоёмкое занятие;</p></li><li><p>пользователю нужно во все это погружаться – учить новый “мини” язык и разбираться, как это работает;</p></li><li><p>также нужно реализовывать поддержку в IDE, как на уровне редактора, так и на уровне работы с выходными данными запроса - UI.</p></li></ul><h3>Наш взгляд на эту проблему</h3><p>В нашем плагине мы уже умели генерировать OpenApi файлы для Spring Controllers (кнопка Run напротив имени класса) и далее открывать Swagger UI в IDEA для созданных файлов.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/076/c67/8d1/076c678d16e3e529dbdd5cb81f93c7ea.png" height="323" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/076/c67/8d1/076c678d16e3e529dbdd5cb81f93c7ea.png" width="990"/></figure><figure class="full-width"><img alt="Информация об этом есть на Wiki Explyt Spring" data-src="https://habrastorage.org/getpro/habr/upload_files/d1e/b53/013/d1eb53013fc536aacdec1fbcbf6df7ae.png" height="791" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/d1e/b53/013/d1eb53013fc536aacdec1fbcbf6df7ae.png" title="Информация об этом есть на Wiki Explyt Spring" width="1600"/><div><figcaption>Информация об этом есть на <a href="https://github.com/explyt/spring-plugin/wiki/Screenshots">Wiki</a> Explyt Spring</figcaption></div></figure><p>Созданные файлы в таком виде могли использоваться для получения OpenApi документации по проекту прямо из IDEA без запуска проекта и для возможности протестировать приложение локально через сваггер. Но для последнего пункта, нужно чтобы приложение было запущено локально, а эту проблему можно решить, просто добавив в зависимости проекта спринг-стартер: <abbr class="habraabbr" data-abbr="springdoc-openapi-starter-webmvc-ui" data-title="&lt;p&gt;&lt;/p&gt;" title="">springdoc-openapi-starter-webmvc-ui</abbr> и в целом для тестирования текущего приложения это было не очень полезно.</p><p>Но нам пришла в голову идея - а почему бы не сделать из этого механизма свой HTTP клиент на минималках, который можно применять для вызова внешних сервисов, используя OpenApi, Swagger UI и аннотации Spring Web в качестве DSL для написания запросов. Это может быть полезно, когда есть задача по интеграции с каким нибудь сторонним HTTP сервисом. И хочется сначала “пощупать” его на примерах, сделав тестовые обращения к нему, чтобы понять как оно работает и какой результат возвращает.</p><p>Более подробно по шагам это процесс можно представить вот так:</p><ul><li><p>с помощью хорошо знакомых Spring Web аннотаций пользователь на любом удобном ему языке  Java\Kotlin описывает REST метод;</p></li><li><p>генерируем OpenApi схему, на основании написанного метода;</p></li><li><p>запускаем Swagger UI прямо в IDEA по файлу из предыдущего пункта;</p></li><li><p>и далее пользователь в уже хорошо знакомом ему инструменте, с готовым UI, сможет выполнять HTTP запросы.</p></li></ul><p>Плюсы:</p><ul><li><p>не надо делать свой UI -  используем Swagger UI;</p></li><li><p>не надо делать свой DSL - используем Spring Web Annotations &amp; OpenApi;</p></li><li><p>используем готовые проверенные решения, которые знают большинство разработчиков; </p></li><li><p>относительная простая реализация.</p></li></ul><p>Минусы:</p><ul><li><p>решение менее гибкое и менее расширяемое, т.к используем уже готовые компоненты. Соответственно имеем только то что имеем и кастомизировать это очень трудно.</p></li><li><p>узкоспециализированное решение для Spring (требует Spring Web в зависимостях, но в планах есть добавить поддержку JAX-RS, чтобы быть более платформо-независимыми)</p></li><li><p>нельзя строить сложные сценарии, состоящие из последовательности  запросов, когда один вызов, является отправной точкой для другого;</p></li><li><p>используемые решения тоже не лишены недостатков и имеют свои проблемы (например CORS в Swagger UI). </p></li></ul><h3>Как это работает</h3><p>Чтобы лучше понять, что получилось и как этим пользоваться, рассмотрим это на примере интеграции с одним из сервисов погоды - <a href="https://api.openweathermap.org">https://api.openweathermap.org</a></p><p>Допустим мы хотим проверить что возвращает нам сервис -<a href="https://api.openweathermap.org/data/2.5/weather?lat=55&amp;lon=44&amp;appid=1&amp;mode=json">https://api.openweathermap.org/data/2.5/weather</a>.</p><p>Для этого просто возьмем и создадим метод на основе Spring аннотаций. Заметим что реализация этого метода нам не важна, как в целом и возвращаемый тип. Т.к. HTTP это текстовый протокол, то в качестве выходного типа можно указать строку или вообще ничего не указывать и оставить void. Swagger UI сам, на основании заголовков ответа, сможет понять, как отобразить результат запроса, в самом общем случае это просто текст.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/25a/f8a/d99/25af8ad9967e13d2ba2939762858d712.png" height="217" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/25a/f8a/d99/25af8ad9967e13d2ba2939762858d712.png" width="1018"/></figure><p>Метод можно создать абсолютно в любом классе, наш плагин умеет распознавать методы, помеченные Spring Web аннотациями и если там обнаруживаем абсолютный урл, то это значит что мы можем попытаться выполнить такой метод сразу и напротив его имени появляется соответствующий маркер.</p><p>Если нажать на него, то откроется хорошо знакомый всем Swagger UI:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/a4c/e96/7b9/a4ce967b91fe883f2ef7e7a30e3c339f.png" height="506" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/a4c/e96/7b9/a4ce967b91fe883f2ef7e7a30e3c339f.png" width="1366"/></figure><p>Где мы через интерфейс Swagger UI сможем выполнить запрос:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/e22/aec/103/e22aec103f43fe93fd013acf13808491.png" height="1238" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/e22/aec/103/e22aec103f43fe93fd013acf13808491.png" width="1327"/></figure><p>Также тут мы сразу получаем запрос в curl формате и можем это использовать, чтобы выполнить его через командную строку или как-то его модифицировать, сохранив себе в блокнотик. После выполнения запроса, получаем результат:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/399/ad4/520/399ad452026bd0187809e3d848f9f2fc.png" height="639" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/399/ad4/520/399ad452026bd0187809e3d848f9f2fc.png" width="1307"/></figure><p>Т.к. наши генераторы OpenApi файлов по коду, могут отличатся от оригинальных и давать другой результат, то вы всегда можете отредактировать OpenApi вручную, если чувствуете в себе силы или завести <a href="https://github.com/explyt/spring-plugin/issues">баг</a>. Чтобы перейти в режим редактирования, можно нажать “переключалку” вверху справа в окне Swagger. Кстати для OpenApi файлов, мы поддерживаем различные подсказки и авто-дополнения, так что редактирование OpenApi - это на самом деле не так страшно, как может показаться). </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/fbc/64b/002/fbc64b0022d0195741b5e43f6e882f5c.png" height="265" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/fbc/64b/002/fbc64b0022d0195741b5e43f6e882f5c.png" width="1308"/></figure><p>Также если у нас несколько сервисов, имеют общий URL, то такие методы можно сгруппировать в классы:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/4e0/86f/990/4e086f99036ce79eaeb795905806894e.png" height="246" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/4e0/86f/990/4e086f99036ce79eaeb795905806894e.png" width="892"/></figure><p>Если мы используем Kotlin, то можем создавать тестовые HTTP методы прямо в Kotlin файле, минуя создание лишних классов, в результате это будет выглядеть почти как Ultimate HTTP Client):</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/6eb/56d/1ee/6eb56d1eec1e334fcb92b9f3519131aa.png" height="478" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/6eb/56d/1ee/6eb56d1eec1e334fcb92b9f3519131aa.png" width="1007"/></figure><p>Если вы используете Feign для создания клиента, то в таком случае, можно сразу попробовать вызвать удаленный сервис:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/b88/220/e6c/b88220e6cd424b9933b0069c5f8107fe.png" height="158" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/b88/220/e6c/b88220e6cd424b9933b0069c5f8107fe.png" width="824"/></figure><p>Для обычных Spring контроллеров, мы генерируем OpenApi файл с тегом <abbr class="habraabbr" data-abbr="servers: localhost:8080" data-title="&lt;p&gt;&lt;/p&gt;" title="">servers: localhost:8080</abbr>, чтобы  пользователь смог протестировать свое приложение локально, хотя это не совсем актуальный кейс как мы писали в начале статьи, или сможет сам указать нужный host в теге server OpenApi файла, чтобы проверить как работает сервис например на тестовом контуре.</p><p>Благодаря тому что мы используем Swagger UI, нам не пришлось делать свой интерфейс для работы с результатами запросов. Вот так например, выглядит HTTP запрос который скачивает наш плагин с GitHub:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/ad3/6d7/bdf/ad36d7bdf05357b808fe7123d45aca13.png" height="85" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/ad3/6d7/bdf/ad36d7bdf05357b808fe7123d45aca13.png" width="1224"/></figure><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/910/455/549/9104555491d7d12cf8753ec49af72d77.png" height="669" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/910/455/549/9104555491d7d12cf8753ec49af72d77.png" width="1442"/></figure><p>Вот так выглядит результат, если пытаемся загрузить картинку:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/303/300/ea9/303300ea91e7ddcd3dd5be7765713dd4.png" height="559" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/303/300/ea9/303300ea91e7ddcd3dd5be7765713dd4.png" width="1425"/></figure><h3>Генераторы кода</h3><p>Чтобы меньше “набирать” код, мы сделали удобные генераторы, для создания Spring Web методов прямо из Url или Curl (тут опять может пригодиться Curl запрос, который генерирует Swagger). Данные функции доступны из меню Alt+Insert или (Command ⌘)+N на Mac, когда мы находимся внутри Java класса или Kotlin файла:</p><div class="floating-image"><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/ae0/883/be6/ae0883be65373c59ac0b4adf47810f48.png" height="372" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/ae0/883/be6/ae0883be65373c59ac0b4adf47810f48.png" width="1040"/></figure></div><p>Результат:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/7c2/3fa/c05/7c23fac059ca7b151454788be7f7a453.png" height="292" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/7c2/3fa/c05/7c23fac059ca7b151454788be7f7a453.png" width="1111"/></figure><p>Или вариант с Curl:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/d41/152/afe/d41152afe2f15e7b740ea0dcfe67efe9.png" height="269" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/d41/152/afe/d41152afe2f15e7b740ea0dcfe67efe9.png" width="536"/></figure><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/605/34f/0ce/60534f0ce08ac593059fb4a8aa521f61.png" height="338" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/605/34f/0ce/60534f0ce08ac593059fb4a8aa521f61.png" width="1223"/></figure><p>Многие пользователи для тестирования HTTP запросов предпочитают создавать обычные методы, где используя привычный им HTTP клиент, непосредственно в Java/Kotlin пишут код для вызова сервиса. Для таких случаев мы также поддержали конвертация из Url/Curl сразу в <abbr class="habraabbr" data-abbr="java.net.http.HttpClient" data-title="&lt;p&gt;&lt;/p&gt;" title="">java.net.http.HttpClient</abbr> в меню Alt+Insert это опция “HttpClient method”. Вызвав которую для примера выше, получим код:</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/372/8d8/e6f/3728d8e6f62a18393873fb91f2d2dd1c.png" height="317" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/372/8d8/e6f/3728d8e6f62a18393873fb91f2d2dd1c.png" width="1034"/></figure><p>По итогу у нас есть следующие конверторы (генераторы кода):</p><ul><li><p>из URL/CURL в Spring Web method</p></li><li><p>из URL/CURL в нативный Java клиент -  <abbr class="habraabbr" data-abbr="java.net.http.HttpClient" data-title="&lt;p&gt;&lt;/p&gt;" title="">java.net.http.HttpClient</abbr></p></li></ul><p><strong>Проблемы и детали реализации</strong></p><p>IntelliJ IDEA включает в себя <a href="https://plugins.jetbrains.com/docs/intellij/jcef.html?from=jetbrains.org">Embedded Browser</a> (Java Chromium Embedded Framework - JCEF). Поэтому с тем чтобы запустить Swagger UI прямо в IDE не возникло никаких проблем. Просто добавили Standalone Swagger UI в ресурсы плагина и используем его как превью-генератор для OpenApi файлов. Посмотреть детали нашей реализации можно в репозитории на GitHub - <a href="https://github.com/explyt/spring-plugin/blob/main/modules/spring-web/src/main/kotlin/com/explyt/spring/web/editor/openapi/OpenApiCefBrowser.kt">OpenApiCefBrowser</a>.</p><p>Но возникла другая проблема, связанная с тем, что мы выполняем веб запросы из браузера - это <a href="https://en.wikipedia.org/wiki/Cross-origin_resource_sharing">CORS</a>. Кстати, на эту тему есть хорошая статья на <a href="https://habr.com/ru/companies/otus/articles/706908/">Habr</a>. Чтобы обойти эту проблему, пришлось написать свой request handler. Для JCEF клиента мы можем задать свой обработчик запросов. Где делегируем исполнение запросов Java HttpClient, а результат отдаем в браузер. Детали реализации можно посмотреть опять же в нашем <a href="https://github.com/explyt/spring-plugin/blob/main/modules/spring-web/src/main/kotlin/com/explyt/spring/web/editor/openapi/OpenApiProxyRequestHandler.kt">репозитории</a>.</p><p>Также для загрузки файлов, необходимо было реализовать CefDownloadHandler - пример нашей реализации можно посмотреть <a href="https://github.com/explyt/spring-plugin/blob/main/modules/spring-web/src/main/kotlin/com/explyt/spring/web/editor/openapi/OpenApiCefDownloadHandler.kt">тут</a>.</p><div class="floating-image"><figure class="float full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/9a1/ce0/f9b/9a1ce0f9beaa6ccd133b5bf468aa650f.png" height="846" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/9a1/ce0/f9b/9a1ce0f9beaa6ccd133b5bf468aa650f.png" width="653"/></figure><p>Для более удобной работы с "эндпоинтами" мы доработали наше Explyt Endpoints Tool Window. Где постарались улучшить дизайн и отображаем все возможные URL'ы что нашли в проекте и предоставляем навигацию к ним, а также поиск и фильтрацию. Если там чего то не хватает на ваш взгляд, то можете смело заводить <a href="https://github.com/explyt/spring-plugin/issues">issue</a> на доработку, мы всегда рады обратной связи и предложениям.</p><p></p><p></p><p></p><p></p></div><div class="floating-image"><p></p><p><strong>Заключение</strong></p></div><p>Мы постарались упростить процесс тестирования Rest API, надеемся так оно и вышло. Также мы продолжаем работать над нашими генераторами схем OpenAPi по коду проекта, чтобы сделать их лучше. Скачать последнюю версию плагина с HTTP клиентом можно <a href="https://explyt.com/en-download/?utm_source=habr&amp;t.coutm_campaign=post_1">тут</a> или напрямую с <a href="https://github.com/explyt/spring-plugin/releases/">GitHub Releases</a>. HTTP клиент доступен, начиная с версии 2*.1.3155.</p><p>Приглашаем вас попробовать наш плагин, а также делиться своими отзывами и предложениями. Ваша обратная связь поможет нам сделать инструмент более полезным и удобным. Также напоминаем, что плагин имеет открытый исходный код, который доступен на <a href="https://github.com/explyt/spring-plugin">GitHub</a>. </p><p>Будем благодарны за ваши вопросы и идеи: <a href="https://github.com/explyt/spring-plugin/issues/new/choose">GitHub Issues</a>, <a href="https://t.me/explytspring">Telegram-чат</a>, а также личные сообщения в нашем профиле на Habr.</p><p></p></div></div></div><!-- --><!-- --></div><!-- --><!-- --></div><!--]--><!-- --><div class="tm-article-presenter__meta" data-test-id="article-meta-links"><div class="tm-separated-list
