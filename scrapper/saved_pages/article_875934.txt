xmlns="http://www.w3.org/1999/xhtml"><p>Новый гайд от команды <a href="https://amplicode.ru">Amplicode</a> придётся по душе тем, кто предпочитает разрабатывать схему базы данных перед реализацией JPA-модели или создаёт приложение на основе уже существующей БД.</p><p>За 15 минут вы узнаете, как можно легко преобразовать таблицы базы данных в JPA-сущности, учитывая ограничения, индексы, свойства столбцов, а также ассоциативные связи (1:1, 1:M, M:1, M:M).</p><p>Статья также доступна в формате видео, так что можно и смотреть, и читать — как вам удобнее!</p><div class="tm-iframe_temp" data-src="https://embedd.srv.habr.com/iframe/679b41c628ebd7e4cccd98ec" data-style="" id="679b41c628ebd7e4cccd98ec" width=""></div><p><a href="https://vkvideo.ru/video-222549074_456239052">Смотреть в VK</a> || <a href="https://youtu.be/uxdec6WIWEg?feature=shared">Смотреть на YouTube</a></p><hr/><details class="spoiler"><summary>Спойлер</summary><div class="spoiler__content"><p><em>В большинстве случаев я буду использовать изображения для демонстрации фрагментов кода. Такой подход позволит мне выделить важные части и более подробно их объяснить. Если вы хотите проверить всё самостоятельно, запустив код, о котором пойдет речь, то найти его можно на </em><a href="https://github.com/Amplicode/amplicode-tutorials/tree/main/generate-entities-from-db"><em>GitHub</em></a><em>.</em> </p></div></details><h3>Задачи на сегодня</h3><p>Сегодня мы будем заниматься разработкой приложения для небольшого блога. В нашей базе данных мы будем хранить пользователей, их посты, комментарии и отображать статистику: количество постов, созданных каждым пользователем.  </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/5da/fa3/76e/5dafa376e1164f8be4e1308751e01319.png" height="1096" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/5da/fa3/76e/5dafa376e1164f8be4e1308751e01319.png" width="1730"/></figure><p>Шаги, которые необходимо выполнить в рамках решения поставленной задачи, следующие:</p><ul><li><p>Инициализация схемы БД</p></li><li><p>Создание Spring Boot приложения</p></li><li><p>Настройка подключения к БД</p></li><li><p>Создание JPA сущностей из таблиц базы данных</p></li><li><p>Модификация JPA модели и создание <code>@MappedSuperclass</code></p></li><li><p>Cherry-pick изменений из БД в JPA модель</p></li><li><p>Создание JPA сущности для DB View</p></li></ul><h3>Инициализация схемы БД</h3><p>Для первой версии приложения понадобится создать две таблицы: <code>users</code> и <code>posts</code>. Для этого вам следует выполнить следующие скрипты через pgAdmin.</p><pre><code class="sql">CREATE TABLE users
(
    id                  BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL,
    created_by          VARCHAR(255),
    created_date        TIMESTAMP WITHOUT TIME ZONE,
    last_modified_by    VARCHAR(255),
    last_modified_date  TIMESTAMP WITHOUT TIME ZONE,
    first_name          VARCHAR(255),
    last_name           VARCHAR(255),
    CONSTRAINT pk_users PRIMARY KEY (id)
);

CREATE TABLE posts
(
    id                  BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL,
    created_by          VARCHAR(255),
    created_date        TIMESTAMP WITHOUT TIME ZONE,
    last_modified_by    VARCHAR(255),
    last_modified_date  TIMESTAMP WITHOUT TIME ZONE,
    title               VARCHAR(255),
    test                OID,
    publiched_at        TIMESTAMP WITHOUT TIME ZONE,
    author_id           BIGINT                                  NOT NULL,
    CONSTRAINT pk_posts PRIMARY KEY (id)
);</code></pre><p>Для улучшения производительности запросов к таблице <code>users</code> создайте составной индекс для столбцов имени и фамилии. </p><pre><code class="sql">CREATE INDEX idx_user_names ON "users" (first_name, last_name);</code></pre><p>Один пользователь может создать несколько постов, поэтому кардинальность отношения будет типа "многие к одному", реализованная через внешний ключ в таблице <code>posts</code>. Для ее реализации выполните следующий запрос:</p><pre><code class="sql">ALTER TABLE posts
    ADD CONSTRAINT FK_POSTS_ON_AUTHOR FOREIGN KEY (author_id) REFERENCES users (id);</code></pre><p>После выполнения всех скриптов проверьте актуальное состояние базы данных через pgAdmin, и далее можно будет приступать к разработке собственно Spring Boot приложения. </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/605/8c2/f5d/6058c2f5dcd895cea93273b2c9681389.png" height="1478" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/605/8c2/f5d/6058c2f5dcd895cea93273b2c9681389.png" width="654"/></figure><h3>Создание Spring Boot приложения</h3><p>Чтобы сгенерировать Spring Boot приложение, воспользуйтесь <a href="https://start.spring.io">Spring Initializr</a>. </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/c4f/a0d/609/c4fa0d6095665a66bc9b903b768eca46.png" height="919" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/c4f/a0d/609/c4fa0d6095665a66bc9b903b768eca46.png" width="1552"/></figure><p>На странице генерации приложения:</p><ul><li><p>Установите название группы и артефакты (1)</p></li><li><p>Нажмите на <strong>ADD DEPENDENCIES</strong> (2)</p></li><li><p>Добавьте зависимости для Spring Data JPA и PostgreSQL (3)</p></li><li><p>Все остальные настройки оставьте по умолчанию</p></li><li><p>Нажмите <strong>GENERATE</strong> (4)</p></li></ul><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/d71/a50/378/d71a50378c53f2df87361b9a1e9a50d6.png" height="919" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/d71/a50/378/d71a50378c53f2df87361b9a1e9a50d6.png" width="1552"/></figure><p>Теперь откройте сгенерированное приложение в IntelliJ IDEA.</p><h3>Настройка подключения к БД</h3><p>Теперь надо настроить источник данных для корректной работы со Spring Data JPA. С Amplicode настроить источник данных довольно легко: </p><ul><li><p>Нажмите правой кнопкой мыши по узлу <strong>Data Sources</strong> </p></li><li><p>Выберите опцию <strong>New</strong> -&gt; <strong>Default Data Source</strong> </p></li></ul><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/095/ecb/f58/095ecbf58b6d18ae79062b0e605376c7.png" height="370" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/095/ecb/f58/095ecbf58b6d18ae79062b0e605376c7.png" width="928"/></figure><ul><li><p>В открывшемся окне выберите тип базы данных (PostgreSQL)  (1)</p></li><li><p>Укажите необходимые параметры для подключения к источнику данных (2)</p></li><li><p>Благодаря действию <strong>Test Connection</strong> можно убедиться в корректности указанных параметров для источника данных (3)</p></li><li><p>Кроме того, имеет смысл сразу же создать подключение к базе данных: для этого отметим чекбокс <strong>Create DB Connection</strong>. (4)</p></li></ul><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/a47/313/71d/a4731371dc864d104efedf2a18429e25.png" height="1392" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/a47/313/71d/a4731371dc864d104efedf2a18429e25.png" width="1438"/></figure><blockquote><p>Подключение к базе данных необходимо для дальнейшей корректной работы таких возможностей Amplicode как создание сущности на основе таблиц в базе данных, а также генерация скриптов миграции.</p></blockquote><ul><li><p>В диалоге создания подключения оставьте те же значения, которые мы указали для источника данных. </p></li></ul><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/605/5a0/f67/6055a0f67355e8d77ab5d7357a29ed0a.png" height="1002" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/605/5a0/f67/6055a0f67355e8d77ab5d7357a29ed0a.png" width="1472"/></figure><ul><li><p>Нажмите <strong>OK</strong></p></li></ul><p>Все необходимые параметры для источника данных будут сгенерированы в <code>application.properties</code>:</p><pre><code>#Datasource configuration
spring.datasource.url=jdbc:postgresql://localhost/blog
spring.datasource.username=root
spring.datasource.password=root
spring.datasource.driver-class-name=org.postgresql.Driver</code></pre><p>Также будет создано подключение к нашей базе данных. Теперь можно приступать к следующему шагу — разработке JPA модели. </p><h3>Создание JPA сущностей из таблиц базы данных</h3><p>С помощью Amplicode легко справиться с задачей создания JPA модели, отталкиваясь от существующей базы данных. Для этого:</p><ul><li><p>В дереве проекта нажмите правую кнопку мыши на имени корневого пакета</p></li></ul><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/7a9/2c9/8fa/7a92c98fafdcde770d480a65bee629ac.png" height="562" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/7a9/2c9/8fa/7a92c98fafdcde770d480a65bee629ac.png" width="578"/></figure><ul><li><p>В появившемся всплывающем меню выберите действие         count(p.id)     AS number_of_posts
FROM users u
        JOIN posts p ON u.id = p.author_id
GROUP BY u.id;</code></pre><p>Воспользуйтесь уже знакомым нам диалогом по созданию сущности на основе базы данных и выберите новое представление. </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/683/52b/ec7/68352bec7d53eb026b98bbf99c83a094.png" height="803" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/683/52b/ec7/68352bec7d53eb026b98bbf99c83a094.png" width="1730"/></figure><p>У представлений в базе данных нет первичных ключей, но у сущности JPA должен быть идентификатор. Поэтому нам нужно выбрать поле или набор полей, которые могут выступать в качестве идентификатора сущности. Для этого нажмите на ссылку <strong>Select columns to be used as primary key</strong>. </p><p>Появится всплывающее окно, позволяющее выбрать одно или несколько полей для этой цели. Выберите идентификатор пользователя, так как в данном случае он выглядит как самый подходящий вариант. </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/aa9/310/b13/aa9310b13b60ac7d61dc2c6e71fd83f5.png" height="474" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/aa9/310/b13/aa9310b13b60ac7d61dc2c6e71fd83f5.png" width="1278"/></figure><p>Нажмите <strong>OK</strong> сначала в маленьком окошке, а затем и большом диалоговом окне.</p><p>Сущность готова. Так как сохранять данные в представлении напрямую нельзя, необходимо отметить нашу сущность аннотацией <code>@Immutable</code>. Amplicode также знает про это и сам добавит необходимую аннотацию. Кроме того, для гарантии невозможности создания новых экземпляров этой сущности Amplicode сгенерирует конструктор без параметров с модификатором <code>protected</code>.</p><p>Наконец, для всех полей будут сгенерированы только геттеры, чтобы не дать разработчикам возможность изменять значения атрибутов. В целом, сущность представления будет сгенерирована в соответствии со всеми Best Practices. Amplicode заботится о качестве кода за нас:</p><pre><code class="java">/**
 * Mapping for DB view
 */
@Entity
@Immutable
@Table(name = "user_statistics")
public class UserStatistic {
    @Id
    @Column(name = "user_id")
    private Long userId;

    @Column(name = "number_of_posts")
    private Long numberOfPosts;

    public Long getUserId() {
        return userId;
    }

    public Long getNumberOfPosts() {
        return numberOfPosts;
    }

    protected UserStatistic() {
    }
}</code></pre><h3>Заключение</h3><p>Подводя итог, все поставленные в данном гайде задачи успешно выполнены примерно за 15 минут.
