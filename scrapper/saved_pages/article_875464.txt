lang="ru"><div><!--[--><!--]--></div><div id="post-content-body"><div><div class="article-formatted-body article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p>Последние 10 лет я занимаюсь java разработкой и на протяжении всего этого времени Intellij Idea является неотъемлемой частью моей(да и многих других джавистов) работы.<br/> К сожалению, некоторых вещей, которые были бы удобны лично мне, в ней нет, но к счастью есть возможность расширять IDE с помощью плагинов. На моём ноутбуке установлен linux и нет какой-то удобной нотификации событий из корпоративного календаря, а IDE практически всегда открыта на главном мониторе. По этой причине(а ещё из-за внезапно появившегося окна свободного времени и простого интереса) я решил, почему бы не интегрировать календарь прямо в IDE, чтобы получать нотификации и точно не пропустить ничего важного?<br/> Об этом и пойдёт речь в статье. Сразу скажу, что я не обладаю каким-то богатыми знаниями в этой области и всё нижеизложенное является исключительно моим личным опытом.</p><p>На работе в качестве корпоративной почты используется решение от yandex(я не являюсь их сотрудником), соответственно и интегрироваться предстоит именно с yandex calendar, но поскольку он поддерживает протокол <a href="https://datatracker.ietf.org/doc/html/rfc4791" rel="noopener noreferrer nofollow">CALDAV(RFC4791)</a>, то интегрироваться можно с любым другим решением, поддерживающим данный протокол(google, outlook, mail.ru).<br/> Почитав [документацию](<a href="https://yandex.ru/support/yandex-360/business/admin/ru/security-service-applications" rel="noopener noreferrer nofollow">https://yandex.ru/support/yandex-360/business/admin/ru/security-service-applications</a>, <a href="https://360.yandex.ru/blog/articles/kak-sinhronizirovat-yandeks-kalendar-s-kalendaryom-na-android" rel="noopener noreferrer nofollow">https://360.yandex.ru/blog/articles/kak-sinhronizirovat-yandeks-kalendar-s-kalendaryom-na-android</a>) и немного поэкспериментировав с API, был найден рабочий вариант как получать события из календаря.<br/> Для этого необходимо зайти в свой аккаунт -&gt; Безопасность -&gt; <a href="https://id.yandex.ru/security/app-passwords" rel="noopener noreferrer nofollow">Пароли приложений</a> и создать пароль для календаря.<br/> Далее используя полученный пароль, можно выполнить HTTP запрос:<br/> <strong>GET</strong> <a href="https://caldav.yandex.ru/calendars/%24%7Bemail%7D/events-default" rel="noopener noreferrer nofollow">https://caldav.yandex.ru/calendars/${email}/events-default</a><br/> Authorization: Basic ${token}</p><p>где <strong>token</strong> - это <code>"{email}:{password}"</code> в base64 и получить список ссылок на все события из вашего календаря вида:</p><p>/calendars/{email}/events-default/{uniqString}yandex.ru.ics<br/> /calendars/{email}/events-default/{uniqString}yandex.ru.ics<br/> /calendars/{email}/events-default/{uniqString}yandex.ru.ics<br/> ....</p><p>Отлично, мы научились вызывать API яндекса. Но это не совсем то что нас интересует. Чтобы получить события какого-то одного конкретного дня, согласно <a href="https://datatracker.ietf.org/doc/html/rfc4791#section-7.8.1" rel="noopener noreferrer nofollow">RFC4791 7.8.1</a><br/> необходимо выполнить следующий HTTP запрос:</p><p><strong>REPORT</strong> <a href="https://caldav.yandex.ru/calendars/%7Bemail%7D/events-default/" rel="noopener noreferrer nofollow">https://caldav.yandex.ru/calendars/{email}/events-default/</a><br/> Authorization: Basic {secret}</p><pre><code class="xml">    &lt;?xml version="1.0" encoding="utf-8" ?&gt;
    &lt;C:calendar-query xmlns:D="DAV:" xmlns:C="urn:ietf:params:xml:ns:caldav"&gt;    
      &lt;D:prop&gt;    
        &lt;D:getetag/&gt;    
          &lt;C:calendar-data&gt;    
          &lt;C:comp name="VCALENDAR"&gt;    
          &lt;C:prop name="VERSION"/&gt;    
            &lt;C:comp name="VEVENT"&gt;    
            &lt;C:prop name="SUMMARY"/&gt;    
            &lt;C:prop name="UID"/&gt;    
            &lt;C:prop name="DTSTART"/&gt;    
            &lt;C:prop name="DTEND"/&gt;    
            &lt;C:prop name="DURATION"/&gt;    
            &lt;C:prop name="RRULE"/&gt;    
            &lt;C:prop name="RDATE"/&gt;    
            &lt;C:prop name="EXRULE"/&gt;    
            &lt;C:prop name="EXDATE"/&gt;    
            &lt;C:prop name="RECURRENCE-ID"/&gt;    
          &lt;/C:comp&gt;    
          &lt;C:comp name="VTIMEZONE"/&gt;    
          &lt;/C:comp&gt;    
        &lt;/C:calendar-data&gt;    
      &lt;/D:prop&gt;    
      &lt;C:filter&gt;    
        &lt;C:comp-filter name="VCALENDAR"&gt;    
          &lt;C:comp-filter name="VEVENT"&gt;    
            &lt;C:time-range start="20060104T000000Z" end="20060105T000000Z"/&gt;    
          &lt;/C:comp-filter&gt;    
        &lt;/C:comp-filter&gt;    
      &lt;/C:filter&gt;    
    &lt;/C:calendar-query&gt;    
</code></pre><p>В теге <code>&lt;C:time-range start="20060104T000000Z" end="20060105T000000Z"/&gt;</code> мы указываем временной интервал, события которого мы хотим получить.<br/> В ответ приходит описание события/ий в соответствии с спецификацией. Осталось найти рабочую библиотеку, которая поддерживает <a href="https://datatracker.ietf.org/doc/html/rfc4791" rel="noopener noreferrer nofollow">RFC4791</a>, чтобы не парсить ответ в ручную. Достаточно быстро можно наткнуться на что-то вроде библиотеки <a href="https://github.com/ical4j/ical4j" rel="noopener noreferrer nofollow">ical4j</a>, которая умеет работать с caldav. Теперь у нас есть всё, чтобы начать писать наш плагин. В интернете достаточно информации, о том, как начать, поэтому заострять внимание на создании проекта для разработки плагина я не буду, просто оставлю ссылку на <a href="https://plugins.jetbrains.com/docs/intellij/developing-plugins.html#plugin-development-workflow" rel="noopener noreferrer nofollow">официальную документацию</a>.</p><p>Итак, для начала, определимся, какой UI интерфейс мы бы хотели. Для меня было бы удобно иметь список событий, отсортированных по времени в правой боковой панели(там где у нас окно с gradle, maven и т.д.).</p><p>Чтобы сделать такую панель, необходимо имплементировать интерфейс <strong>com.intellij.openapi.wm.ToolWindowFactory</strong> и реализовать метод <strong>createToolWindowContent(project: Project, toolWindow: ToolWindow)</strong>.<br/> Так же добавить описание в <strong>resources/META-INF/plugin.xml</strong> или зарегистрировать согласно <a href="https://plugins.jetbrains.com/docs/intellij/tool-windows.html#programmatic-setup" rel="noopener noreferrer nofollow">документации</a>. Простейшая реализация выглядит примерно так:</p><pre><code class="kotlin">
    class CalendarToolFactory : ToolWindowFactory {
    
        override fun createToolWindowContent(project: Project, toolWindow: ToolWindow) {
            val contentFactory = ContentFactory.getInstance()
            
            val content = contentFactory.createContent(
                JPanel(),
                "",
                false
            )
            toolWindow.contentManager.addContent(content)
        }
    
    }
</code></pre><p>Теперь необходимо создать саму панель(JPanel), которая будет содержать список с нашими событиями. Панель будет содержать лишь layout, который будет ответственен за размещение списка, и собственно сам элемент списка. Выглядит это примерно так:</p><pre><code class="kotlin">
    class CalendarPanel(private val list: JBList&lt;EventDataDto&gt;) : JPanel() {
    
        init {
            layout = FlowLayout(FlowLayout.LEFT).apply {
                add(list)
            }
            add(list)
        }
    
    }
</code></pre><p>Класс <strong>JBList&lt;*&gt;</strong> - это реализация обычного <strong>JList</strong>, а <strong>EventDataDto</strong> это наш класс, который содержит информацию о нашем календарном событии.<br/> Создадим свой собственный лист, унаследовав его от <strong>JBList&lt;*&gt;</strong>.</p><pre><code class="kotlin">
    @Service(Service.Level.PROJECT)    
    class CalendarList : JBList&lt;EventDataDto&gt;() {    
        init {    
            model = service&lt;CalendarListModel&gt;()    
            cellRenderer = service&lt;CalendarListCellRenderer&gt;()    
        }    
    }    
</code></pre><p>И здесь мы сталкиваемся с концепцией <a href="https://plugins.jetbrains.com/docs/intellij/plugin-services.html" rel="noopener noreferrer nofollow">сервисов</a>, которую стоит немного объяснить.<br/> По факту это некий DI в IntellijIdea sdk, с помощью которого мы можем заинжектить необходимый нам <s>bean</s> сервис в другой сервис.<br/> Сервисы могут быть 2 типов/скоупов: Project и Application. Разница между ними в том, что Application scope - это классический синглтон на всё приложение, в то время как Project scope -<br/> это синглтон в рамках проекта(ну или окна IDE). У Application сервиса конструктор не должен принимать никаких аргументов, а у Project, он может принимать объект<br/> типа <a href="https://github.com/JetBrains/intellij-community/blob/idea/243.22562.218/platform/core-api/src/com/intellij/openapi/project/Project.java" rel="noopener noreferrer nofollow">Project</a>, что часто весьма полезно, например, чтобы получить экземпляры других сервисов с этим же скоупом.</p><p>Так же в конструкторе <strong>CalendarList</strong> мы видим пример внедрения сервисов <strong>CalendarListModel</strong> и <strong>CalendarListCellRenderer</strong>, которые являются Application синглтонами, и ответственны за содержание и отображение нашего списка. Синглтонами они являются по причине того, что независимо от открытого окна IDE, мы хотим видеть одну и ту же актуальную информацию(Тут хочу обратить внимание, что сам CalendarList сделать синглтоном нельзя, т.к. он просто не будет отображаться на UI при открытии нескольких проектов - видимо такое ограничение).</p><p>Далее реализуем http client, для получения наших событий:</p><pre><code class="kotlin">    @Service
    class YandexRestClient {
        private val client = HttpClientBuilder.create().build()
    
        fun getTodayEvents(): Set&lt;EventDataDto&gt; {
            val email = getLogin()
            val password = getPassword()
    
            if (email.isNullOrBlank() || password.isNullOrBlank()) {
                return emptySet()
            }
    
            val secret = Base64.getEncoder().encodeToString(
                "$email:$password".toByteArray(Charset.forName("UTF-8"))
            )
    
            val url = "https://caldav.yandex.ru/calendars/$email/events-default"
            val request = HttpReport(
                url,
                CaldavRequestTemplate.template(),
                mapOf(Pair(HttpHeaders.AUTHORIZATION, "Basic ${secret}"))
            )
    
            val content = client.execute(request, BasicResponseHandler())
    
            return CaldavParser.toEvents(content) ?: emptySet()
        }
    
        private fun getLogin(): String? {
            val state = service&lt;StateService&gt;().state
            return state.login?.trim()
        }
    
        private fun getPassword(): String? {
            val state = service&lt;StateService&gt;().state
    
            return PasswordSafe.instance.getPassword(
                CredentialAttributes(ConfigStateDto::class.java.name, state.login, ConfigStateDto::class.java)
            )?.trim()
        }
    
    }
</code></pre><p>Наш клиент так же является синглтоном и использует apache http client, который по умолчанию уже есть в idea sdk. Единственный нюанс здесь заключается в том, что необходимо реализовать метод REPORT, т.к. он отсутствует, но делается это очень просто:</p><pre><code class="kotlin">    class HttpReport(url: String, body: String, headers: Map&lt;String, String&gt;) : HttpPost() {
        init {
            this.uri = URI(url)
            headers.forEach {
                this.addHeader(it.key, it.value)
            }
            this.entity = StringEntity(body)
        }
    
        override fun getMethod() = "REPORT"
    
    }
</code></pre><p>Чтобы сформировать тело запроса, создадим простой object, задачей которого является сформировать запрос на получение событий за сегодня(в теле запроса мы формируем список запрашиваемых полей, но почему-то яндекс это игнорирует и присылает в ответ всё. Честно сказать я просто не стал уделять много внимания этому моменту и скорее всего я просто что-то делаю не правильно):</p><pre><code class="kotlin">     object CaldavRequestTemplate {
        private val PATTERN = DateTimeFormatter.ofPattern("yyyyMMdd'T'HHmmss'Z'")
        private val zoneId = ZoneId.of("UTC")
    
        fun template(): String {
            val now = LocalDate.now()
            val startDay = now.atTime(LocalTime.MIN).atZone(zoneId) // начало дня
            val endDay = now.atTime(LocalTime.MAX).atZone(zoneId)   // конец дня
            
            return """
                &lt;?xml version="1.0" encoding="utf-8" ?&gt;
                &lt;C:calendar-query xmlns:D="DAV:"
                    xmlns:C="urn:ietf:params:xml:ns:caldav"&gt;
                    &lt;D:prop&gt;
                        &lt;D:getetag/&gt;
                        &lt;C:calendar-data&gt;
                            &lt;C:comp name="VCALENDAR"&gt;
                                &lt;C:prop name="VERSION"/&gt;
                                &lt;C:comp name="VEVENT"&gt;
                                    &lt;C:prop name="SUMMARY"/&gt;
                                    &lt;C:prop name="DTSTART"/&gt;
                                    &lt;C:prop name="DTEND"/&gt;
                                &lt;/C:comp&gt;
                                &lt;C:comp name="VTIMEZONE"/&gt;
                            &lt;/C:comp&gt;
                        &lt;/C:calendar-data&gt;
                    &lt;/D:prop&gt;
                    &lt;C:filter&gt;
                        &lt;C:comp-filter name="VCALENDAR"&gt;
                            &lt;C:comp-filter name="VEVENT"&gt;
                                &lt;C:time-range start="${startDay.format(PATTERN)}"
                                    end="${endDay.format(PATTERN)}"/&gt;
                            &lt;/C:comp-filter&gt;
                        &lt;/C:comp-filter&gt;
                    &lt;/C:filter&gt;
                &lt;/C:calendar-query&gt;
            """.trim()
        }
    }
</code></pre><p>Логин и пароль мы получаем из <strong>StateService</strong> сервиса, который использует механизм <a href="https://plugins.jetbrains.com/docs/intellij/persisting-state-of-components.html" rel="noopener noreferrer nofollow">state</a>.<br/> Чтобы хранить состояние вашего плагина, нужно реализовать типизированный интерфейс PersistentStateComponent&lt;*&gt;, где типом является dto объект, который и будет описывать<br/> хранимое состояние(На практике, как мне показалось, работает это не очень надёжно, т.е. если вы изменили состояние и мгновенно завершили работу, то не факт что оно сохранится.<br/> Я с этим сталкивался достаточно часто). Сам сервис выглядит очень просто:</p><pre><code class="kotlin">    @State(
         name = "dev.calendar.state.ConfigState",
         storages = [Storage("calendar.xml")]
    )
    @Service
    class StateService: PersistentStateComponent&lt;ConfigStateDto&gt; {
    
        private var state = ConfigStateDto()
    
        override fun getState() = state
    
        override fun loadState(loadedState: ConfigStateDto) {
            service&lt;SettingsPanel&gt;().applyState(loadedState)  // просто применяем сохранённое состояние для всяких checkBox и textField для панели с настройками
            this.state = loadedState
        }

    }
    
    class ConfigStateDto {
        var login: String? = null
        var enabled: Boolean = true
        var notificationTime: Int = 0
        var synchronizationFrequencyTime: Long = 5
    }

</code></pre><p>Для хранения sensitive data - в нашем случае поле password, idea предоставляет <a href="https://plugins.jetbrains.com/docs/intellij/persisting-sensitive-data.html" rel="noopener noreferrer nofollow">следующее решение</a>.</p><p>Для получения данных о событиях напишем собственный класс CaldavParser. Нам необходимо сначала извлечь тело ответа из xml, а уже после этого использовать <a href="https://github.com/ical4j/ical4j" rel="noopener noreferrer nofollow">ical4j</a>.<br/> Полностью выкладывать код я не буду, т.к. он достаточно объёмный, а задача не самая сложная. У меня получилось что-то вроде:</p><pre><code class="kotlin">    object CaldavParser {  
    // other methods...
        
    fun toEvents(content: String): Set&lt;EventDataDto&gt;? {
        if(content.isBlank()) {
            return emptySet()
        }
        val value: MultistatusDto? = mapper.readValue(content, MultistatusDto::class.java)

        val now = LocalDateTime.now().toLocalTime()
        return value?.response
            ?.asSequence()
            ?.map { it.propstat?.prop?.calendarData?.text }
            ?.filterNotNull()
            ?.map(this::toCalendar)
            ?.flatMap { it.getComponents&lt;VEvent&gt;("VEVENT").asSequence() }
            ?.map(this::toEventDataDto)
            ?.filter { it.endDate?.toLocalTime()?.isAfter(now) ?: false }
            ?.toSet()
    }

    private fun toCalendar(it: String): Calendar {
        return CalendarBuilder().build(StringReader(it.dropWhitespaces()))
    }

    private fun toEventDataDto(event: VEvent): EventDataDto {
        return EventDataDto().apply {
            startDate = event.getDateTimeStart&lt;ZonedDateTime&gt;().get().date
            endDate = event.getEndDate&lt;ZonedDateTime&gt;().get().date
            name = event.summary.get().value
            conference = getConferenceLink(event)
            conferenceType = getConferenceType(event)
            description = event.description.getOrNull()?.value
            uid = event.uid.getOrNull()?.value
        }
    }
</code></pre><p>На данный момент наш плагин уже имеет панель для отображения событий календаря, и может эти самые события получать. Теперь попробуем соединить это воедино.<br/> Ранее уже упоминалось, что за отображение ответственен класс <strong>CalendarListModel</strong>. Чтобы его написать, нам надо унаследовать его от стандартного <strong>DefaultListModel</strong><br/> и просто добавить логики, которая будет отвечать за добавление/обновление/удаление. Логика в нём достаточно простая, мы храним список отсортированных по дате событий<br/> и переопределяем существующие(ну или пишем свои собственные методы добавления/удаления), поэтому расписывать подробно всё не имеет особого смысла.</p><pre><code class="kotlin">    @Service
    class CalendarListModel : DefaultListModel&lt;EventDataDto&gt;() {

</code></pre><p>Теперь осталось периодически опрашивать календарь и отображать события. Задача достаточно простая, но как это сделать в intellij idea правильно, я не знаю (мой последний вопрос в саппорт остался без ответа, да и зайти туда стало возможно только из под впн). Поэтому ничего лучше, чем использовать <a href="https://plugins.jetbrains.com/docs/intellij/plugin-listeners.html" rel="noopener noreferrer nofollow">Listeners</a> я не придумал.<br/> Idea предоставляет достаточно гибкий механизм событий - возможность создавать собственные события или привязываться к уже существующим.</p><p>Для собственного listener'а создадим новый класс CalendarActivationListener и заимплементим интерфейс AppLifecycleListener.</p><p>Далее нам необходимо зарегистрировать наш класс в <strong>resources/META-INF/plugin.xml</strong></p><pre><code>&lt;applicationListeners&gt;
    &lt;listener class="dev.calendar.listener.app.CalendarActivationListener" topic="com.intellij.com.intellij.ide.AppLifecycleListener"/&gt;
&lt;/applicationListeners&gt;
</code></pre><p>Затем переопределить метод <strong>override fun appFrameCreated(commandLineArgs: MutableList)</strong> , который будет выполняться каждый раз, когда наша IDE запускается.</p><pre><code class="kotlin">    class AppActivationListener : AppLifecycleListener {

    override fun appFrameCreated(commandLineArgs: MutableList&lt;String&gt;) {
        service&lt;SchedulerService&gt;().startNotification(
            service&lt;StateService&gt;().state.notificationFrequencyTime
        )
        service&lt;SchedulerService&gt;().startCalendarSynchronization(
            service&lt;StateService&gt;().state.synchronizationFrequencyTime
        )
    }

}
</code></pre><p><strong>SchedulerService</strong> реализует выполнение повторяющейся логики, которую можно конфигурировать через настройки, т.е. менять частоту опроса API и частоту проверка случившихся событий.</p><pre><code class="kotlin">@Service
class SchedulerService {
    private var calendarSyncFuture: ScheduledFuture&lt;*&gt;? = null
    private var notificationFeature: ScheduledFuture&lt;*&gt;? = null

    fun startCalendarSynchronization(synchronizationFrequencyTime: Long) {
        calendarSyncFuture?.cancel(true)
        calendarSyncFuture = AppExecutorUtil.getAppScheduledExecutorService().scheduleWithFixedDelay(
            {
                service&lt;CalendarService&gt;().updateCalendar()
            },
            0,
            synchronizationFrequencyTime,
            TimeUnit.MINUTES
        )
    }

    fun startNotification(notificationFrequencyTime: Long) {
        notificationFeature?.cancel(true)
        notificationFeature = AppExecutorUtil.getAppScheduledExecutorService().scheduleWithFixedDelay(
            {
                service&lt;NotificationService&gt;().run()
            },
            5,
            notificationFrequencyTime,
            TimeUnit.SECONDS
        )
    }
}

    }
</code></pre><p>Сам класс необходимо добавить к нашему CalendarList в качестве cellRenderer(мы это уже сделали).</p><p>В завершении можно прикрутить UI для конфигурации нашего плагина - чтобы иметь возможность из настроек задавать логин/пароль, а так же различные другие переменные.<br/> Чтобы это сделать, нужно имплементировать интерфейс com.intellij.openapi.options.SearchableConfigurable и зарегистрировать его в <strong>resources/META-INF/plugin.xml</strong>.</p><pre><code>    &lt;applicationConfigurable displayName="YCalendar" instance="dev.calendar.configurable.SettingsConfigurable"/&gt;
</code></pre><p>Главными здесь являются методы <strong>apply()</strong> и <strong>createComponent()</strong>.     }
</code></pre><p>SettingsPanel является наследником JPanel, и представляет собой обычный компонент с layout и различными UI компонентами, логика там достаточна простая.</p><p>Для меня самым сложным оказалось красиво расставить все компоненты с помощью GridBagLayout.
