href="https://amplicode.ru/?utm_source=habr&amp;utm_medium=article&amp;utm_campaign=whats-new-2024.2">Amplicode</a> в 2024 году успешно состоялся. В этот релиз вошло большое количество улучшений, исправлений и новых фич. Вот некоторые из них:</p><ul><li><p>Gutter Icons для Spring-объектов</p></li><li><p>Улучшенная генерация Spring Web тестов</p></li><li><p>Поддержка Extends и Include в Docker Compose</p></li><li><p>Генерация отдельных HTTP-методов для REST-контроллеров</p></li><li><p>Поддержка Kubernetes и Helm</p></li></ul><h2>Gutter Icons – один клик до самых важных действий  </h2><p>Теперь вместо Editor Toolbar в Spring-объектах, JPA-сущностях и Spring Boot тестах вы увидите <a href="https://www-jetbrains-com.translate.goog/help/idea/settings-gutter-icons.html?_x_tr_sl=en&amp;_x_tr_tl=ru&amp;_x_tr_hl=ru&amp;_x_tr_pto=sc"><u>gutter icons</u></a> (gutter иконки) напротив названий классов. Одна из них содержит часто используемые действия, ранее доступные в Editor Toolbar.</p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/b0f/241/b67/b0f241b6709a9264c8fc266c55d3068b.png" height="1546" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/b0f/241/b67/b0f241b6709a9264c8fc266c55d3068b.png" width="2952"/></figure><p>Кроме иконки с часто используемыми действиями, появился ещё один gutter icon –  Bean Navigation, показывающий взаимодействие текущего объекта с другими бинами. </p><p>Из Bean Navigation можно увидеть какие бины заинжектированы в текущий бин и наоборот. Это касается как проектных бинов, так и тех, что используются из сторонних библиотек. Кроме того, если инжектирован интерфейс, можно увидеть всех кандидатов для инжекции. </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/732/5de/859/7325de8598992c88bb164f91037d43a8.png" height="2032" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/732/5de/859/7325de8598992c88bb164f91037d43a8.png" width="2852"/></figure><p>Подробнее про gutter иконки от Amplicode я рассказал в отдельном видео. Посмотрите его, чтобы увидеть эту фичу в действии: </p><div class="tm-iframe_temp" data-src="https://embedd.srv.habr.com/iframe/66b2056b6748d7d46fe0ab79" data-style="" id="66b2056b6748d7d46fe0ab79" width=""></div><h2>Улучшенная генерация Spring Web тестов  </h2><p>Мы получили много запросов в наш <a href="https://t.me/amplicode_chat"><u>телеграм-чат</u></a><u> </u>с просьбами улучшить генерацию тестов для Spring Web. Ваша активность помогла нам приоритизировать эту функциональность. </p><p>Улучшения в генерации тестов можно разделить на две части: переработка UI и улучшение кода. </p><p><strong>Изменения в UI:</strong> </p><ol><li><p>Теперь методы отсортированы так же, как и в контроллере. </p></li><li><p>Рядом с каждым методом отображается связанный с ним путь. </p></li><li><p>Элементы для работы с параметрами запроса и телом запроса переработаны, чтобы соответствовать поведению большинства HTTP-клиентов. </p></li><li><p>PathVariable и RequestParam теперь можно легко отличить благодаря понятным подписям. </p></li></ol><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/747/04f/bba/74704fbbac3e57e874af5f0d5d06155c.png" height="2036" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/747/04f/bba/74704fbbac3e57e874af5f0d5d06155c.png" width="2402"/></figure><p><strong>Изменения в кодогенерации:</strong> </p><ol><li><p>В объекты типа String, представляющие собой JSON, автоматически инжектируется язык JSON, за счёт чего содержимое отображается как JSON файл, а не как обычная строка. </p></li><li><p>Уменьшено количество генерируемых локальных переменных. </p></li><li><p>Исправлен баг с некорректной передачей нескольких id в запросах. </p></li></ol><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/b74/2d6/82d/b742d682db8e7160eb6b424aaf97b4a7.png" height="1158" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/b74/2d6/82d/b742d682db8e7160eb6b424aaf97b4a7.png" width="1758"/></figure><h2>Частичная генерация CRUD REST контроллера </h2><p>В предыдущем релизе Amplicode появилась возможность создать <a href="https://youtu.be/vlqu-9TV4BQ?feature=shared"><u>CRUD Rest Controller для любой JPA сущности</u></a>. Теперь можно создавать отдельные методы для REST-контроллеров, а не только весь контроллер целиком. Это особенно удобно, когда нужно добавить новый эндпоинт в уже существующий контроллер. </p><p>Действия по созданию отдельных эндпоинтов доступны в Gutter Icon (1), панели Amplicode Designer (2) и меню «Generate» (3): </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/b23/eb9/b11/b23eb9b11f85ffbd05be6887fe33f1b2.png" height="1934" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/b23/eb9/b11/b23eb9b11f85ffbd05be6887fe33f1b2.png" width="3106"/></figure><p>Для каждого HTTP-метода можно указать или создать JPA Repository, DTO/Domain Object, Proxy Service и Request Path.  </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/9d4/a03/1b0/9d4a031b08ecb44db96c2614bb9978d0.png" height="816" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/9d4/a03/1b0/9d4a031b08ecb44db96c2614bb9978d0.png" width="1208"/></figure><h2>Переиспользование сервисов в Docker Compose  </h2><p>Docker Compose предлагает несколько способов переиспользования сервисов из одного файла в другой. Один из таких способов подразумевает использование <a href="https://docs.docker.com/compose/multiple-compose-files/include/"><u>include</u></a>. </p><p>Ключевое слово include позволяет включить один Docker Compose файл в другой. Этот способ включения одного файла в другой удобен, когда нужно просто переиспользовать одни и те же сервисы без дополнительной конфигурации. </p><p>Amplicode знает про Include в Docker Compose и учитывает его для отображения элементов в панелях Amplicode Explorer и Structure: </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/d94/635/efe/d94635efe050d26de1a180656742f469.png" height="1802" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/d94/635/efe/d94635efe050d26de1a180656742f469.png" width="3126"/></figure><p>Но вот если вам нужно настроить переиспользуемые сервисы под свои нужды —   лучше использовать ключевое слово <a href="https://docs.docker.com/compose/multiple-compose-files/extends/"><u>extends</u></a>. С его помощью, как и с помощью include, можно включить сервис из одного Docker Compose файла в другой, но при этом переопределить любые его свойства.  </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/3c1/77a/618/3c177a6185091f37a35cd4a8691391ee.png" height="1924" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/3c1/77a/618/3c177a6185091f37a35cd4a8691391ee.png" width="3126"/></figure><p>Amplicode поддерживает эту возможность не только с точки зрения визуального отображения, но и помогает непосредственно во время расширения сервисов. </p><p>Теперь в меню «Generate» вы найдете не только действия для создания сервисов с нуля и рекомендации по генерации сервисов на основе уже объявленных в текущем compose файле, но и действие «Extends Existing Service»: </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/3a3/843/a34/3a3843a3488b38f3a57a1db6bda188a9.png" height="1924" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/3a3/843/a34/3a3843a3488b38f3a57a1db6bda188a9.png" width="3126"/></figure><p>Что самое потрясающее, панель Amplicode Designer в том числе будет учитывать те свойства, которые не объявлены для сервиса в текущем файле, но объявлены в том, на который мы ссылаемся: </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/884/19b/36c/88419b36ce9019a3d5db4f05e157e382.png" height="1924" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/884/19b/36c/88419b36ce9019a3d5db4f05e157e382.png" width="2328"/></figure><p>Подробнее про эту фичу я рассказал в отдельном видео: </p><div class="tm-iframe_temp" data-src="https://embedd.srv.habr.com/iframe/66b2067a315cc3d4a14db07a" data-style="" id="66b2067a315cc3d4a14db07a" width=""></div><h2>Поддержка Kubernetes и Helm  </h2><p>Docker Compose отлично подходит для настройки окружения и деплоя нескольких сервисов на одной машине. Однако для продакшена часто нужны более мощные инструменты, такие как Kubernetes. </p><p>Теперь Amplicode поддерживает Kubernetes и помогает генерировать Helm чарты! Вы можете создавать как пустые Helm чарты, так и готовые чарты для PostgreSQL, Kafka и Spring Boot приложений. </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/9ec/afb/bfb/9ecafbbfb0998abb0a6620e596519046.png" height="2132" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/9ec/afb/bfb/9ecafbbfb0998abb0a6620e596519046.png" width="2436"/></figure><p>Кроме того, запуск чартов стал еще проще – теперь это можно делать прямо из панели Amplicode Explorer, без необходимости обращаться к консоли:  </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/246/d40/77d/246d4077d165d11a3a17015120581323.png" height="1450" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/246/d40/77d/246d4077d165d11a3a17015120581323.png" width="1562"/></figure><p>Наконец, состояние запущенных чартов можно проверять в панели Services:  </p><figure class="full-width"><img data-src="https://habrastorage.org/getpro/habr/upload_files/311/46c/797/31146c79712dd81a5e106311b1013131.png" height="1714" src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/311/46c/797/31146c79712dd81a5e106311b1013131.png" width="3080"/></figure><p>Можно сказать, что поддержка Kubernetes находится на стадии preview, и мы будем рады вашей обратной связи, чтобы улучшить функциональность в соответствии с вашими потребностями :)  </p><h2>И это ещё не всё!  </h2><p>На самом деле, это далеко не всё, что вошло в релиз Amplicode 2024.2. Помимо крупных новых фич, мы внесли множество мелких улучшений и исправлений – всего было закрыто <strong>более 170 тикетов</strong>! </p><p>Напомню, что текущая версия Amplicode на данный момент доступна для 2-х версий IntelliJ IDEA: 2023.3, 2024.1. Поддержать IntelliJ IDEA 2024.2  мы планируем в ближайшее время. Если вы испытываете трудности с переходом на новые версии IntelliJ IDEA, вы всё ещё можете скачать и уставновить <a href="https://github.com/Amplicode/amplicode/blob/main/releases/2024.1.md"><u>Amplicode 2024.1</u></a>, он доступен для бóльшего количества версий IntelliJ IDEA.  </p><p>Огромное спасибо всем, кто делится своим опытом использования Amplicode в нашем <a href="https://t.me/amplicode_chat"><u>телеграм-чате</u></a>.
